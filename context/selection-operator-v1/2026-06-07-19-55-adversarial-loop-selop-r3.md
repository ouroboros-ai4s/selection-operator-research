## 2a — 攻击 + 不变量体检(multiagent-debate: critic-defender-judge)

**【CRITIC 攻击】**(站在"条件依赖可能是装饰"一方)

指控 1(打乱对照可能假阴性):你说打乱抗原标签后 f̂ 性能会掉,所以证明条件依赖真。但 CR9114 只有 3 个抗原(H1/H3/B)。打乱 3 个标签,信息损失很小——模型可能靠"序列→平均亲和力"就拿到大部分性能,打乱后只掉一点点,你怎么定"显著掉"?3 个条件的打乱对照统计功效太弱,可能假阴性(条件依赖真存在但打乱没测出来),也可能假阳性(随机波动看着像掉)。

指控 2(条件依赖 vs 抗原相似度混淆):H1/H3 都是 group 1 流感,B 是另一谱系。f̂ 在 H1+H3 学、预测 B,如果预测对了,可能只是因为"B 与 H1/H3 有共享表位",f̂ 学的是"通用结合"不是"条件特异选择"。你的 cross-condition(CV-c)证明的可能是"抗原间相似性"而非"条件依赖机制"。

指控 3(P09/P10 该不该留):你想砍 Hanke(P09)和 SVD(P10)。但如果 CR9114 三抗原的条件依赖检验如指控 1 说的统计功效弱,你恰恰需要 Hanke 的多条件(更多 condition)来补足条件轴的丰富度;砍了 P09,I3 就只能靠 3 抗原,脆弱。SVD(P10)正是用来诊断"条件轴有效秩够不够"的——你砍了它,怎么知道 3 个抗原不是冗余的(有效秩可能只有 1-2)?

---

**【DEFENDER 应战】**(弹药:CR9114 nested 结构 + Phillips 2021 发现 + r2 分区裁决)

回应指控 1(打乱对照功效):关键不在抗原数量(3个),在 CR9114 的 **nested/hierarchical 结构**(Phillips 2021 核心发现)。CR9114 对 B 的亲和力需要 H1/H3 不需要的特定 nested 突变组合——这意味着"哪些突变组合在哪个抗原下有效"是强条件依赖的,不是"平均亲和力 + 小扰动"。打乱抗原标签会把"对 B 有效的 nested 组合"错配到 H1/H3,模型必须学一个自相矛盾的映射,性能会大幅掉(不是小幅)。统计功效不靠抗原数,靠 nested 结构的强度——而 Phillips 已证 CR9114 的 nested 极强(这正是它被选为数据源的原因)。补充:打乱可做多次随机置换построить null 分布,看真实性能是否落在尾部,功效可量化。

回应指控 2(相似度 vs 条件依赖):这正是 nested 结构的鉴别力所在。如果 f̂ 只学"通用结合/抗原相似度",它无法预测 B 的 nested 组合需求(因为 nested 是 B 特有的、H1/H3 没有的高阶 epistasis)。CV-c 预测 B 成功 + CV-d 同变体跨抗原方向一致,共同排除"纯相似度"——纯相似度模型在 B 的 nested 位点上必然预测错方向。再加 N3 baseline(序列相似度模型)对照:如果 f̂ 只是相似度,它不会显著打败序列相似度 baseline。N3 正是为这个指控设计的。

回应指控 3(P09/P10):
- P10 SVD:critic 说得有道理但结论反了。CR9114 的条件依赖**不体现在"条件矩阵的秩"上,而体现在"跨抗原的 epistasis 差异"上**。SVD 秩诊断是为"多条件 panning 矩阵(P09 那种)"设计的,对 CR9114 的 nested DMS 不适用——3 抗原的秩本来就低(≤3),但条件依赖照样强(在 epistasis 结构里)。所以 P10 不是"砍了就不知道秩够不够",而是"它诊断的指标(秩)对 CR9114 根本不是正确指标"。用 nested epistasis 的跨抗原差异(CV-a/d)替代 SVD 诊断更对题。
- P09 Hanke:承认 Hanke 能补条件轴丰富度。但**核心方法验证(I1/I2/I3 在完备真值上成立)用 CR9114 已充分**;Hanke 是 phage panning(轮次×条件),无完备 Kd 真值,它能加"条件多样性"但不能加"验证严格性"。所以 Hanke 降为可选补充(开源加分项),不进核心脊柱。这与 r2 的 CS 声明一致——核心验证在 CR9114 完备区,Hanke 属"更多条件的探索性扩展"。

**P09 Hanke 去留裁决**:**降为可选补充(移出核心脊柱,标注为开源探索性扩展)**。核心方法验证用 CR9114 完备真值已充分;Hanke 加条件多样性但不加验证严格性(无完备 Kd)。不是砍(它有补充价值),是降级出核心。

## 2b — 张力提炼(deep-insight → problem-reformulation: tension-mining)

**候选张力筛查**:
- R4 的对抗中,critic 的核心是"3 抗原够不够撑条件依赖 claim"。这在 2a 被 defender 用 nested 结构化解,但留下一个张力——

**本轮核心张力(精确表述)**:

> **T-r3:条件依赖的"统计功效" ⟂ 条件维度的"完备真值"。** 加条件维度(Hanke 多条件)能提高条件依赖检验的统计功效(更多 condition → 打乱对照更有力),但 Hanke 没有完备 Kd 真值(phage panning 是富集计数,非 Kd);用 CR9114(完备真值)则只有 3 抗原(条件维度少)。**功效(多条件)与真值完备性(CR9114)在开源数据上不可兼得。**

**卡在哪个部件**:卡在 **CV-b 打乱对照的统计设计**上。解法不是加 Hanke(那牺牲真值完备性),而是在 CR9114 的 3 抗原上**用多次随机置换построить null 分布**来榨取统计功效——把"3 个抗原"的少量条件,通过"在 65536 变体 × 3 抗原的完备矩阵上多次置换"转化为有统计功效的检验。功效来自变体数(65536)× 置换次数,不只来自抗原数(3)。

**张力的意义**:T-r3 揭示了一个反直觉的点——条件依赖的统计功效**不必来自条件数量**,可以来自"完备真值矩阵上的置换检验"。这让方案不必为了功效引入 Hanke(从而牺牲真值完备性),保持了极简(只用 CR9114)。这是 2a 裁决"砍 P10、降 P09"背后的张力根源:开源数据逼我们在"多条件但无完备真值"和"完备真值但少条件"间选,loop 选了后者 + 置换检验补功效。

### 2b 产出

- **核心张力 T-r3**:条件依赖统计功效 ⟂ 条件维度完备真值,开源不可兼得;解法 = 用 CR9114 完备矩阵上的多次置换检验榨功效,不靠加条件数。
- **卡点**:CV-b 统计设计——功效来自变体数×置换次数(65536×N),非抗原数(3)。
- **意义**:证明极简(只用 CR9114)不牺牲 I3 功效,无需引入 Hanke。支撑 2a 的 P09 降级、P10 砍。

## 2c — 简化候选生成(creative-ideation → assumption-destruction + systematic-enumeration)

针对 T-r3 生成 v3。本轮是收尾轮,变动小(砍 P10、降 P09、CV-b 加置换设计)。

### assumption-destruction

- 假设"条件依赖功效需要多条件"→ 摧毁(T-r3)。功效可从完备矩阵的置换检验得,变体数×置换次数。
- 假设"SVD 秩诊断是条件轴有效性的通用工具"→ 摧毁。对 nested DMS,条件依赖在 epistasis 差异里,不在矩阵秩里。秩诊断不对题。
- 假设"更多开源数据=更强验证"→ 摧毁。Hanke 加条件多样性但减真值完备性;对核心验证(I1/I2/I3 在完备真值上)反而是稀释。

### systematic-enumeration(还能更简吗)

1. CV 模块四子检验(a/b/c/d)能否再合并?→ 检验。CV-a(扫条件)与 CV-d(同变体跨抗原方向)高度重叠(都是"看 f̂ 跨抗原变化 vs 实测")。可合并为一个"跨抗原一致性检验"。CV-b(打乱)与 CV-c(cross-condition 预测)是两个独立的强检验(一个反向排除、一个正向预测),不合并。→ CV 从 4 子检验精简为 3:CV-Ⅰ跨抗原一致性(原a+d)、CV-Ⅱ打乱对照(原b,加置换)、CV-Ⅲ cross-condition预测+N3(原c)。

3. P03 σ 模型本体还需独立列吗?→ 它已是 P07 内部步骤(扣突变)的依赖,且 Spike0 的 censored 不需要它。降为 P07 内部依赖,不独立列。

## 2d — 抢救被砍部件(stress-test → counterfactual-probing: necessity-sufficiency + factor-removal)

本轮砍 P10、降 P09/P03、合并 CV-a+d。逐个 resurrection。

**P10 SVD 秩诊断(砍)**:
- 抢救论证:需诊断条件轴有效秩,确认 3 抗原不是冗余(秩≥2)。
- necessity-sufficiency:PN——砍 P10,I3 是否塌?否。I3 靠 CV-Ⅱ 打乱对照(nested 结构置换检验)验条件依赖,不靠秩。CR9114 的条件依赖在 epistasis 跨抗原差异里,秩诊断测不到这个。PS——P10 在场是否保证 I3?否,秩高不等于条件依赖强(反例:3 抗原线性独立秩=3 但若 epistasis 跨抗原相同则无条件依赖)。
- factor-removal:砍 P10 → I3 不塌(CV-Ⅱ 承担),且避免了"秩=3 误判为条件依赖强"的假阳风险。
- **裁决:P10 真冗余,砍。** 秩诊断对 nested DMS 是错误指标。

**P09 Hanke(降级)**:
- 抢救论证:Hanke 多条件能提升条件依赖检验功效,且是真实生物数据(非 DMS)。
- necessity-sufficiency:PN——把 Hanke 移出核心(降可选),I1/I2/I3 是否塌?否。核心验证用 CR9114 完备真值已充分(I3 功效由置换检验榨取,T-r3)。Hanke 加多样性不加严格性。PS——Hanke 在核心是否有害?轻度——它无完备 Kd 真值,若当核心验证会稀释"在完备真值上验证"的严格性,且引入 phage panning 的富集计数噪声(非 Kd)。
- factor-removal:Hanke 移出核心 → 三不变量不塌(CR9114 撑全)。
- **裁决:P09 降级正确(非砍,保留为可选探索性扩展)。** 它有补充价值(更多条件、贴近真实 panning),但不属核心方法验证。

**P03 σ 模型(降为 P07 内部依赖)**:
- 抢救论证:σ 模型是 I2 的"已知扩散"前提,承重。
- necessity-sufficiency:PN——σ 模型作为"独立部件"移除(降为 P07 内部)后 I2 是否塌?否,σ 仍在(作为 P07 玩具 SDE 的已知扩散输入 + 扣突变步骤),只是不独立列为部件。PS——独立列是否有额外价值?无。
- **裁决:P03 独立性冗余,功能保留为 P07 内部依赖。** 归属转移,非砍。

**CV-a + CV-d 合并为 CV-Ⅰ**:
- 抢救论证:扫条件(a)与跨抗原方向(d)是两个不同检验,不该合并。
- necessity-sufficiency:PN——合并后是否丢失检验能力?否。两者都验"f̂ 跨抗原变化是否与实测 nested 一致",CV-a 是定性扫描、CV-d 是定量方向一致,合并为 CV-Ⅰ 同时含定性+定量,无能力损失。
- **裁决:合并正确,无能力损失。**

## Checkpoint: Stage 2 r3 完成(I3 判绿,三不变量首次全绿,砍 P10,v3 接近定稿,需 r4 确认轮)

### Objective

r3 主攻 R4 条件依赖(I3),目标把 I3 从 🟡 正式裁绿——确立 CV 模块的打乱抗原标签对照能真正证明"f(seq,cond) 用上了条件、非退化成 f(seq)"。同时收尾两个待审部件 P09 Hanke 与 P10 SVD。r3 是预期的收敛轮:R1/R2/R3 已在前两轮裁决,本轮裁 R4 后,四个核心维度(R1-R4)全决,只剩 R5 部署项(不阻塞 saturation)。

### Process Summary

2a:critic 三项攻击(3 抗原打乱对照统计功效弱 / cross-condition 可能是抗原相似度非条件依赖 / P09P10 不该砍)→ defender 用 CR9114 nested epistasis 结构(Phillips 2021 核心发现)+ N3 baseline + r2 分区裁决应战 → judge 裁 I3 绿(有条件:多次置换)、P10 砍、P09 降级。2b:抽张力 T-r3(统计功效 ⟂ 完备真值不可兼得;置换检验榨功效)。2c:CV 模块精简 4→3 子检验,砍 P10,降 P09/P03,生成 v3。2d:resurrection——P10 真冗余确认,P09/P03 降级正确,CV-a+d 合并无损失,复活清单空。saturation 判据2(全绿)✅、判据3(砍不动)基本✅、判据1(连续2轮无复活)需 r4 确认。

### Key Findings

**发现 1(条件依赖的鉴别力来自 nested 结构,非条件数量)**:critic 的"3 抗原统计功效弱"是直觉上有力的攻击——通常条件依赖检验靠条件维度数量。但 defender 的反击揭示了 CR9114 的特殊性:它的条件依赖鉴别力来自 nested/hierarchical epistasis 结构强度(Phillips 2021 证 CR9114 对 B 的亲和需要 H1/H3 不需要的特定 nested 突变组合),不是抗原数量。打乱抗原标签会把"对 B 有效的 nested 组合"错配到 H1/H3,模型被迫学自相矛盾的映射,性能大幅掉。这把"功效"的来源从"条件数"转移到"nested 结构强度 + 完备矩阵置换",是 r3 的关键认知。

**发现 2(T-r3:功效与完备真值的开源不可兼得 + 置换检验解法)**:开源数据逼一个选择——Hanke 有多条件(高功效)但无完备 Kd 真值;CR9114 有完备真值但只 3 抗原(低条件数)。loop 选 CR9114 + 置换检验:在 65536 变体 × 3 抗原的完备矩阵上多次随机置换抗原标签построить null 分布,功效来自变体数 × 置换次数(65536 × N),不靠抗原数(3)。这让方案保持极简(只用 CR9114)而不牺牲 I3 统计功效。这是"反直觉的简化"——通常会以为"少条件=弱检验",但完备矩阵的置换检验绕过了这个限制。

**发现 3(SVD 秩诊断对 nested DMS 不对题)**:老方案 Layer 2 的 SVD 条件有效秩诊断(P10)是为"多条件 panning 矩阵"设计的,判"条件轴有效秩够不够"。但对 CR9114 的 nested DMS,条件依赖体现在"跨抗原的 epistasis 差异",不在"条件矩阵的秩"。3 抗原秩必然 ≤3(低),但条件依赖照样强(在高阶 epistasis 里)。所以 SVD 不仅冗余,还会误导(秩低被误判为条件依赖弱,或秩=3 被误判为条件依赖强)。其意图(确认条件轴有信息)由 CV-Ⅰ 的 nested 跨抗原差异更对题地承担。这是 r3 砍 P10 的依据。

**发现 4(CV 模块精简到 3 子检验)**:CV-a(扫条件)与 CV-d(同变体跨抗原方向)高度重叠,合并为 CV-Ⅰ(跨抗原一致性,含定性扫描 + 定量方向)。CV-b→CV-Ⅱ(打乱对照,加多次置换)、CV-c→CV-Ⅲ(cross-condition 预测 + N3)是两个独立强检验,保留。CV 从 4 子检验精简为 3,无能力损失。

**发现 5(三不变量首次全绿)**:r3 末,I1🟢/I2🟢/I3🟢 首次全部绿(均带 Stage 4 必验条件)。这满足 saturation 判据2。同时 R1-R4 全决、核心件无冗余(2c enumeration 确认),判据3 基本满足。只差判据1(连续2轮无复活)——r3 是 r2 复活 P07 后的第 1 个无复活轮,需 r4 再确认一轮。

### v2 → v3 变化

砍 P10 SVD 秩诊断(对 nested DMS 不对题)。降级 P09 Hanke(移出核心,可选探索性扩展)、P03 σ模型(降为 P07 内部依赖)。CV 模块精简 4→3 子检验(CV-a+d 合并为 CV-Ⅰ)。CV-Ⅱ 加强制多次置换 null 分布 + p 值。v3 比 v2 更精炼(砍 1 降 2 合并 1),核心脊柱稳定为 5 件:Spike0 / P07 / CV(3子) / P16 / CS。

### Decisions Made

1. **I3 判绿,条件 = CV-Ⅱ 多次随机置换 null 分布 + p 值量化**。理由:回应 critic 的统计功效关切,把"打乱一次看掉多少"升级为"多次置换 + 量化 p 值"。
2. **P10 SVD 砍**。理由:秩诊断对 nested DMS 是错误指标,意图由 CV-Ⅰ 承担。
3. **P09 Hanke 降为可选扩展(非砍)**。理由:加条件多样性但不加验证严格性(无完备 Kd），核心验证 CR9114 已充分;保留补充价值。
4. **P03 σ模型降为 P07 内部依赖**。理由:Spike0 censored 不需要 σ,σ 只在 P07 玩具 SDE 用,不独立列。
5. **CV 4→3 精简**。理由:CV-a/d 重叠,合并无损失。

### 附录 r3-A:CV 模块最终规格(3 子检验,Stage 4 直接操作化)

r3 把 CV 精简为 3 子检验,这里给最终规格,Stage 4 实验设计直接取用。

**CV-Ⅰ 跨抗原一致性检验(原 CV-a + CV-d 合并,验 I3 正向 + I1 真值复现)**:
- 定性(原 a):固定一组变体,扫 H1/H3/B,观察 f̂(x,·) 跨抗原变化模式。
- 定量(原 d):同变体跨抗原的 f̂ 差,与实测 −logK_D 差的方向一致率。
- 通过判据:f̂ 跨抗原变化方向与实测 nested 模式的一致率显著(预注册阈值);且复现 Phillips 报告的"对 B 需 nested 组合"模式。
- 服务:I3(条件依赖正向证据)+ I1(条件依赖在真值上复现)。

**CV-Ⅱ 打乱抗原标签对照(原 CV-b,验 I3 反向排除,核心)**:
- 步骤:把训练集 (变体, 抗原, −logK_D) 的抗原标签随机置换,重训 f̂_shuffled,记录测试性能。重复 N 次(N≥100)построить null 分布。
- 通过判据(r3 强化):真实 f̂ 的性能须落在 null 分布尾部(p < 0.05),即 f̂_shuffled 显著劣于 f̂。
- 服务:I3 核心(证 f 用上条件,非退化 f(seq))。
- r3 强化点:不是"打乱一次看掉多少",是"多次置换 + 量化 p 值",功效来自 65536 变体 × N 置换。

**CV-Ⅲ cross-condition 预测 + N3 baseline(原 CV-c,验 I1 核心)**:
- 步骤:在 H1+H3 上训练 f̂,在留出抗原 B 上预测变体相对 −logK_D 排序。
- 通过判据(双重):(1) 预测 Spearman 显著 > 0;(2) 显著优于 N3 baseline(序列相似度 + Phillips pairwise epistasis 模型)。
- 服务:I1 核心(非循环 + 机制鉴别力,断伪非循环)。
- 前提:P16 零泄漏(B 抗原 + 对照突变组合未进训练)。

**CV 模块综合输出**:I1 判定(CV-Ⅲ 过)+ I3 判定(CV-Ⅱ 过 + CV-Ⅰ 一致)。三子检验覆盖 I1+I3,在单一 CR9114 数据上完成,无需 Hanke。
