## 2a — 攻击 + 不变量体检(multiagent-debate: critic-defender-judge)

### 主攻:R2 仿真器必要性(精确化为 T-core)+ R3 可辨识性。被告:Spike0(P15+P11)的充分性 / P07 玩具SDE去留。

核心问题(T-core):Spike0 在 CR9114 上直接监督拟合静态 f(seq,cond),是否充分支撑 North Star 的核心 claim「学到了驱动亲和力成熟的选择算子(动力学量 ∇f)」?还是必须保留一个"从演化反推 ∇f"的动力学步骤?

---

**【CRITIC 攻击】**(站在"Spike0 不充分、偷换问题"一方)

指控 1(静态 ≠ 动力学):North Star 白纸黑字——选择算子是 SDE `dX=∇f·dt+σdW` 里的 drift,一个**动力学驱动力**。Spike0 拟合的是 CR9114 的平衡解离常数 −logK_D,这是**终点热力学量(平衡亲和力)**,不是动力学量。热力学平衡态 ≠ 演化驱动力。你把"学选择算子"偷换成了"拟合一张静态适应度地形",这是普通 DMS 回归,任何蛋白突变扫描论文都做过,毫无新颖性,且根本不是反问题。

指控 2(丢失动力学信息):真实选择含静态地形抓不到的东西——(a) frequency-dependent selection(频率依赖:某变体适应度随其在群体中频率变化,克隆竞争);(b) 路径/历史依赖(到达某状态的路径影响后续选择);(c) 时变选择压(免疫过程中抗原浓度/竞争随时间变)。CR9114 的静态 Kd 把这些全压成一个数。学这个数,你永远学不到"群体怎么被推着演化"。

指控 3(可辨识性反成摆设):如果你真不做动力学反推,那 I2 可辨识性(从轨迹反推 drift 的唯一性)整个就是空的——你根本没有"反推"动作,何谈"可辨识"?那 Stage 0 辛苦核实的三条 SDE 定理、玩具 SDE(P07)全都白费。要么承认你做的不是 SDE 反问题(违 North Star),要么保留动力学步(那 Spike0 不充分)。

---

**【DEFENDER 应战】**(弹药:Stage 0 I2 定理 + gradient-flow 结构 + r1 附录 r1-I)

回应指控 1(静态 vs 动力学的统一条件):North Star 已确认问题是 **gradient-flow SDE**——drift 是某势函数的梯度 ∇f。这是关键结构。在 gradient-flow 假设下,**势函数 f 本身就是适应度地形,∇f 是它的梯度**。Stage 0 定理 T1(Guan 2025)/T2(Lavenant)都建立在 drift=−∇Ψ 的梯度结构上。所以如果 CR9114 的 −logK_D 地形就是(或单调对应)这个势 f,那 Spike0 学到 f 后,∇f 直接由微分得到——静态与动力学在梯度流下统一。我不是偷换,是利用问题自带的梯度流结构。

回应指控 2(频率/路径/时变依赖):承认这三者是静态地形抓不到的。但要分清——它们属于"选择算子 f 的自变量里有没有群体频率/时间"。North Star 定义 f(seq, cond),cond 是外部条件(抗原/浓度),**不含群体内频率**。如果 f 真依赖群体频率(frequency-dependent),那是另一个更复杂的问题(f(seq,cond,频率)),超出 North Star。在 North Star 的 f(seq,cond) 设定下,选择压只通过外部 cond 进入,这正是 CR9114 三抗原能覆盖的(换抗原=换cond)。所以在问题定义的边界内,Spike0 不丢失 f(seq,cond) 的信息。frequency-dependent 留作未来扩展,不属当前 claim。

回应指控 3(可辨识性的真正角色):这正是 r1 附录 r1-I 厘清的。可辨识性(I2)问的是"在已知 σ + population 边际观测下,从分布演化反推 ∇f 是否 well-posed(唯一)"。这是一个**数学适定性问题**,独立于"我用静态拟合还是动力学反推来实操"。两种实操:(A) 若 gradient-flow 成立,静态拟合 f 取梯度 = 动力学 ∇f(P07 玩具 SDE 验证这个等价);(B) 若不成立,需独立动力学反推。**哪条路对,取决于 gating① 实测,不能靠嘴辩。** 玩具 SDE 不是摆设,它正是裁决 A/B 的工具(验证"静态取梯度"与"轨迹反推"是否一致)。

---

**【关键实测:gating① + T-core 决战(P07 玩具 SDE 承载)】**

judge 要求:这一仗不能纯逻辑裁,必须看玩具 SDE 实测(r1 附录 r1-I 方案)。在本对抗中做思想实验级推演(真实数值留 Stage 4 执行):

设玩具:2 维连续空间,已知势 f_toy(x,c) = 二次型 + 条件依赖项,已知 σ_toy。前向跑 gradient-flow SDE 生成 population 边际快照。

- 检验 A(可辨识性,I2):从快照反推 f̂_toy,看是否恢复 f_toy。**理论预期:在 σ 已知 + 梯度流 + 非平衡瞬态下,T2(Lavenant)保证唯一恢复——所以反推 well-posed。** 但前提 P2.2 非退化(gating②)需 σ_toy 满秩(玩具里可设满秩,真实 PLM 空间待 Stage 4 测)。
- 检验 B(gating① T-core 决战):比较两条路径——(路径1)静态拟合 x→f_toy 值,取梯度得 ∇f̂_static;(路径2)从轨迹反推 drift 得 ∇f̂_dyn。**理论预期:若数据生成确为 gradient-flow,两路径在群体探索过的区域一致;但在群体未充分探索的区域,路径1(静态)依赖回归外推,路径2(动力学)依赖轨迹覆盖——两者可能偏离。**

**核心洞察(本轮最重要)**:静态拟合与动力学反推的等价性**不是全局的,而是区域性的**——在数据(变体/轨迹)充分覆盖的序列区域,两者一致(分支A局部成立);在覆盖稀疏区,静态回归外推与动力学反推会分歧(分支B局部成立)。CR9114 是组合完备的(2^16 全测),所以在 CR9114 覆盖的 16 突变组合空间内,**静态拟合 f 取梯度 ≈ 动力学 ∇f(分支A 在此空间成立)**;但 CR9114 只覆盖 16 个位点的组合,序列空间的其余维度(其他位点、其他抗体)未覆盖,那里需要动力学信息或更多数据(分支B)。

**【JUDGE 裁决】**

R2 + T-core 裁决:**分支 A 与 B 不是二选一,而是按"数据覆盖"分区共存。**

- 在 CR9114 组合完备覆盖的空间内:Spike0 静态拟合 f 取梯度 ≈ 动力学 ∇f(gradient-flow + 完备覆盖保证)。**Spike0 在此充分**,可验证"方法能学到选择算子(势函数 f)且其梯度=驱动力"。这回应指控 1:不是偷换,CR9114 完备性 + 梯度流结构使静态=动力学在此空间成立。
- 超出 CR9114 覆盖的空间(其他位点/抗体/真实免疫时序):需要动力学反推 + 更多数据。这是公司数据(轴 T 时序、跨 campaign)的不可替代价值,留 R5/入职后。
- 指控 2(frequency-dependent 等)裁决:出 North Star 的 f(seq,cond) 边界,不属当前 claim,留未来扩展。defender 守住了边界。

**P07 玩具 SDE 去留裁决**:**复活,但形态明确**。P07 承重两件事——(1) 验证可辨识性 well-posed(I2 数学适定性);(2) 实测 gating① 即"静态取梯度 vs 轨迹反推"的一致性(裁决分支A的局部成立范围)。没有 P07,I2 完全悬空、T-core 无法裁决。factor-removal:砍掉 P07 → I2 无任何验证 → I2 塌。**P07 是 load-bearing,复活。** 形态 = 玩具级最小可辨识性 + gating① 检验工具(非定稿方案的运行组件,是方法成立性的一次性数学验证)。

## 2b — 张力提炼(deep-insight → problem-reformulation: tension-mining)

## 2d — 抢救被砍部件(stress-test → counterfactual-probing: necessity-sufficiency + factor-removal)

### 附录 r2-A:gradient-flow 势函数 与 适应度地形 的等价性深析(Stage 4 理论依据)

r2 的核心裁决"静态=动力学在 CR9114 完备区成立"依赖一个理论主张:gradient-flow SDE 的势函数 f 等于(或单调对应)CR9114 测的平衡亲和力地形。这里把这个主张展开,供 Stage 4 操作化时心里有数,也供 Stage 3 终检复核其严格性。

**主张的精确形式**:在 gradient-flow SDE `dX=∇f(X,c)·dt + σdW` 中,平稳分布(若存在)是 Gibbs 形式 π(x) ∝ exp(2f(x,c)/σ²)(σ 标量时)。即势函数 f 决定了群体在序列空间的平稳停留密度——f 高的地方群体密度高。而 CR9114 的 −logK_D 是"该变体的结合强度",在亲和力成熟的选择逻辑下,结合强(−logK_D 大)的变体被正选择、在群体中富集。所以 −logK_D 与 f 都刻画"哪些序列被选择青睐",两者应单调相关。

**但有三个必须 Stage 4 检验的缝隙(Stage 3 复核点)**:
1. **平衡 vs 瞬态的矛盾**:Gibbs 平稳分布是 t→∞ 的平衡结果,但 Stage 0 定理 T1 要求观测在**非平衡瞬态**才可辨识。这看似矛盾,实则不:CR9114 是直接测每个变体的 Kd(不是测群体分布),它给的是 f 的"逐点值"真值,不依赖群体到没到平衡。所以 CR9114 拟合 f 不受瞬态要求约束;瞬态要求只约束"从群体轨迹反推 f"的 P07 玩具 SDE 部分。这个区分很重要:**Spike0 学 f 用的是逐点 Kd 真值(无平衡/瞬态问题),P07 验可辨识性用的是群体演化(有瞬态要求)。** 两者数据形态不同,约束不同。
2. **−logK_D 与 f 的单调对应是否线性**:f 是选择势,−logK_D 是结合自由能(∝ logK_D)。两者单调相关,但可能非线性(选择强度未必正比于结合自由能;还有表达量、稳定性等混杂)。Stage 4 拟合时,Spike0 学的是"−logK_D 地形",声称它是 f 的代理需要这个单调性假设。若非线性严重,f 的梯度方向仍对(单调保序),但梯度大小会偏。对"学到选择算子方向"(I1 的排序验证)够用,对"学到选择算子精确数值"不够。CV 模块的 gate 用的是排序(Spearman)而非数值,所以稳健。
3. **σ 各向异性对 Gibbs 形式的修正**:σ 非标量时,平稳分布不是简单 exp(2f/σ²),而是依赖 a(x)=σσᵀ 的更复杂形式。这关联 gating②。Stage 4 需在这个修正下重新建立 f 与 −logK_D 的对应。

**对 Stage 4 的交代**:Spike0 学 −logK_D 地形作为 f 的代理,验证"梯度方向=选择驱动方向",用排序判据(对单调非线性稳健)。精确数值对应留作 limitation,在 CS 声明里标注。这三个缝隙都不推翻 r2 的"分区共存"裁决(它们影响的是 f 的数值精度,不是"静态能否代理动力学"的定性结论),但 Stage 3 终检要确认它们被诚实标注。

**预期 r3 末状态**:I1🟢/I2🟢(有条件)/I3🟢,三不变量全绿(I2 带 Stage4 条件)。复活清单:预期空(R4 不引入新部件)。砍:P09/P10 可能降级。saturation 第1条"连续2轮无复活"——r2 有复活,r3 若无复活,则 r3 是"第1个无复活轮",还需 r4 确认"连续2轮"。所以**最快 r4 末达 saturation**(除非放宽:若 r3 后明显砍不动且全绿,可论证 r2 的复活是"最后一次实质变动",r3/r4 连续无复活)。

**潜在提前收敛论证**:spec saturation 第1条是"连续2轮抢救不出新承重部件"。r2 复活 P07 是"抢救出新承重部件";若 r3 抢救不出新的(R4 不引入新部件,P09/P10 是砍不是复活),r3 是第1个"无新复活"轮。严格需 r4 再确认1轮。但若 r3 末三不变量全绿 + 砍不动(判据2、3 满足),且 r3 无复活,可在 r4 做一个"确认轮"(快速验证无新复活)即达 saturation。预期总轮数 3-4 轮,符合 spec 预算(硬下限3,软上限8)。

### 附录 r2-D:v2 组件级完整规格(Stage 3 终检 / Stage 4 操作化直接取用)

把 v2 每个存活组件写成完整规格,作为接近定稿的方案快照。

**组件 1 — Spike0(主干:学选择算子的势函数)**
- 输入:CR9114 Source data 1(65536 变体 × 3 抗原 H1/H3/B 的 −logK_D)。
- 模型:序列表征(PLM embedding 或 16维突变向量 baseline)+ 条件(抗原 one-hot/embedding)→ 标量 f̂(seq,cond)。架构遵循偏好(Transformer/Mamba,graph 作 attention bias)。
- loss:censored 回归(Tobit)处理检测限 floor——未截断观测用 MSE,左截断(低于检测限)只惩罚 f̂>floor。
- 理论身份:gradient-flow 下 f̂ 是选择势函数,∇f̂(微分)= 动力学驱动力。在 CR9114 完备区静态=动力学(r2 裁决)。
- 产出:f̂(seq,cond) + 其梯度场 ∇f̂。

**组件 2 — P07 玩具 SDE 可辨识性验证(一次性数学适定性,=Spike1)**
- 目的:验证 I2——在已知 σ + population 边际下,从演化反推 ∇f 是否 well-posed(唯一)。
- 步骤:构造低维玩具(已知 f_toy + 已知 σ_toy)→ 前向跑 gradient-flow SDE 生成 population 边际快照 → 反推 f̂_toy → 检验恢复精度。
- 含 gating① 检验:比较"静态拟合 f_toy 取梯度"vs"轨迹反推 drift"是否一致(裁决分区范围)。
- 含 P08 边界扫描:扫噪声/数据量/时间点数,看可辨识性何时失效。
- 含 P14 内部步骤:轨迹反推时用已知 σ 扣除突变贡献。
- 不循环厘清:claim 是"反推数学适定",非"学对真实生物学"。服务 I2 非 I1。
- 产出:可辨识性 well-posed 判定 + gating① 一致性结论 + 有效边界图。

**组件 3 — CV 条件轴验证模块(I1+I3)**
- CV-a 正向条件依赖:固定变体扫抗原,f̂ 跨抗原变化 vs 实测 nested 一致。
- CV-b 打乱标签对照(I3 核心):打乱抗原标签重训,f̂_shuffled 性能须显著掉。
- CV-c cross-condition 非循环 gate(I1 核心):H1+H3 学→预测留出 B 的 Kd 排序,且显著打败 N3 baseline(序列相似度 + Phillips pairwise epistasis)。
- CV-d 条件依赖真值复现:同变体跨抗原 f̂ 变化方向 vs 实测 Kd 方向一致。
- 前提:P16 零泄漏(留出抗原/突变组合未进训练)。
- 产出:I1 判定(CV-c 过=非循环+鉴别力)+ I3 判定(CV-b 过=条件依赖真)。

**组件 4 — CS 覆盖范围诚实声明(I1 诚实性)**
- 严格验证区:CR9114 组合完备空间(16 位点 2^16,三抗原)。此区内"学到选择算子"是已验证结论。
- claim 外推区:一般抗体、其他位点、真实免疫时序——待验假设,需公司数据(轴 T 时序、跨 campaign 泛化)。
- 数值精度 limitation:f̂ 学的是 −logK_D 地形作为选择势 f 的代理,梯度方向可靠(排序判据),精确数值受单调非线性 + σ 各向异性影响(附录 r2-A 三缝隙)。

**待 r3 处理**:P09 Hanke(预判降可选)、P10 SVD(预判砍/降)。

这四个组件 + 待定项构成 v2 的完整快照。r3 主要确认 I3(CV-b 打乱对照)+ 收尾 P09/P10,预期 v2→v3 变动很小(接近定稿)。

### 附录 r2-E:r2 方法论自检

1. 真对抗?→ 是。critic 的"静态≠动力学、偷换问题"是实质科学质疑,defender 用 gradient-flow 结构 + 数据覆盖论证反击,judge 给出"分区共存"这个非平凡裁决(不是简单选一边)。
2. 玩具级激进守住?→ 是。没有因为"要严格"就把逼真仿真器请回来;P07 复活的是玩具级最小验证,不是 r1 砍掉的逼真流水线。
3. resurrection 认真?→ 是。P07 做了正反两向 necessity-sufficiency(既论证它承重,又反向质疑是否过度),确认 I2 唯一支柱。

### 附录 r2-F:三不变量跨轮演化追踪(r0→r2,供 Stage 3 终检)

把三不变量从 Stage 1 起点到 r2 末的演化轨迹记录,Stage 3 据此判断 loop 是否真在"硬化验证"而非"为简化而放水"。

**I1 验证非循环性**:
- r0(Stage1起点):🟢 大幅夯实(Stage 0 给了 8 先例 + 理论收束,路径存在,未在方案上判绿)。
- r1:🟢 绿(有条件)。在 v1 上判绿,且**升级**——从"非循环"升到"非循环+机制鉴别力",cross-condition(O₂)+ N3 baseline 升为必要 gate。这是硬化(判据变严)。
- r2:🟢 维持。本轮未动 I1 路径,但 P07/CV 的分工厘清强化了 I1 的边界(P07 不混入 I1)。
- 演化结论:I1 一路硬化,从"找到路径"到"立判据"到"判据精确化",无放水。

**I2 可辨识性**:
- r0:🟡 AMBER(Stage 0 判,gating①②无定理支撑)。
- r1:🟡 维持(r1 主攻 R1,未深入 I2)。
- r2:🟡→🟢(有条件)。本轮实质推进——T2 定理(σ已知)+ gradient-flow + CR9114完备覆盖 + P07玩具SDE验证,确立"反推 well-posed";但保留两条 Stage 4 必验条件(gating② λmin 实测 + 覆盖区限制)。

**I3 条件依赖**:
- r0:⚪ 未验(真值场 CR9114 三抗原就绪)。
- r1:🟡 微升(B1 cross-condition 防线间接调用了 I3 机制)。
- r2:🟡 维持(CV 模块设计就位,打乱对照思想实验级,未正式跑)。
- 演化结论:I3 稳步推进,待 r3 正式跑 CV-b 打乱对照转🟢。

**整体判断(供 Stage 3)**:三不变量在 loop 中全部朝"更严/更实"演化,无一为了简化而被放松。I1 判据变严、I2 升级但诚实保留条件、I3 稳步确立。这证明 loop 的"简化"是"砍冗余工程"而非"放松验证标准"——v0→v2 工程量大降,但三不变量的验证强度同步上升。这正是 spec"简化与硬化同一动作两面"的实证。Stage 3 终检应确认:被砍的 8+2 件(P01/02/04/05/06原/20外壳 + P14/P08独立性)无一是为了"让验证更容易通过"而砍,全是对三不变量零贡献或负贡献(循环)的冗余。
