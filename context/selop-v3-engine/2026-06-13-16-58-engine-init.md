## Plan Context

引擎是三件交付物里的第一件,因为它产出"涌现真值数据",另外两件(恢复模型 / 基线)都吃它的输出。本子项目目标:造一个 agent-based 个体级模拟引擎,满足对抗轮三条非循环铁律——① 引擎必须个体级,绝不直接实现测度流 PDE/生成元当真值源;② φ/σ/a_invivo 走涌现+记录(写个体级机制规则,真值从跑的过程涌现并记录),不手写真值函数;③ do() 必须是个体级真干预。

**真实数据基底约束(用户硬要求)**: 不得拍脑袋造异常简单数据;必须在真实开源 dataset 上适度合成。引擎承重的真实数据角色: ① a_DMS 亲和地形 = CR9114(2¹⁶)/CR6261 真实 DMS(本地已有 parquet);② 突变算子 M = S5F/SHM 已发表模型;③ 种群尺度 N/μ/轮次校准真实 repertoire。合成严格只限待恢复的 φ/σ/a_invivo 三算子,且按铁律②机制化涌现而非手写。保真度校准: 模拟统计量须对真实数据,对不上不许用。

**输出(供恢复模型/基线消费)**: 每轮 dump 幸存者序列时序(恢复方法只能看见这个)+ 旁路记录的涌现真值 φ/σ/a_invivo(只用于对账,恢复方法看不到)+ do() 介入数据。

## Checkpoint: 引擎定位收口 (round-1 brainstorm) — world model 底座 + 三物理算子

> Appended: 2026-06-13 21:04
> 阶段: selop-v3-engine 子项目 brainstorming 第一段
> 触发: 与用户连续 6 轮对话把"引擎到底是什么"从模糊澄清到锁定六条决议

### Objective

这一段 brainstorming 的目标是把三件交付物里的第一件——个体级模拟引擎——的顶层定位彻底锁死。不是定实现细节(数据结构、tick 顺序、损失函数那些后面再说),而是先回答一个更前置、更要命的问题:这个引擎到底是个什么性质的东西,它凭什么不是"自娱自乐的回声室"。这个定位问题如果不收口,后面所有工程决策都站在流沙上。本段的产出是六条决议,它们共同把引擎从"一个模拟器"重新定义成"一个 world-model 驱动的逼真环境 + 三个物理算子真值源",并且明确了 A 层(逼真度)与 B 层(真值)的职责隔离这条命根子。

### 为什么这一段如此重要 (背景给零上下文读者)

整个 Round 3 的验证逻辑站在一个硬事实上: Weinstein 2022 证明了适应度 φ 无法从纯观测数据反推 (φ 是因果量, 从 observational data 不可辨识)。这条定理的直接后果是: 任何"从幸存者序列恢复 φ/σ/a_invivo"的方法, 都无法用真实数据验证其正确性——因为真实世界里没人知道真的 φ 是什么, 它从来没被测过、原理上也测不出。于是唯一诚实的验证路径是: 造一个我们自己掌握并记录了真 φ/σ/a_invivo 的合成世界, 让恢复方法只能看到真实实验能看到的东西 (幸存者序列时序), 然后拿它的恢复结果去和我们旁路记下的真值对账。引擎就是这个世界。它是另外两件交付物 (恢复模型、基线) 唯一的数据来源。正因为它是真值的唯一来源, 它的设计纯度直接决定整个 Round 3 是真科学还是自欺。

### Process Summary — 对话怎么一步步逼出这六条决议

这一段不是我单方面设计后交付, 而是用户用六个连续的、一次比一次更尖锐的追问把定位逼出来的。完整记录这个推演链条对未来恢复极其重要, 因为每一条决议都是为了堵住某一个具体的"自欺漏洞"才立的, 脱离了它要堵的漏洞, 决议本身会显得武断。

第一步 分辨率选择。我给出三档生物分辨率: 甲=GC (生发中心) 忠实的 agent-based 模型 (显式模拟暗区增殖+突变 / 光区竞争抢抗原+抢 T-help / 循环或出营); 乙=序列空间上的群体遗传过程 (Wright-Fisher/Moran 在真实 DMS 序列格上跑生灭); 丙=混合 (借 GC 机制的形但跑在 DMS 序列格上不显式建细胞空间)。我推荐甲并给出理由: 只有 GC 现场让 σ/φ/a_invivo 是三个物理上真正不同的机制 (a_invivo=B 细胞从 FDC 上用力拽抗原的 Bell-Evans catch-bond 力学; σ=哪些 B 细胞拿到足够 T-help 能循环回暗区或出营的存活采样瓶颈; φ=竞争下的克隆扩增即频率依赖+克隆干涉), 退到乙会把三层悄悄揉回"采样规则+亲和力变换", 等于在引擎层面就把待验问题抹平 (一种隐蔽循环)。用户选甲。

第二步 解释 agent-based 是什么。用户坦诚不懂"为什么叫 agent-based", 并要求我后续讲解更简洁、先给大纲、让他看得懂。我用"自上而下 (直接写群体公式, 那个系数就是手写答案, 绝对不能用) vs 自下而上 (只写单个个体规则, 群体规律自己长出来)"的对照解释清楚: agent = 一个 B 细胞, 身上带序列/所在区/抓到的抗原量/T-help 信号; 三个算子是这群细胞三种不同行为的统计涌现。命根子是: 我们从头到尾没写过任何一行"φ(序列)=多少", 写的全是局部机制规则, 真值是跑出来再被旁路记账的, 不是定义出来的——这就是它和"把答案塞进去再取出来"的废物 MVP 的根本区别。

第三步 用户质疑"是不是自娱自乐"。这是关键转折。用户问: "本质上还是要造一个完全沙盒模拟器自娱自乐先验证一批结果是吧"。我没有否认是沙盒, 而是把"是不是沙盒"和"是不是回声室"分开: 回声室=我自己定规则又验证我能恢复自己定的规则。区分它和回声室的有三道关——(1) 地形是真的不是编的 (a_DMS 用真实 DMS 实测, 我无权编哪个序列亲和力高); (2) 沙盒必须先通过保真度闸门体检 (克隆大小分布/突变谱/多样性衰减对得上真实 repertoire) 不通过不许用; (3) 沙盒不是终点是筛子——产出不是"我们证明了 φ 能恢复"的自我表扬, 而是核心预测 P1 在一个长得像真 GC 的世界里到底成不成立这个真信号, P1 成立→有了可证伪预言可投 wet-lab (D 段), P1 垮了→诚实省下做注定失败真实实验的钱和时间。结论: 不是自娱自乐, 是"在花真钱做湿实验前, 先在最接近真实的廉价环境里把理论逼到该垮就垮"。但丑话说前面: 这一切只在三道关守住的前提下成立, 闸门松了或机制偷塞答案就立刻退化成自欺。

第四步 用户提出 world model 设想。用户说"那还是很恶心啊这种东西, 为什么不想办法做出一个 world model 一样的环境呢? 用开源数据先尽可能打造出这么一个 environment"。这一步价值极高, 因为它逼出了 A/B 分层。我先精确定位"恶心"的来源: 不是模拟本身, 而是"机制规则我手写"那部分 (那种'抓抗原概率取决于亲和力和浓度'的规则是我拍的, 用户本能觉得是私货)。然后把引擎拆两层: A 层=地形 (序列形态/亲和力/突变/克隆大小分布/repertoire 多样性演化, 全是真实可观测统计, world model 能学); B 层=答案 (φ/σ/a_invivo 三算子真值)。关键事实: B 层在真实数据里根本不存在 (Weinstein 墙), world model 无论多强都学不到——它会非常逼真但作为验证台是空的。所以正解是 world model 吃掉 A 层 (我一行私货都不写, 数据说了算, 且应尽量做满), B 层只保留极少量可证伪的机制假设 (a_invivo 走 Bell-Evans catch-bond 有单分子力谱实验支撑; σ 是 T-help 瓶颈采样是 GC 既定生物学), 我能动的私货被压缩到只剩"几个已知机制怎么组合"且每条标注来源可质疑。这点机制不是瑕疵是必要的——没有它 B 层就没真值, 是 Weinstein 墙逼出来的不是偷懒。

第五步 用户用一句话收口定位。用户说"也就是说我们的底座就是一个 world model 上搭载三个物理定律算子是吧"。我确认这句话对, 但补了一个会决定成败的精确化: 三个算子不是平起平坐"搭载"在 world model 上面, 它们是 world model 的驱动律——是让世界往前演化的那只手 (导演)。打的比方: world model 是舞台 (知道演员长什么样、能站哪、怎么移到相邻位置, 全是真实数据学来的), 但舞台本身不知道为什么这个演员火那个被淘汰; 三个算子就是导演, 决定谁繁殖谁出局谁能从抗原上拽下资源。为什么要命: 如果算子只是搭载在上面的旁观标签, world model 自己就能跑完演化, 算子可有可无→那它们就是事后贴的标签→回声室; 只有当算子是真正驱动世界演化的因果之手时, 幸存者序列时序才是这三个算子作用的真实后果, 恢复模型才有东西可逆向。真值必须是因果上游, 不能是事后注释。用户认这个"舞台=底座, 算子=驱动律"的定位。

第六步 用户提出"厚度=非常厚 + 高维还原"的设想, 并问这算不算 world model 技术。用户原话: "厚, 非常厚, 能用的开源数据我们尽可能全用上。假如说这个问题本身是一个多维空间的话, 这批开源数据可能每个只覆盖了部分维度, 比如只有两到三个维度, 你需要设计一种数据驱动的机制, 从每个 dataset 的 2-3 维度当中还原出原本问题的高维空间出来"。这是本段技术含量最高的一问, 我用三层结构回答 (见下面 Key Findings 的多视图整合部分)。

### Key Findings

#### 发现 1: 引擎本质 = A 层逼真环境 + B 层物理算子, 两层职责严格隔离

最终锁定的引擎不是"一个模拟器", 而是分两层、职责互不越界的结构。A 层 (舞台/底座) = 数据驱动的逼真生发中心环境, 提供真实的序列空间、亲和力地形、突变谱、邻接结构、克隆动力学、repertoire 时序行为, 这一层全部由开源真实数据撑起, 不写任何私货, 目标是尽可能逼真。B 层 (导演/真值) = 三个有物理依据、可证伪、被旁路记录的真值算子 φ/σ/a_invivo, 它们是驱动 A 层舞台往前演化的因果律, 用极少量已知物理 (catch-bond / T-help 瓶颈 / 频率依赖竞争) 显式埋入并全程记录。两层隔离的意义: A 层负责"像不像真的", B 层负责"有没有真值可对账", 任一层的私货不许渗到另一层。

#### 发现 2: 为什么 world model 给得了逼真却给不了真值 (A/B 必须分层的根本原因)

这是整段最重要的认识论结论。B 层 (φ 真值) 在真实数据里根本不存在——不是数据量不够或模型不够强, 而是 Weinstein 2022 证明的硬墙: φ 是因果量, 从纯观测数据原理上不可辨识。真实 repertoire 数据里只有"谁活下来了", 没有"谁的真适应度是多少", 后者从来没被测过、原理上也测不出。所以若训一个 world model 去拟合真实数据再拿它当真值源, 会得到一个能完美复现真实统计、但对 φ 这个问题保持完全沉默的环境: 它非常逼真, 但作为验证台是空的——它不知道真 φ, 你也就没法验证恢复模型恢复得对不对。一句话: world model 给得了逼真给不了真值, 而验证恰恰需要真值。这就是为什么不能"做个 world model 完事", 必须在逼真环境之上额外埋入有物理依据的真值算子。

#### 发现 3: 用户的高维还原设想 = 多视图对角整合 (diagonal integration), 不是 world model 技术

用户的直觉非常准: 手上每个开源数据集只覆盖完整问题空间的 2-3 个维度, 需要一种数据驱动机制把它们拼成高维全貌。这件事有专门的名字, 但不在 world model 名下。world model (Dreamer 那类) 是"学一个环境的动力学拿来做规划", 和"把多个只覆盖部分维度的数据集拼成高维全貌"不是一回事。用户描述的准确名称是多视图数据整合 (multi-view data integration), 而且是其中最难的一种——对角整合 (diagonal integration): 当每个数据集测的不是同一批样本、只测了不同的几个维度时, 把它们对齐到一个共享隐空间里, 让每个数据集都成为这个高维隐空间的一个投影/切片。落到我们的问题: 完整空间维度大致是 {序列, 平衡亲和力, 力依赖 off-rate, 突变倾向, 克隆丰度, 时间, 暗区/光区, T-help 信号, 抗原浓度}; 而手上数据 DMS (CR9114/CR6261) 只覆盖 序列×平衡亲和力, S5F 只覆盖 序列×突变倾向, repertoire 时序只覆盖 序列×丰度×时间, 力谱数据只覆盖 亲和力×力依赖off-rate, 每个只摸到 2-3 维。要的就是找一个共享隐空间让这四个都是它的投影然后还原全貌。领域里 MOFA / GLUE / totalVI 一族就是干这个的, 但直接套它们会撞原创红线 (memory feedback_originality_requirement), 必须记着。

#### 发现 4: 多边际耦合的数学硬真相——从低维边际拼高维联合原理上不唯一

这里有一堵必须先讲明的墙: 从低维边际拼高维联合, 原理上不唯一。最小例子: 已知 P(身高,体重) 和 P(体重,血压), 不能唯一还原 P(身高,体重,血压)——身高和血压之间的关系两个数据集都没测到, 有无穷多种填法同时满足那两个边际。数学上叫多边际耦合问题, 它是欠定的。所以"把 2-3 维数据集拼成高维全貌"没有唯一答案, 必须额外塞一个结构假设 (比如假设一个低维共享隐因子、或某些维度条件独立) 才能定下来, 而塞进去的这个假设数据本身验证不了。这个真相不是用来否决用户的想法, 而是用来给它划安全边界 (见发现 6)。

#### 发现 5: 危险共振——拼接本身就是一个带零空间的反问题, 和我们研究对象同形

最要命的一点: 这个"拼接"本身就是一个带零空间的反问题, 而我们整个 Round 3 研究的对象恰恰就是"带零空间的反问题" (Φ 的零谱、不可辨核 R_irr)。这是同一个数学形状。危险在于: 如果用一个不唯一的拼接结果去搭那个逼真环境, 再拿这个环境当真值源去验证恢复模型, 那等于把一个不可验证的假设偷偷塞进了验证台本身——环境可能在我看不见的那些维度上是错的, 而且原理上没法发现它错。这就从"逼真"滑回最讨厌的那种自欺, 只是藏得更深。这条发现是整段的"命门级"警告, 它直接催生了发现 6 的两条破解原则。

#### 发现 6: 破解办法——拼接关进 A 层 + 拼接不唯一性反当压力测试

发现 5 的危险不是要放弃用户的想法 (想法是对的), 破解办法恰好也是我们项目的核心精神, 两条: (原则一) 把拼接假设关进 A 层 (舞台), 绝不让它碰 B 层 (三个算子=答案)。拼接只用来让环境的地形和群体行为逼真; 三个真值算子仍然是用已知物理 (catch-bond / T-help 瓶颈) 显式埋入并记录, 不依赖拼接。这样就算拼接在隐维度上有偏, 真值依然干净。(原则二) 不要只信一种拼接, 要做一族。既然拼接不唯一, 那就生成多个都同样满足全部边际的合理全貌, 把核心结论 P1 在这一族上全跑一遍: P1 在所有合理拼接下都成立→它稳健, 不是拼接选法的人造产物→真信号; P1 只在某一种拼接下成立→它是 artifact, 当场识破。这一步把"拼接的不唯一性"从隐患变成了压力测试工具, 正好和项目里"部分辨识+适用包络"的精神同一套打法。

#### 发现 7: 用户的 embedding 设想 (待展开) — 用成熟分子模型先嵌到共享隐空间

用户在本段末尾提出一个尚未充分回答的技术设想 (留作下一段 brainstorm 的起点): 既然各数据集测的是不同生物分子、得到不同形态的数据, 能不能先用各种成熟模型把它们各自 embedding 一下, 嵌到同一片隐空间再处理? 用户进一步的猜想是: 如果这些模型都在类似的"生物分子空间"里训练出来, 或许它们最终会把不同的 input 嵌到类似的一片隐空间, 这也是对多边际耦合不唯一性的一个回应 (用预训练表征作为共享锚点, 缩小欠定的解空间)。这个设想的价值在于: 它把"凭空假设一个共享隐因子"换成了"用大规模预训练的分子表征当作经验性的共享坐标", 等于给欠定问题注入了一个数据驱动的、外部的结构先验, 而不是我拍脑袋的假设。这恰好缓解了发现 4/5 的核心顾虑。但它也带来新问题需要下一段回答: 不同预训练模型 (蛋白序列 PLM 如 ESM、结构模型、突变效应模型) 的隐空间并不天然对齐, "嵌到类似隐空间"是猜想不是保证; 而且预训练表征本身带着它训练数据的偏置, 这个偏置会不会渗进 A 层进而污染逼真度判断, 需要纳入保真度闸门体检。这条留作 open question, 是下一段的主问题。

### Decisions Made (六条决议, 已写入本文件决议记录段顶部摘要)

决议一: 引擎生物分辨率定为甲——GC 忠实的 agent-based 模型。理由: 只有生发中心现场让 σ/φ/a_invivo 是三个物理上真正独立的机制, 退到群体遗传 (乙) 会把三层揉回"采样+亲和力变换"从而在引擎层抹平待验问题, 构成隐蔽循环。代价是工程量大, 但这是让"三正交支柱"成为可证伪物理命题的唯一现场, 该付。

决议二: 引擎定位修正为两层结构——A 层 = world model 驱动的逼真环境 (数据说了算, 不写私货, 尽量做厚); B 层 = 三个有物理依据、可证伪、被旁路记录的真值算子 (极少量已知物理)。理由: Weinstein 墙决定 world model 学不到 φ 真值, 逼真和真值必须分别由数据和已知物理两套来源提供。

决议三: 三个算子是驱动世界演化的因果上游 (导演), 不是事后贴的标签。理由: 只有当算子是因果之手时, 幸存者序列时序才是其作用的真实后果, 恢复模型才有可逆向的对象; 否则退化为回声室。

决议四: A 层用开源数据搭到"非常厚", 能用的数据尽量全用上; 通过多视图对角整合 / 共享隐空间还原, 把各只覆盖 2-3 维的数据集拼成高维全貌。理由: 用户硬要求 + 越厚越逼真越难自欺; 克隆动力学那层尤其必须数据驱动, 因为 P1 (克隆干涉) 就出生在那层, 那层若是瞎编 P1 检验不可信。

决议五: 拼接原理上欠定不唯一, 所以只准用来撑 A 层逼真度并接受保真度闸门体检, 绝不碰 B 层真值。理由: 防止把不可验证的拼接假设偷偷塞进验证台本身 (发现 5 的命门警告)。

决议六: 拼接的不唯一性反过来当 P1 稳健性压力测试——生成一族都满足全部边际的合理全貌, P1 在全族都成立才算真信号, 只在某一种成立则判 artifact。理由: 把隐患转化为检验工具, 与"部分辨识+适用包络"精神一致。

### 深入技术记录 1: 三个算子在 GC 生物学里到底对应什么 (为什么甲分辨率让它们物理独立)

这一节展开决议一的生物学根据, 因为"σ/φ/a_invivo 在 GC 里是三个物理上真正不同的机制"这句话是决议一的全部分量所在, 必须把它落到具体的生发中心生物过程上, 否则它只是一句口号。生发中心 (germinal center, GC) 是淋巴结/脾脏里抗体亲和力成熟实际发生的微结构, 分暗区 (dark zone, DZ) 和光区 (light zone, LZ) 两个解剖区域, B 细胞在两区之间循环往复, 每一轮循环对应一次"突变-选择"。

a_invivo 对应的机制: 在光区, B 细胞表面的 B 细胞受体 (BCR, 即膜结合抗体) 要从滤泡树突细胞 (FDC) 表面把抗原"拽下来"内吞。这个"拽"是一个力学过程——BCR 与抗原的键在细胞施加的牵引力下断裂或维持, 决定 B 细胞实际获取多少抗原。这正是 Bell-Evans 理论描述的受力键 (有时表现为 catch-bond, 即受力反而更稳的反常键) 力学, 它依赖的是力依赖的解离率 (force-dependent off-rate), 而不是 DMS 测的零力平衡解离常数 K_D。这就是裂缝 B-S2 的物理本质: a_DMS (平衡 K_D, 体外 DMS 可测) ≠ a_invivo (力依赖提取效率, 体内真正起作用的量)。a_invivo 升为恢复目标正是因为它和 a_DMS 的这道物理鸿沟。

σ 对应的机制: 内吞抗原后, B 细胞把抗原肽提呈给滤泡辅助 T 细胞 (Tfh), 根据提呈量竞争性地获取 T 细胞帮助信号 (T-help)。只有拿到足够 T-help 的 B 细胞才能存活、被选中返回暗区继续增殖突变, 或分化出营成为浆细胞/记忆细胞; 拿不到的凋亡。这是一个多对一、不可逆的存活采样瓶颈——大量 B 细胞克隆竞争有限的 Tfh 资源, 只有少数通过。σ 就是这个"谁通过瓶颈"的算子, 它的不可逆 (死掉的不回来) 和多对一 (信息坍缩) 正是它"不可被动力学吸收"的根源, 也是为什么 v3 必须把它从被忽略的隐含算子升为显式正交支柱。

φ 对应的机制: 通过选择的 B 细胞返回暗区后增殖, 增殖代数/速率与它获取的信号强度相关。在 Nμ≫1 (大群体高突变率) 的区域, 多个有益突变克隆同时存在并相互竞争同一资源池, 产生克隆干涉 (clonal interference)——这会破坏"富集比单调反映适应度"的朴素假设, 正是 P1 预测非单调性的出生地。φ 是这个竞争扩增层的适应度, 它的频率依赖性 (一个克隆的命运取决于场上别的克隆) 使它无法还原成每条序列的静态标量。

三者物理独立的意义: 力学提取 (a_invivo)、信号瓶颈采样 (σ)、竞争扩增 (φ) 是三个发生在不同时空、由不同分子机制驱动的过程。只有 agent-based 显式模拟这套循环, 它们才各自从各自的机制里独立涌现; 任何聚合层建模都会把它们投影压缩回一两个有效量, 这正是退到乙/丙分辨率会犯的隐蔽循环。

### 深入技术记录 2: 真实数据底基清单 (A 层素材, 维度覆盖与角色)

这一节把 A 层"非常厚"的素材清单和各自覆盖的维度列清楚, 这是决议四 (多视图整合) 的输入盘点, 也是问题 A (还原机制设计) 的原料表。

CR9114 DMS: 抗流感广谱抗体 CR9114 的深度突变扫描, 本地已有 parquet (65536 行 = 2^16, 11 列), 覆盖 16 个位点的全组合, 对 H1/H3/B 三种血凝素的结合 (floor H1=7/H3=6/B=6 已记录在 memory project_cr9114_data)。覆盖维度 = 序列 × 平衡亲和力 (多抗原)。这是 a_DMS 地形的主力来源。

CR6261 DMS: 另一株抗流感广谱抗体, 本地 parquet (2048 行 = 2^11, 8 列)。覆盖维度同 CR9114 但位点数更少。作为第二套独立地形, 用于检验结论不依赖单一抗体 (适用包络的一个轴)。

S5F / SHM 谱: 已发表的体细胞超突变 (somatic hypermutation) 模型, 给出每个核苷酸 motif 的突变倾向 (mutability) 和替换偏好。覆盖维度 = 序列 (motif) × 突变倾向。这是突变算子 M 的来源, 让引擎里的突变不是均匀随机翻字母而是真实 SHM 偏置的。

repertoire 时序数据: 真实抗体 repertoire 的纵向测序数据 (待具体确认可得性, 见问题 H 关联)。覆盖维度 = 序列 × 克隆丰度 × 时间。这是校准种群尺度 N / 突变率 μ / 轮次, 以及保真度闸门里"克隆大小分布/多样性衰减"的来源, 也是 Regime 2 (真实数据验证) 的主力。

单分子力谱数据: 抗体-抗原键在外力下的解离行为数据 (待确认具体数据集)。覆盖维度 = 亲和力 × 力依赖 off-rate。这是 a_invivo (Bell-Evans catch-bond) 机制规则的物理参数来源——注意它进的是 B 层 (给机制规则定参数), 但参数本身来自真实力谱而非我拍的, 这是 B 层"极少量已知物理"里"已知"二字的兑现。

维度覆盖全景: 把上面五类拼起来, 完整问题空间 {序列, 平衡亲和力, 力依赖off-rate, 突变倾向, 克隆丰度, 时间, 暗区/光区, T-help信号, 抗原浓度} 里, 数据直接覆盖了 {序列, 平衡亲和力, 力依赖off-rate, 突变倾向, 克隆丰度, 时间}, 而 {暗区/光区, T-help信号, 抗原浓度} 这三个维度没有任何数据集直接测到——它们恰恰是 GC 内部状态, 只能靠 agent-based 机制涌现 + 多视图整合的隐空间外推。这张覆盖表清楚地显示了多视图整合要还原的"缺口维度"在哪, 也显示了为什么这些缺口维度天然属于 B 层机制管辖 (它们是过程内部量, 不是可观测统计), 进一步印证了 A/B 分层的自然性。

### 深入技术记录 3: 三条非循环铁律在本架构下如何被守住 (逐条核对)

对抗轮交棒时立了三条非循环铁律, 它们是引擎合法性的宪法。本节逐条核对新锁定的 A 层+B 层架构是否守住每一条, 因为定位修正 (引入 world model 当 A 层) 是一个重大改动, 必须确认它没有偷偷违反任何一条铁律。

铁律① (引擎必须个体级, 绝不直接实现测度流 PDE/生成元当真值源): 守住。决议一定甲 = agent-based 个体级, B 层真值从个体机制涌现。但定位修正引入了一个新的审查点: A 层的 world model 是否可能变相成为"测度流 PDE 当真值源"? 答案是不会, 只要守住决议五 (A 层只撑逼真度不碰 B 层真值)。A 层 world model 学的是可观测统计的分布 (序列形态、克隆大小分布), 它不生成 φ/σ/a_invivo 真值, 真值仍由 B 层个体机制涌现并旁路记录。换句话说 A 层是"舞台道具的生成器", 不是"剧情的生成器"; 剧情 (真值) 永远来自 B 层个体级机制。这条审查的结论是: 定位修正不违反铁律①, 但它把"A 层不得生成真值"提升为一条必须时刻盯防的红线 (这正是决议五的由来)。

铁律② (φ/σ/a_invivo 走涌现+记录, 不手写真值函数): 守住且强化。原本的风险是我手写闭式 φ(x); 新架构把私货压缩到只剩 B 层"几个已知物理机制怎么组合"且每条标注来源 (catch-bond 来自力谱、T-help 瓶颈来自 GC 生物学)。真值仍是跑出来再记账的。强化点在于: world model 接管 A 层后, 我连"地形长什么样"都不再手写 (以前可能用解析地形), 私货面积进一步缩小。但新增风险见铁律②的尾巴——预训练表征 (问题 A 的 embedding 方案) 的偏置算不算变相私货? 这条留给下一段, 暂定的防线是把表征偏置纳入保真度闸门体检。

铁律③ (do() 必须个体级真干预): 守住, 但实现待细化 (问题 E)。新架构不改变这条的执行方式——do(冻结环境) 仍是去钉死个体级机制 (抗原浓度/竞争场), 不在 A 层 world model 的分布参数上动手脚。需要警惕的新点: 不能图省事用"在 A 层 world model 的条件变量上做干预"来冒充个体级 do(), 那会是把因果干预降级成观测条件化, 是一种新型循环。这条审查给问题 E 加了一个约束: do() 必须穿透 A 层直接作用于 B 层个体机制。

三条核对的总结论: 定位修正 (world model 当 A 层) 与三条铁律相容, 但它新增了三个必须盯防的审查点, 全部归结到同一条原则——A 层与 B 层的职责隔离 (决议五) 必须是硬墙, 任何让 A 层渗入真值生成、或让 do()/真值绕过 B 层个体机制的设计都是新型循环。这条原则的重要性因定位修正而上升, 应在下一段及后续设计中反复回检。

### 深入技术记录 4: 用户 embedding 设想的完整技术展开 (下一段的主战场预热)

用户在本段末尾抛出的 embedding 设想 (发现 7) 是下一段 brainstorm 的核心, 这里把它的技术骨架、优势、风险、以及它如何回应多边际耦合不唯一性的逻辑链, 完整记录下来, 以便下一段直接接续而不必重建。

设想的核心: 不直接在原始低维边际上做对角整合 (那是 MOFA/GLUE 的做法, 撞原创红线且数学上欠定), 而是先用各领域成熟的预训练模型把每个数据集的样本各自 embedding 成向量, 把这些向量投到一个 (希望是) 共享的隐空间, 再在隐空间里做整合与高维还原。用户的关键猜想是: 如果这些预训练模型都是在"类似的生物分子空间"上训出来的, 它们可能天然把语义相近的输入映到隐空间里相近的位置, 于是不同数据集的 embedding 会落在可对齐的同一片区域, 从而提供一个经验性的共享坐标系。

为什么这缓解了不唯一性 (逻辑链): 发现 4 的硬真相是多边际耦合欠定, 必须额外注入一个结构假设, 而那个假设数据本身验证不了。用户的设想等于把"我拍脑袋的结构假设"换成"大规模预训练表征隐含的结构先验"——预训练模型在海量序列上学到的表征几何, 携带了关于"哪些分子相似、哪些功能相关"的经验知识, 用它当共享坐标, 等于用外部数据驱动的先验来约束欠定的解空间。这比凭空假设低维共享隐因子更可辩护, 因为先验来源是公开、可复现、可被第三方质疑的预训练模型, 而不是我私下的选择。这正是它比直接套 MOFA 更符合本项目精神 (原创性 + 可证伪 + 不藏私货) 的地方。

候选预训练表征 (下一段要逐个评估): (1) 蛋白序列语言模型 (PLM) 如 ESM-2 / ESM-C 系列, 把抗体序列嵌成残基级或序列级向量, 携带进化耦合与结构倾向信息; (2) 抗体专用模型如 AntiBERTy / IgLM, 在抗体 repertoire 上预训练, 可能比通用 PLM 更贴合 CDR 区语义; (3) 突变效应 / 适应度预测模型, 把"突变→效应"的先验带进来; (4) 结构表征 (如从 AF 类模型取的结构嵌入), 把序列锚到 3D 构象空间。每个覆盖问题空间的不同维度, 把它们的 embedding 对齐, 概念上就是在隐空间里做对角整合。

风险与必须回答的问题 (下一段议程): (风险一) 隐空间不天然对齐——不同模型的隐空间各有各的坐标与尺度, "嵌到类似隐空间"是猜想不是保证, 需要一个对齐机制 (能否用少量配对样本/锚点做跨模型对齐? 用 CCA/最优传输/对比学习? 但又不能直接套现成工具而违反原创红线, 需由问题结构逼出)。(风险二) 预训练偏置渗入 A 层——表征带着训练数据的偏置 (比如 PLM 偏向天然蛋白、对人工远缘突变体表征失真), 这个偏置若进了 A 层会扭曲逼真度, 必须把"表征忠实度"纳入保真度闸门体检 (例如检验 embedding 重构的统计量是否仍匹配真实 DMS)。(风险三) 还原机制的可生成性——对齐到共享隐空间只是第一步, 还要能从隐空间生成"缺口维度"的全貌 (暗区/光区、T-help、抗原浓度), 这一步怎么由问题结构逼出一个原创的生成机制, 是下一段最硬的骨头。(风险四) 表征是否反而把 B 层真值信息泄漏进 A 层——如果某个预训练模型其实隐含了适应度信息, 用它搭 A 层可能把答案泄漏进环境, 又是循环; 需要审查每个候选表征"知不知道 φ"。

下一段的判定目标: 在上述四个风险都有应对方案、且还原机制由问题结构原创逼出 (非套现成多组学整合工具) 的前提下, 确认 embedding-to-shared-latent 路线可作为 A 层"非常厚"环境的实现路径; 否则退而设计替代的高维还原机制。这条是把决议四从"方向"落到"可实现机制"的关键一跳。

### 深入技术记录 5: 引擎要服务的核心预测 P1/P2 的精确陈述 (引擎为什么这样设计的下游目的)

引擎的全部设计最终是为了让 P1/P2 这两个预测变得可检验。把它们的精确陈述记下来, 因为引擎的每个机制决策 (尤其 tick 顺序、克隆动力学层、do() 实现) 都应回溯到"它能不能让 P1/P2 被干净地测出来"。

P1 (克隆干涉下富集比与 φ 非单调): 在克隆干涉区 (Nμ≫1, 即大群体高突变率, 多个有益克隆同时竞争), 一条序列从一轮到下一轮的富集比 (enrichment ratio, 即丰度变化倍数) 与它的真适应度 φ 之间不是单调关系。朴素直觉认为"适应度越高富集越多", P1 断言在克隆干涉下这条会断裂——因为一个高 φ 克隆可能因为搭车 (hitchhiking) 或被更高 φ 的克隆压制而富集异常。这是骨架动力学原理的一个可证伪推论。引擎必须能产生真正的克隆干涉 (所以克隆动力学层必须数据驱动逼真, 决议四), 并旁路记录每条序列的真 φ, 才能把"富集比 vs φ"这条曲线画出来检验单调性。

P2 (do(冻结环境) 恢复单调性): 如果对引擎施加 do(冻结环境)——即钉死抗原浓度与竞争场, 消除频率依赖与克隆干涉——那么富集比与 φ 的单调关系应当恢复。P2 是 P1 的因果对照: 它断言 P1 的非单调性确实来自频率依赖/克隆干涉这个特定机制, 而非别的混杂。P2 要求 do() 是个体级真干预 (铁律③), 且干预必须穿透 A 层直达 B 层个体机制 (技术记录 3 的铁律③审查), 否则 P2 测的就不是因果而是观测条件化。

P1/P2 对引擎的设计倒逼: (1) 必须有真克隆干涉 → 克隆动力学逼真 (决议四, 数据驱动那一层); (2) 必须旁路记录每序列真 φ → B 层涌现真值记账粒度要到"每序列每轮" (问题 D); (3) 必须能干净冻结环境 → do() 个体级实现 (问题 E); (4) 富集比是直接可观测量 (恢复模型/基线都能算), 而 φ 是隐藏真值, 这个可观测 vs 隐藏的落差正是整个验证的支点。

### 深入技术记录 6: 5 个对抗性真值世界如何映射到 A 层+B 层架构 (问题 G 的预分析)

5 个对抗世界的设计目的是让理论在该失败的地方诚实失败。本节预分析每个世界改的是 A 层还是 B 层的什么, 为问题 G 的后续细化打底。

世界1 (σ 部分可被动力学吸收): 改 B 层 σ 机制, 让存活采样的一部分变异可以被 φ 动力学解释掉 (即 σ 不完全正交)。考验恢复方法会不会错误地把可吸收部分也当成独立 σ。A 层不变。

世界2 (外加噪声): 在 A 层观测层加入测量噪声/采样噪声 (模拟真实测序的有限读深), 不改 B 层真值。考验恢复方法在噪声下是否仍诚实报告辨识上限, 而不是把噪声拟合成虚假结构。

世界3 (a_invivo 走非 ΔG catch-bond): 改 B 层 a_invivo 机制, 让力依赖 off-rate 表现为真正的 catch-bond (受力更稳) 而非简单 ΔG 单调。考验恢复方法是否会错误假设 a_invivo 是 a_DMS 的单调变换 (裂缝 B-S2 的极端化)。

世界4 (三通道强共线): 调 B 层让 φ/σ/a_invivo 三个算子高度共线 (在数据里几乎同步变化)。考验恢复方法的部分辨识诚实性——共线时三者本就不可分, 方法应报告"不可辨"而非强行给出三个分离的估计。这是直接攻击"三正交支柱"主张的世界。

世界5 (真存在静态 φ): 这是最关键的对抗世界——构造一个真的存在每序列静态适应度 φ 的世界 (即骨架的核心反命题成立)。考验我们的方法和基线会不会在"静态 φ 真存在"时正确地检测到它、收敛到它。这个世界同时是判别第四路的试金石: 如果连真有静态 φ 时方法都恢复不出来, 那方法有 bug; 如果真没有静态 φ (世界1-4) 时方法却报告有, 那是 framework 诱导的假阳性。世界5 主要靠 B 层把 φ 机制换成静态标量, A 层逼真度照旧。

横向观察: 5 个世界里 4 个 (1/3/4/5) 改 B 层机制, 1 个 (世界2) 改 A 层观测噪声。这印证了 A/B 分层的工程便利——对抗世界的构造绝大多数落在 B 层算子机制的替换上, A 层逼真环境基本可复用, 大幅降低造 5 个世界的工程量。这是决议二 (A/B 分层) 的一个意外红利, 值得记下。

### 深入技术记录 7: 引擎到底在验证什么——诚实骨架的精确内容 (上游来源)

引擎不是凭空造的, 它服务于对抗轮降级后留下的那个"诚实骨架"。把骨架的精确结构记下来, 因为引擎的每一个机制都对应骨架的某一条主张, 脱离骨架就不知道引擎为什么要这样而不那样。骨架 = 1 EARNED 动力学原理 + 2 正交独立支柱 + 1 诚实残余 + 1 适用包络。

1 EARNED 动力学原理: 抗体选择不存在每条序列的静态适应度 φ; 存在序列空间测度 μ_t 的单一非平衡随机过程 (Doi-Peliti/MSR 场论作用量描述)。这条之所以 EARNED (挣来的, 非装饰), 是因为它锚定在两条框架无关的硬禁止上, 且这两条在对抗轮中是 SUPPORTED (有独立证据): (硬禁止一) Price 协方差同号约束; (硬禁止二) 噪声-有效种群规模 N_e 标度。引擎必须能让"测度流"从个体级涌现 (铁律①), 而 P1/P2 是这条原理的可证伪推论。

2 正交独立支柱: σ (存活算子, 力学/采样读出, 不可被动力学吸收) 和 辨识 (Φ 观测算子的零谱/部分辨识)。对抗轮的关键降级是: 原本宣称"单一骨架统一三层"被判 elegance-trap, 骨架只生成通道1动力学, σ 和辨识是正交独立支柱不是骨架的推论。引擎必须让 σ 作为独立机制涌现 (这正是甲分辨率/GC 现场的理由), 世界1 和世界4 专门攻击这条。

1 诚实残余 R_irr: 不可辨核 = Φ 零谱 = Noether 对称轨道 (N_e·s 标度对称 ⊕ s−κ 移位对称)。这是原理上无法从观测恢复的部分, 引擎必须能旁路记录真值从而量化"恢复方法到底碰到了多少 R_irr 这堵墙", 命题①的阈值 (三子空间主夹角 θ + dim R_irr/dim 全谱 ε) 就是量这个的。

1 适用包络: 简单连通 / 线性化 / 单势垒 / 平衡主导。骨架只在包络内成立, 包络外退化。引擎必须能造包络内和包络外两种世界分别测 (对抗轮交棒约束⑥), 检验方法是否诚实地在包络外报告失败。

骨架对引擎的总倒逼: 引擎不是要"证明骨架对", 而是要给骨架的每一条可证伪推论 (P1/P2/θ/ε/包络内外) 提供一个有真值可对账的测试现场, 并通过 5 个对抗世界主动寻找骨架失败的边界。这与对抗轮的求真精神一脉相承——赢条件是诚实的 BROKEN/CORROBORATED 判定, 不是 resilience 分。

### 深入技术记录 8: 引擎输出契约——恢复模型与基线到底吃什么 (下游接口)

引擎是另外两件交付物的唯一数据源, 它的输出格式就是整个 Round 3 的内部 API, 必须在引擎设计时就钉死, 否则恢复模型/基线无法对接。本节记录已锁定的输出契约方向 (具体 schema 待引擎细化阶段定)。

输出1 幸存者序列时序 (恢复方法唯一能看见的): 每一轮 (tick) dump 当前存活的 B 细胞群体, 以 (序列, 丰度) 的形式。这模拟真实实验唯一能观测到的东西——你只能测到活下来的细胞测序结果, 看不到死掉的、更看不到背后的真值算子。这是恢复模型和基线的共同输入。粒度待定 (问题 D 关联): 是否记录空间 (暗区/光区) 标签? 真实实验通常测不到空间分区, 所以默认幸存者序列时序不含空间标签 (与真实可观测性对齐), 但引擎内部记录空间以供旁路真值使用。

输出2 旁路记录的涌现真值 (只用于对账, 恢复方法看不到): 每轮每序列的真 φ/σ/a_invivo (具体是有效繁殖数/存活通过率/抗原获取量等涌现量的记账)。这是验证的金标准, 严格隔离, 恢复方法在训练和推理时都不可访问, 只在评估对账时由我们使用。这条的存在性正是合成引擎相对真实数据的唯一优势 (真实数据没有这个), 也是 Weinstein 墙逼出合成引擎的根本理由。

输出3 do() 介入数据: 个体级真干预 (如 do(冻结环境)) 产出的反事实序列时序。用于检验 P2 和因果辨识。每个 do() 配一个未干预的对照轨迹 (同初始条件同随机种子)。

下游消费方式: 恢复模型 (子项目2) 吃输出1, 吐对 φ/σ/a_invivo 的恢复, 与输出2 对账; 基线 (子项目3, CoSiNE/DASM 类) 同样只吃输出1, 作为原理性对比, 同时充当判别第四路 (对测度流答案盲、纯统计视角) 的载体。两者吃同一个输出1 保证对比公平。这个共享输入契约是三子项目依赖关系 (引擎→恢复/基线) 的物理体现。

### 深入技术记录 9: 保真度闸门的设计原则 (决议五的执行机构, 防自欺的体检站)

保真度闸门是把"适度合成"从口号变成可检验承诺的那道关 (决议五的执行机构)。本节展开它要体检什么、判据怎么定、以及它和 A/B 分层的关系。

闸门体检的对象只是 A 层逼真度, 不是 B 层真值。这点必须先厘清: 我们不可能拿真实数据去验证 B 层真值 (Weinstein 墙, 真实数据没有真 φ), 所以闸门体检的是"引擎跑出来的可观测统计像不像真实可观测统计"。如果连可观测统计都对不上真实数据, 说明 A 层环境是瞎编的, 那么即使 B 层真值是干净的, 整个实验的外推效力也归零——因为它发生在一个不像真生物的世界里。

候选体检统计量 (待逐个定度量与阈值, 问题 F): (1) 克隆大小分布——引擎产出的克隆丰度分布与真实 repertoire 的克隆大小分布比对, 度量候选 KS 距离或对数-对数尾部斜率; (2) 突变谱——引擎里实际发生的突变在 motif 上的分布与 S5F 预测谱比对, 度量候选 KL 散度; (3) 多样性衰减曲线——随轮次推进 repertoire 多样性 (如 Shannon/Simpson 指数) 的衰减轨迹与真实纵向数据比对; (4) 谱系树形状——克隆谱系树的拓扑统计量 (如 Sackin 指数) 与真实 B 细胞谱系树比对; (5) 亲和力成熟轨迹——平均亲和力随轮次提升的曲线与真实 GC 亲和力成熟数据比对。

判据原则: 闸门是"不通过不许用", 即硬门槛而非软评分。某个统计量偏离真实数据超过阈值, 该次引擎配置作废重调, 而不是降权继续用。这与对抗轮的求真精神一致——宁可承认环境造得不像、推倒重来, 也不在一个不像真的环境上跑出漂亮但无意义的结果。

闸门与 embedding 偏置的关联 (技术记录4 风险二的回收): 如果 A 层用了预训练表征 (embedding 方案), 那么"表征重构忠实度"也要进闸门——例如检验从共享隐空间重构回的序列/亲和力统计是否仍匹配真实 DMS, 以此捕捉预训练偏置对 A 层的污染。这把一个抽象的偏置风险变成了一条具体的体检项。

闸门的位置: 它在引擎产出数据、交给恢复模型/基线之前。是一道前置准入, 不是事后诊断。只有通过闸门的引擎输出才进入下游验证流程。这保证了"我们验证恢复方法"这件事永远发生在一个已被认证为逼真的环境里。

### 深入技术记录 10: 多视图高维还原的概念流程 (决议四的可视化, 待原创化)

把决议四的多视图整合流程用概念步骤记下来, 注意全部步骤都受原创红线约束——不是套现成多组学工具, 而是要由本问题结构逼出。这里记的是"要达成什么", 具体"怎么原创地达成"是问题 A 留给下一段的。

步骤1 各数据集编码: 每个开源数据集 (DMS / S5F / repertoire 时序 / 力谱) 各自被编码进一个表示。用户设想是用成熟预训练模型做 embedding (技术记录4)。目标是把异构的原始数据 (不同分子、不同测量) 变成同一种数学对象 (向量)。

步骤2 共享隐空间对齐: 把各数据集的表示对齐到一个共享隐空间, 使得每个数据集成为这个高维隐空间的一个投影/切片。难点是不同来源的隐空间不天然对齐 (技术记录4 风险一)。

步骤3 高维全貌生成: 从共享隐空间生成完整问题空间的样本, 包括没有任何数据集直接覆盖的缺口维度 ({暗区/光区, T-help, 抗原浓度}, 见技术记录2)。这一步是欠定的 (发现4), 所以产出的不是唯一全貌而是一族合理全貌。

步骤4 保真度体检: 每个生成的全貌过保真度闸门 (技术记录9), 不通过的丢弃。

步骤5 喂入 A 层并当压力测试: 通过体检的一族全貌都拿去搭 A 层环境, 核心结论 P1 在全族上跑, 全族成立=真信号, 仅某一种成立=artifact (决议六)。

这个流程的产出不是"一个还原好的高维数据集", 而是"一族通过体检的逼真环境 + 一个跨族的 P1 稳健性判定"。它把多视图整合的数学欠定性 (本来是缺陷) 直接编织进了验证逻辑 (变成稳健性检验), 这是本段在方法论上最自洽的一点。

### 深入技术记录 11: 架构硬约束对引擎的影响 (继承自 memory feedback_architecture_preference)

恢复模型那边有架构硬约束 (限 Transformer/Mamba/Diffusion, 图仅作 attention bias, 无独立 GNN/CNN/RNN)。这条约束主要约束子项目2 (恢复模型), 但它对引擎 (子项目1) 有间接影响, 记下来以免设计时遗漏。

影响1 A 层 world model 的实现架构: A 层若用预训练表征 + 生成模型来搭逼真环境, 那个生成模型本身的架构也应落在 Transformer/Mamba/Diffusion 范围内, 以保持全项目架构一致性。相2 已暴露的架构落点可直接复用: Diffusion (score = ∇log μ_t = FV drift) 天然适合生成序列空间测度的演化, 用它当 A 层克隆动力学的生成器在概念上与"测度流"骨架同源; Transformer-over-population (注意力 = 频率依赖 + 连锁) 适合建模群体内序列间的相互作用; Mamba 适合 μ_t 的时序。注意这里有一个微妙的红线: A 层用 Diffusion 生成测度流逼真度 ≠ 用 Diffusion 当 B 层真值源 (那会违反铁律①)。A 层的 Diffusion 只生成可观测统计的逼真样本, 不生成 φ/σ/a_invivo 真值。这个区分必须在实现时反复确认。

影响2 引擎本体 (agent-based 模拟器) 不受架构约束: 架构白名单约束的是"学习模型", agent-based 引擎本身是一个机制模拟器不是学习模型, 它的实现 (个体规则、tick 循环、事件调度) 不在白名单约束内, 用常规模拟代码即可。约束只在引擎里嵌入了学习组件 (A 层 world model / embedding) 时才触发。

影响3 全原创红线对引擎的约束 (memory feedback_originality_requirement): 禁"现成工具套新域"。这对引擎的多视图整合那部分 (问题 A) 是最强约束——不能直接拿 MOFA/GLUE/totalVI 套上, 还原机制必须由抗体选择这个问题本身的结构逼出。这条是问题 A 在下一段必须正面应对的红线, 也是为什么本段反复强调"由问题结构逼出"而不是"用某个现成方法"。

### 深入技术记录 12: 与 digital-evolution-sandbox 的区分 (防混淆, 重要)

项目里还有另一个沙盒——digital-evolution-sandbox (512×512 红皇后式对称混战进化沙盒), 它和本 selop-v3 验证沙盒是两个完全不同的东西, 极易混淆, 必须钉清楚以免未来误读。

区别1 目的: digital-evolution-sandbox 的目的是跑出一个大型时序原始数据集 (每 tick dump 512×512 格子世界), 它明确声明"不再反推 φ"(因为 f=φ∘a 那条公式在 selop 这边已判否), 纯粹忠实采集进化全过程数据。而 selop-v3 引擎的全部目的恰恰相反——就是为了验证"能不能从幸存者反推 φ/σ/a_invivo", 真值的旁路记录与对账是它的核心。

区别2 真值: digital-evolution-sandbox 不预设真值算子 (表型=序列的固定函数, 严禁手写"谁强"), 它是无目标的红皇后混战。selop-v3 引擎则显式埋入三个被记录的真值算子 (B 层) 作为因果驱动律。

区别3 数据基底: digital-evolution-sandbox 用合成的字母族/注册表基元自造世界。selop-v3 引擎严格建立在真实开源数据 (CR9114/CR6261 DMS, S5F, repertoire) 之上, 只合成三个待恢复算子。

共同点 (为什么易混): 两者都是 agent-based、都 dump 时序数据、都守"无私货红线"。但前者是"忠实采集无目标进化", 后者是"带记录真值的受控验证台"。一句话区分: digital-evolution-sandbox 是观察工具 (看红皇后跑出什么), selop-v3 引擎是验证工具 (看恢复方法能否逆向已知真值)。未来读到"沙盒"二字务必先确认是哪一个。

### 深入技术记录 13: tick 事件顺序问题预览 (问题 C, 下一个机制决策)

分辨率定甲之后, 紧接着要拍的机制决策是"一个 tick 内事件按什么顺序发生"。本节预览这个问题的结构, 为下一段 (或本段之后) 做准备。GC 一轮循环的生物学事件大致是: 暗区增殖 → SHM 突变 → 迁移到光区 → 表达 BCR → 从 FDC 力学提取抗原 (a_invivo 起作用) → 向 Tfh 提呈竞争 T-help (σ 起作用) → 选择存活 → 返回暗区或出营 (φ 在返回后的增殖里起作用)。

为什么顺序要命: 涌现真值是"在某个事件点记账"得到的, 记账点的位置 (在突变前还是突变后、在选择前还是选择后) 直接决定记到的 φ/σ/a_invivo 是什么含义。如果顺序错位, 比如在选择之前就记 φ, 那记到的就不是"选择压力下的适应度"而是别的东西, 涌现真值的语义就错了, 后面恢复模型再准也是恢复了一个错误定义的量。所以 tick 顺序不是工程细节, 是真值定义的一部分。

异相位时钟问题: 真实 GC 里不同 B 细胞不在同一节拍 (有的刚进光区有的在暗区中途)。digital-evolution-sandbox 用了异相位时钟 (T-birth)%period 来处理这个。selop-v3 引擎是否需要异相位、还是可以同步轮次, 取决于异相位是否影响 P1 (克隆干涉本身就和时序错配有关, 所以可能需要), 这是问题 C 的一个子问题。

候选顺序方案 (待下一段定): (方案甲) 严格同步轮次, 所有细胞同节拍, 简单但可能抹掉时序错配带来的真实克隆干涉; (方案乙) 异相位时钟, 每个克隆有自己的节拍相位, 更真实但更复杂; (方案丙) 连续时间 Gillespie 类事件驱动, 最真实 (事件按速率随机发生) 但计算最重。倾向乙或丙, 因为 P1 (克隆干涉) 是核心检验对象, 不能用同步轮次把它人为简化掉——这又是一次"不能怎么简单怎么来"的应用。

### 深入技术记录 14: 本段方法论自洽性的三个亮点 (值得未来复用的思路)

本段产出里有三个方法论思路特别自洽, 值得在后续子项目和未来课题里复用, 单独记下。

亮点1 把不可验证的欠定性编织成稳健性检验 (决议六): 多视图整合数学上欠定 (拼接不唯一) 本来是致命缺陷, 但通过"生成一族合理拼接 + 要求结论在全族成立"把它转成了 P1 的稳健性压力测试。这是一个通用模式——当一个不可避免的不确定性无法消除时, 不要假装它不存在, 而是把它显式参数化成一族, 让关键结论在全族上经受考验。这与对抗轮"部分辨识 + 适用包络"完全同构。

亮点2 用外部可复现先验替代私人假设 (发现7 的 embedding 逻辑): 欠定问题必须注入结构假设, 但"我私下选的假设"和"大规模预训练表征隐含的先验"在可辩护性上天差地别。后者公开、可复现、可被第三方质疑。这是把"减少私货"具体化的一个可操作手段——能用外部可复现先验的地方, 绝不用私人选择。

亮点3 职责隔离作为防循环的总原则 (决议二/五 + 技术记录3): A 层 (逼真) 和 B 层 (真值) 严格隔离, 是防止"把答案塞进验证台"这类循环的总闸。技术记录3 显示, 几乎所有新型循环风险 (A 层渗入真值、do() 绕过个体机制、表征泄漏 φ) 都归结到职责隔离被破坏。把"防循环"这个抽象目标落成"职责隔离"这条可检查的工程原则, 是本段最有操作性的认识论产出。

### 深入技术记录 15: Regime 2 真实数据验证与判别第四路 (引擎在大局里的位置)

引擎主要服务 Regime 1 (合成真值验证), 但它也间接关联 Regime 2 (真实数据验证) 和判别第四路, 记下这层关联以免引擎设计时与下游脱节。

Regime 2 真实数据: 在真实 repertoire 时序上跑 P1。注意真实数据没有真 φ (Weinstein 墙), 所以 Regime 2 不能直接验证"恢复准不准", 它验证的是"P1 这个可观测现象 (富集比非单调) 在真实数据里是否也出现"。如果 Regime 1 (合成) 里 P1 成立、Regime 2 (真实) 里也观测到同样的非单调模式, 那是骨架的强支持; 如果合成里成立但真实里看不到, 那暴露合成与真实的差距, 是诚实的红旗。引擎的作用是为 Regime 1 提供有真值的对照, 让 Regime 2 的真实观测有一个"在已知真值下该长什么样"的参照系。

判别第四路 (对抗轮交棒约束⑤): 这是基线 (子项目3) 承担的角色——自底向上、从 CR9114/CR6261 真实 DMS 出发、对测度流答案盲、纯统计视角地拟合。它和引擎的关联是: 拿引擎产出的合成数据 (已知真值) 喂判别第四路, 看它收不收敛到与我们骨架同一结构。收敛 = 独立佐证 (直接回应对抗轮 N_eff≈1.3 的框架诱导质疑——证明结论不是只有"我们这一路"才得出); 不收敛 = 红旗。所以引擎的输出1 (幸存者序列时序) 必须设计成判别第四路也能直接消费的纯统计形式, 不能预设任何测度流结构 (否则就给第四路喂了答案, 又是循环)。这对输出契约 (技术记录8) 加了一条: 输出1 必须是"对所有视角中立"的原始幸存者数据, 不带任何理论框架的预处理。

引擎在三子项目大局里的位置总结: 引擎是真值工厂, 同时喂三条下游——恢复模型 (子项目2, 主角, 检验能否逆向真值)、基线/判别第四路 (子项目3, 既是被超越的对照又是独立佐证的载体)。它产的同一份输出1 服务所有下游, 保证可比性; 它独有的输出2 (旁路真值) 是合成路线相对真实数据的唯一且决定性的优势。引擎设计的每一个决策, 最终都要回答"它是否让这三条下游都能公平、无循环地消费"。

### 术语表 (本段出现的关键概念, 供零上下文读者)

agent-based 模型: 以单个个体 (这里是 B 细胞) 为单位、只定义个体规则、让群体规律自涌现的模拟方法。对立面是自上而下直接写群体公式。

A 层 / B 层: 引擎的两层结构。A 层 = 数据驱动的逼真环境 (舞台), 不含私货; B 层 = 三个有物理依据被记录的真值算子 (导演), 极少量已知物理。两层职责严格隔离。

world model: 学习一个环境动力学的模型。本项目里用于搭 A 层逼真环境, 但明确不能当真值源 (它学不到 φ)。

φ / σ / a_invivo: 三个待恢复的真值算子。φ = 竞争扩增适应度 (频率依赖); σ = 存活采样算子 (T-help 瓶颈, 不可逆多对一); a_invivo = 体内力依赖抗原提取能力 (Bell-Evans catch-bond)。

a_DMS vs a_invivo: a_DMS = DMS 测的零力平衡 K_D (体外可测, 输入); a_invivo = 体内力依赖 off-rate (恢复目标)。两者的物理鸿沟是裂缝 B-S2。

Weinstein 墙: Weinstein 2022 证明 φ 从纯观测数据原理上不可辨识。这是为什么必须造有真值的合成引擎来验证。

回声室 / 自欺: 自己定规则又验证自己能恢复自己定的规则, 看似成功实则零信息。引擎设计的头号敌人。

保真度闸门: 体检引擎产出的可观测统计像不像真实数据的硬门槛, 不通过不许用。把"适度合成"变成可检验承诺。

多视图对角整合 (diagonal integration): 把各只覆盖部分维度、且测的不是同一批样本的数据集对齐到共享隐空间、还原高维全貌的技术。数学上欠定不唯一。

P1 / P2: 骨架的两个可证伪预测。P1 = 克隆干涉区富集比与 φ 非单调; P2 = do(冻结环境) 恢复单调性。

R_irr: 不可辨核 = Φ 零谱 = Noether 对称轨道, 原理上无法从观测恢复的残余。

三条非循环铁律: ① 引擎个体级、不直接写测度流 PDE 当真值源; ② φ/σ/a_invivo 涌现+记录、不手写真值函数; ③ do() 个体级真干预。引擎合法性的宪法。

### 深入技术记录 16: do() 个体级干预的设计空间 (问题 E 预分析)

do() 是因果验证 (P2) 的执行机构, 也是铁律③的落点。本节预分析它在 agent-based 引擎里的可能实现, 为问题 E 打底。do() 的本质是"反事实生成"——在同一初始条件、同一随机种子下, 跑一条干预轨迹和一条对照轨迹, 两者之差就是干预的因果效应。

do(冻结环境) 的候选实现: (实现甲) 钉死抗原浓度——在每个 tick 把光区可获取抗原量设为常数, 不再随被消耗而下降, 消除"抗原稀缺导致的频率依赖"; (实现乙) 钉死竞争场——让每个 B 细胞获取 T-help 的概率不依赖于同场其他细胞的相对提呈量, 而是只依赖自身绝对量, 消除"克隆间竞争"; (实现丙) 同时钉死两者。P2 要消除的是频率依赖/克隆干涉, 所以实现乙是最直接的 (竞争场是频率依赖的核心), 实现甲是补充。具体选哪个或组合, 取决于 P1 的非单调到底主要由哪个机制产生, 这要靠引擎跑出来才知道 (本身就是一个待验子问题)。

do() 必须穿透 A 层的约束 (技术记录3 铁律③审查的落实): 不能用"在 A 层 world model 的条件变量上设值"来冒充 do()。如果 A 层是一个学了真实分布的生成模型, 在它的条件输入上设值得到的是"观测条件化" P(·|环境=冻结), 而不是因果干预 do(环境=冻结)——前者会带入环境与其他变量的混杂相关, 后者切断了所有指向环境的因果箭头。正确做法是 do() 直接作用于 B 层个体机制 (改细胞实际遵守的抗原获取/竞争规则), 让 A 层环境随之自然响应, 而不是在 A 层分布上做手脚。这个区分是 P2 能否真正测到因果的命门, 也是整个项目"因果不可辨"主题在引擎层面的直接体现。

反事实配对的随机种子管理: 对照轨迹和干预轨迹必须共享随机种子直到干预点, 干预点之后干预轨迹按新机制跑、对照轨迹按原机制跑。这样两条轨迹的差异纯粹来自干预而非随机性差异。这是 agent-based 引擎实现 do() 的一个具体工程要求, 容易遗漏但对 P2 的统计功效至关重要。

### 深入技术记录 17: 涌现真值的记账粒度 (问题 D 预分析)

涌现真值"记什么、记多细"直接决定恢复模型能对账到什么程度, 也决定旁路记录的存储开销。本节预分析记账粒度的设计空间。

候选粒度: (粒度1 每序列每轮) 记录每个独特序列在每一轮的真 φ/σ/a_invivo 涌现值——最细, 恢复模型可逐序列对账, 但若序列空间巨大存储重; (粒度2 每克隆每轮) 按克隆 (谱系) 聚合记录, 比每序列粗, 但克隆是 P1 (克隆干涉) 的天然单位, 可能更贴合检验; (粒度3 每群体每轮) 只记群体级统计 (如平均 φ、φ 分布矩)——最粗, 省存储但丢失逐序列辨识能力, 与"恢复每序列算子"目标不符。倾向粒度1或2: P1 检验需要"富集比 (可观测) vs φ (真值)"的逐单元配对, 单元是序列还是克隆取决于富集比在哪个粒度上定义。

记账的语义锚点 (与 tick 顺序耦合, 技术记录13): φ 的真值要在"选择之后的增殖"事件点记 (否则不是选择压力下的适应度); σ 的真值要在"T-help 瓶颈"事件点记 (通过/未通过的概率); a_invivo 的真值要在"力学提取"事件点记 (实际获取抗原量)。三个算子的记账点不同, 这再次说明 tick 顺序是真值定义的一部分 (技术记录13)。记账本质上是在每个机制事件发生时, 旁路写下"这个细胞在这个机制里实际经历了什么", 这些原始记账事后聚合成 φ/σ/a_invivo 的涌现真值。

记账 vs 输出1 的严格隔离: 记账数据 (输出2) 和幸存者序列时序 (输出1) 必须在数据管线上物理隔离, 确保恢复模型在任何环节都接触不到输出2。一个工程保障是: 输出1 和输出2 写到不同文件/不同访问权限, 恢复模型的数据加载器只允许读输出1。这把"恢复方法看不到真值"从一个口头约定变成一个代码层强制的隔离, 防止无意泄漏。

### 深入技术记录 18: 工程分期与算力的初步考量 (问题 H 预分析)

用户明令"不要怎么简单怎么来", 但也不能一次铺到跑不动。本节预分析工程分期, 在"够真实"和"跑得动"之间找一条不牺牲核心验证力的路径。注意分期不等于做 MVP——MVP 是"先做个能跑通逻辑的小玩意交付", 分期是"先建能产出真信号的最小完整链路, 后续按需加厚", 区别在于每一期都必须能产出对核心主张有判定力的结果, 而不是只验证逻辑闭环。

期1 单一逼真世界 + P1 主检验: 用 CR9114 真实 DMS 搭 A 层 (先用一种拼接而非全族, 但 A 层逼真度必须过保真度闸门)、埋三个 B 层算子、跑出克隆干涉、检验 P1 (富集比 vs φ 非单调)。这一期的判定力: 如果 P1 在一个过了保真度闸门的逼真世界里都不成立, 骨架核心推论就已经动摇, 这是有真信号的, 不是 MVP。期1 不是为了"先跑通", 而是为了"先拿到骨架最核心推论的第一个真判定"。

期2 do() + P2 因果对照: 加入 do(冻结环境), 检验 P2。这一期把 P1 的非单调性从"相关"升级到"因果"——确认非单调确实来自频率依赖/克隆干涉。

期3 一族拼接 + P1 稳健性 (决议六): 把单一拼接扩成一族通过闸门的拼接, 跑 P1 稳健性压力测试, 区分真信号与 artifact。

期4 5 个对抗世界 + 包络内外: 逐个造 5 个对抗真值世界 (技术记录6), 检验方法在该失败处是否诚实失败, 并分包络内外测。

期5 第二抗体 (CR6261) + Regime 2 真实数据: 用 CR6261 验证结论不依赖单一抗体, 并在真实 repertoire 时序上观测 P1 模式。

算力考量: agent-based GC 模拟的成本主要在群体规模 N × 轮次 × 序列空间探索。真实 GC 的 B 细胞数量级在 10^3–10^4/GC, 轮次在几十次循环, 这个规模常规计算可承受。成本大头可能在 A 层 world model 的训练 (若用大预训练表征 + 生成模型) 和一族拼接的多次重跑。分期让算力随判定需求逐步投入, 而非一次性全押。

### 深入技术记录 19: 逼真度与诚实失败的张力 (一个必须时刻平衡的设计哲学)

引擎设计里有一个贯穿始终的张力, 值得显式记下: 逼真度 (A 层要像真生物) 和诚实失败 (5 个对抗世界要让理论该垮就垮) 之间, 有时会拉扯。

张力的来源: 保真度闸门要求引擎产出像真实数据, 但对抗世界 (比如世界5 真存在静态 φ) 故意构造一个可能不那么像真实生物的极端世界来考验方法。如果一刀切要求所有对抗世界都过保真度闸门, 可能造不出足够极端的对抗世界; 如果对抗世界不过闸门, 又怕它太不真实导致失败判定无意义。

解法原则: 区分"逼真度闸门"和"对抗性"作用在不同层。A 层逼真度闸门保证"环境物理 (克隆动力学、突变谱) 像真实"; 对抗性作用在 B 层算子机制的替换 (技术记录6 显示 5 个世界里 4 个改 B 层)。所以一个对抗世界可以同时是"A 层逼真 (过闸门) + B 层算子是对抗性配置"。世界5 (静态 φ) 是唯一可能挑战这个解法的——它改的是 B 层 φ 的本质, 但只要 A 层环境物理仍过闸门, 这个世界就仍是"逼真环境里跑一个静态 φ 真值", 失败判定依然有意义 (它检验的是方法在静态 φ 真存在时能否检测到, 这本身就是该测的)。

这条张力的意义: 它提醒后续设计不要把"逼真"和"对抗"对立起来。正确的心智模型是: A 层永远逼真 (这是外推效力的保证), B 层在 5 个世界里取不同的真值配置 (这是对抗的来源)。逼真是环境的属性, 对抗是真值的属性, 两者作用在不同层所以不冲突。这又一次印证了 A/B 分层 (决议二) 是本段最具生产力的结构决策——它不仅防循环 (技术记录3)、降工程量 (技术记录6), 还化解了逼真与对抗的张力。

本段把引擎的顶层定位锁死 (六条决议), 但所有机制细节还没动。下一段 brainstorm 的议程按优先级排定如下, 供续接时直接开工。

议程的总顺序逻辑: 先定 A 层怎么搭 (问题 A) 和 tick 怎么转 (问题 C), 因为这两个决定了引擎的骨架; 再定真值怎么记 (问题 D) 和怎么接真实数据 (问题 B), 因为这两个决定了引擎产什么; 最后定怎么干预 (问题 E)、怎么体检 (问题 F)、怎么造对抗世界 (问题 G)、怎么分期 (问题 H), 因为这些是在骨架之上的扩展。

### 深入技术记录 21: 与 D 段 (wet-lab 最小介入) 的连接 (终点预埋)

用户选了 A+D 路线, D 段 (投影成 wet-lab 可行的最小介入) 留到沙盒完再定。但引擎设计时应预埋通往 D 的接口, 以免到时发现引擎产出无法导出 wet-lab 可执行的东西。

连接点1 P1 的可观测性: P1 (富集比 vs φ 非单调) 里, 富集比是 wet-lab 真能测的 (测序前后丰度比), φ 是隐藏的。如果引擎确认 P1 成立, D 段的 wet-lab 实验就是"在真实 GC 或体外系统里测富集比, 看是否出现预测的非单调"。引擎要保证它定义的"富集比"和 wet-lab 能测的富集比是同一个量 (同样的时间窗、同样的丰度定义), 否则 D 段无法对接。

连接点2 do(冻结环境) 的 wet-lab 对应: P2 的 do(冻结环境) 在 wet-lab 里对应什么真实干预? 可能是体外固定抗原浓度的培养系统、或阻断频率依赖的实验设置。引擎设计 do() 时 (问题 E), 应优先选那些有真实 wet-lab 对应物的干预, 而不是只有模拟里能做的抽象干预。这让 P2 的因果检验有朝一日能落到真实实验。

连接点3 最小介入的可计算性 (B5 奥卡姆诊断): 对抗轮交棒约束⑦要求 B5——最小介入优化的子模性 vs 三子空间直和度。引擎要能产出计算这个诊断所需的量 (三子空间的几何、介入的效应), 这样"最小介入是什么"成为一个可在引擎里计算、再投影到 wet-lab 的优化问题, 而不是拍脑袋指定。

D 段预埋的总原则: 引擎不只是验证理论的内部工具, 它的产出要能"导出"成真实实验的设计输入。这呼应用户的最终目标——不为发论文, 为实际推动科学。引擎里每一个可观测量 (富集比、介入效应) 都应问一句"它在 wet-lab 里测得到吗", 测得到的才是通往 D 的桥。这条原则现在不强约束设计 (D 留到最后定), 但作为方向预埋, 避免引擎产出和真实实验脱节。

### 深入技术记录 22: 本段没有走的弯路与被否决的选项 (留档以免重蹈)

记录本段中被否决或主动收回的选项, 这些"没走的路"对未来同样有价值——它们标记了已经探查过的死胡同。

被收回的选项1 我自己推的"B 最小垂直切片": 在本段开头, 我曾建议先做一个能闭环的最小切片 (引擎+1恢复模型+1对抗世界+P1)。用户的核心要求 ("做真能用的东西, 不要好验证的 MVP") 让我当场收回这个建议, 并认识到对引擎而言"怎么简单怎么来"在逻辑上自我作废 (简单引擎=手写闭式 φ=循环=零信息)。这个收回是本段的一个重要转折, 它把整段定调为"宁可工程量大也要真值干净"。注意: 技术记录18 的分期 (期1-5) 不是对这个收回的违背——分期的每一期都产出对核心主张有判定力的真信号, 而被否决的 MVP 是"先验证逻辑闭环"。两者的区别是判定力的有无, 不是规模大小。

被否决的选项2 直接用 world model 当真值源: 用户提出 world model 设想后, 一个诱人的简化是"训个强 world model 拟合真实数据, 直接拿它当真值环境"。本段否决了它, 理由是 Weinstein 墙——world model 学不到 φ 真值, 这样的环境作为验证台是空的 (发现2)。这个否决催生了 A/B 分层。

被否决的选项3 直接套 MOFA/GLUE 做多视图整合: 多视图整合有现成成熟工具, 直接套是最省事的。但原创红线 (memory feedback_originality_requirement) 禁止"现成工具套新域", 所以还原机制必须由问题结构原创逼出 (问题 A)。这个约束让 A 层的实现变难, 但守住了项目的原创性底线。

被否决的选项4 退到乙/丙分辨率: 群体遗传 (乙) 或混合 (丙) 分辨率更轻, 但会把 σ/φ/a_invivo 三层揉回一两个有效量, 在引擎层抹平待验问题 (隐蔽循环)。用户选甲, 否决了这个简化。

这四个被否决选项有一个共同模式: 它们都是"更省事但会损害真值纯度或原创性"的选项, 而本段的每一次否决都选择了"更难但更诚实"的那条路。这与用户反复强调的底线 ("做真的有用的最终拿来用的东西") 完全一致, 也与对抗轮的求真精神一脉相承。把这个模式记下来, 是为了在后续子项目里继续用同一把尺子做取舍。

### 深入技术记录 25: B 层三算子的"已知物理"到底已知到什么程度 (兑现"极少量已知物理"的承诺)

决议二说 B 层用"极少量已知物理"。这句话的可信度取决于这些物理到底有多"已知"。本节逐个核查三个算子背后的物理依据的成熟度, 因为如果某个"已知物理"其实是我包装的私货, 那决议二就破功了。

a_invivo 的物理 = Bell-Evans 受力键动力学: 这是单分子力谱领域的成熟理论, 起于 Bell 1978、Evans-Ritchie 1997。核心是键的解离率随外力指数变化 koff(F)=koff(0)·exp(F·xβ/kBT) (slip bond), 以及 catch bond 的反常增稳。这套理论有大量单分子实验 (AFM、光镊、生物膜力探针) 支撑, 不是我编的。把它用到 B 细胞从 FDC 拽抗原, 是有文献基础的 (B 细胞用力学测试 BCR-抗原键已被实验观察)。成熟度: 高, 是真"已知物理"。引擎里需要的私货只是"用哪组力谱参数", 而参数来自真实力谱数据 (技术记录2)。

σ 的物理 = T-help 限制的竞争性选择: GC 光区 B 细胞竞争有限 Tfh 帮助、按抗原提呈量获取存活信号, 是免疫学的既定共识 (Victora-Nussenzweig 等的 GC 动力学工作)。"多对一不可逆采样瓶颈"是对这个共识的数学抽象。成熟度: 机制定性已知 (谁竞争、竞争什么), 定量函数形式 (提呈量→存活概率的精确映射) 较少直接测量, 所以这里我有一点建模自由度。这点自由度是 B 层私货的主要驻地, 必须显式标注并在对抗世界里测其敏感性 (世界1 σ 部分可吸收正是测这个)。诚实地说: σ 是三算子里"已知物理"成色最弱的一个, 这恰恰也是为什么它在 v3 里被特别升为正交支柱、被对抗世界重点攻击。

φ 的物理 = 频率依赖竞争与克隆干涉: 群体遗传学里克隆干涉是成熟概念 (Gerrish-Lenski 1998 等), 频率依赖选择也是。把它用到 GC 克隆扩增是合理的。成熟度: 概念成熟, 但"GC 里克隆干涉的具体强度"是经验问题。φ 的关键不是它的函数形式而是它的非静态性 (骨架主张:不存在每序列静态 φ), 引擎要做的是让 φ 从竞争里涌现而非赋一个静态值, 这本身就是在检验骨架而非假设它。

核查总结: a_invivo 的物理最硬 (高成熟度), φ 次之 (概念成熟), σ 最软 (定性已知定量有自由度)。这个梯度很重要——它告诉我们 B 层私货风险主要集中在 σ 的定量形式上, 所以保真度闸门和对抗世界应对 σ 给予最高关注。把这个"已知物理成熟度梯度"记下来, 是对决议二"极少量已知物理"承诺的诚实兑现与风险定位: 不是三个算子都同等已知, σ 是需要盯防的薄弱点。

### 深入技术记录 26: 引擎与求真四轴的关系 (评价标准的延续)

对抗轮用"求真四轴"给主张打分 (A1-A4 那套, 含"打过基线"A3 等)。引擎产出的结果最终也要按求真精神评价, 记下这层延续以保持评价标准一致。

引擎不产 resilience 分: 延续对抗轮的反 paper 化立场, 引擎跑出的结果不打"稳健性得分"这类发表导向指标, 而是产出三桶判定 (BROKEN/CORROBORATED/UNFALSIFIABLE) 的素材。P1 在逼真世界 + 全族拼接下成立 → CORROBORATED 素材; P1 垮 → BROKEN 素材; P1 结果依赖不可检验的拼接选择且无法区分 → UNFALSIFIABLE 警告。

引擎让"打过基线"(A3) 可操作: 基线 (子项目3) 吃同一份输出1, 引擎因此让"我们的恢复方法 vs 基线"成为同输入下的公平对比, A3 从口号变成可测。同时基线作判别第四路, 引擎的合成真值让"第四路收不收敛到骨架结构"可测 (技术记录15)。

引擎服从 falsifiability 优先于 novelty: 引擎设计的每个选择问的是"它能让什么计算证伪骨架", 而非"它新不新颖"。这与 memory feedback_anti_paper_ification 一致。本段反复出现的"这个设计能让 P1/P2/θ/ε 被干净测出吗"就是这条标准的应用。

评价标准延续的意义: 引擎不是一个中立的工具, 它继承了对抗轮的整套求真价值观——宁可诚实 BROKEN 不要虚假 CORROBORATED, 宁可承认 UNFALSIFIABLE 不要强行结论。这保证 Round 3 和 Round 2 是同一场求真活动的延续, 而非另起炉灶的新发表尝试。这也再次呼应用户最终目标: 推动科学, 不发论文。

### 深入技术记录 27: 为什么"合成引擎"在逻辑上是必要的而非偷懒 (回应可能的根本质疑)

有人 (包括未来的我们自己, 或对抗第四路) 可能质疑: 既然有真实 DMS 数据, 为什么不直接在真实数据上验证恢复方法, 非要造合成引擎? 这个质疑如果不正面回答, 整个引擎的存在合法性就悬空。本节把"为什么必须合成"的逻辑链完整钉死。

逻辑链第一环: 验证一个恢复方法, 需要知道真值 (ground truth)。要判断"方法恢复出的 φ 对不对", 必须有一个"真的 φ"来对账。这是验证的定义本身。

逻辑链第二环: 真实数据里没有 φ 的真值, 而且原理上没有。Weinstein 2022 证明 φ 是因果量, 从纯观测数据不可辨识——不是"现在还没测到", 而是"原理上测不到"。真实 repertoire 只记录了谁活下来, 从未也无法记录每序列的真适应度。所以真实数据不能提供验证所需的 ground truth。

逻辑链第三环: 唯一能同时满足"有真值"和"像真实"的, 是建立在真实数据底基上的、机制化涌现真值的合成引擎。合成引擎的 B 层旁路记录给出真值 (满足第一环), A 层真实数据底基 + 保真度闸门给出逼真度 (尽量接近真实), 两者兼得。这不是偷懒的替代品, 而是 Weinstein 墙下逻辑上唯一可行的验证路径。

逻辑链的反面确认: 如果有人说"那合成的 φ 不是真实的 φ 啊", 这话对但不构成反驳——我们验证的不是"恢复方法在真实世界对不对"(那原理上不可验证), 而是"恢复方法在一个真值已知的逼真世界里, 能否逆向出真值"。如果方法连真值已知时都恢复不出, 它在真实世界更不可能可信; 如果方法在多个逼真合成世界 (含对抗世界) 里都能诚实表现 (该恢复时恢复、该报失败时报失败), 那才有资格去碰真实数据。合成引擎是恢复方法的"资格考试", 通不过的没资格上真实数据。

这条逻辑链的最终落点: 合成引擎不是因为真实数据难搞而退而求其次, 而是因为验证这件事在 Weinstein 墙下逻辑上要求一个有真值的世界, 而这样的世界只能合成。把这条钉死, 引擎的存在合法性就不再悬空——它是被问题的认识论结构 (φ 不可辨识) 逼出来的必然, 不是工程便利的选择。这也正面回应了用户最初的"自娱自乐"之问的最深一层: 不是自娱自乐, 是认识论上唯一开得出的门。

### 深入技术记录 29: A 层 world model 与"测度流骨架"的同源性及其陷阱 (一个需要警惕的诱惑)

本段反复提到 A 层可以用 Diffusion 生成序列空间测度的演化 (技术记录11), 而骨架的核心动力学原理正是序列空间测度 μ_t 的非平衡随机过程。这两者的同源性是一把双刃剑, 必须把它的诱惑和陷阱讲清楚, 否则下一段设计 A 层时极易踩雷。

同源性是真的: Diffusion 模型的 score function ∇log p 与 Fokker-Planck/测度流的 drift 项在数学上确实同构 (相2 已暴露这个落点: score = ∇log μ_t = FV drift)。所以用 Diffusion 当 A 层克隆动力学生成器, 在数学语言上和骨架的测度流是同一套对象。这让"用 Diffusion 搭 A 层"显得格外自然、优雅。

陷阱在于: 正是这个同源性, 让"A 层 Diffusion 偷偷变成 B 层真值源"的循环风险达到最大。如果 A 层 Diffusion 学的是"真实数据里测度怎么流", 而骨架主张"测度流就是真值", 那么用 A 层 Diffusion 生成的演化去验证"恢复方法能否恢复测度流", 就是拿一个学了答案的模型去考一个该独立推出答案的方法——典型循环。优雅的数学同源性在这里变成了认识论的陷阱: 越是同源, 越容易混淆"逼真度生成器"和"真值源"。

防线: 必须坚持技术记录3 铁律①审查的结论——A 层 Diffusion 只生成可观测统计 (序列丰度分布) 的逼真样本, B 层真值仍由个体级机制涌现并旁路记录, 两者数据物理隔离。具体到实现: A 层 Diffusion 的训练目标是"匹配真实可观测统计", 它的输出喂给 agent-based 引擎当环境素材 (比如初始群体、抗原可得性的逼真背景); 它绝不直接输出 φ/σ/a_invivo, 也绝不被恢复模型当对账目标。换句话说 A 层 Diffusion 和 B 层真值即使数学同源, 在数据管线上也必须是两条不相交的河。

为什么现在记这条: 因为下一段一旦开始设计 A 层实现, "用 Diffusion 又优雅又同源"的诱惑会非常强, 而它恰恰是本项目最隐蔽的循环入口。把这个陷阱提前钉在记录里, 是给下一段的自己一个明确的警告标志: 同源 ≠ 可以共用, 优雅 ≠ 安全。这也是对抗轮"elegance-trap"教训 (四语言同构曾被误当统一证据而 BROKEN 降级) 在引擎实现层面的直接复现与防范——同一个优雅陷阱, 换了个地方又会出现, 必须用职责隔离这条硬墙挡住。

如果未来续接者只有一分钟, 这一段就是全部: selop-v3 引擎是一个真值工厂——底座是用尽量多开源真实数据 (CR9114/CR6261 DMS + S5F + repertoire + 力谱) 通过多视图对角整合搭出的"非常厚"的逼真生发中心环境 (A 层, 不含私货, 过保真度闸门), 上面由三个有物理依据、被旁路记录的真值算子 (B 层: a_invivo 走 catch-bond / σ 走 T-help 瓶颈 / φ 走频率依赖竞争) 作为因果驱动律推动世界演化; 它产出幸存者序列时序 (恢复方法唯一能看见的) + 旁路真值 (只用于对账) + do() 干预数据, 喂给恢复模型 (逆向真值) 和基线/判别第四路 (公平对比 + 独立佐证); 它的存在是被 Weinstein 墙 (φ 原理上不可辨识) 逼出的认识论必然, 不是工程偷懒; 它的头号敌人是循环/自欺, 唯一防线是 A 层 (逼真) 与 B 层 (真值) 的硬隔离; 它服务于把骨架的可证伪预测 P1/P2 翻成 CORROBORATED/BROKEN, 并最终预埋通往 wet-lab 最小介入 (D 段) 的桥。本段锁了六条定位决议, 下一段攻 A 层还原机制 (原创, 不套 MOFA) 和 tick 事件顺序 (真值定义的一部分)。

### 深入技术记录 31: 三个常见误读的预先澄清 (给未来读者排雷)

本段的定位涉及几个反直觉的点, 极易被未来读者 (或对抗者) 误读。预先澄清三个最可能的误读, 避免续接时浪费时间重新辩论。

误读1 "合成数据=不可信": 直觉上"合成"听起来不如"真实"可信。但本段已论证 (技术记录27), 在 φ 不可辨识的认识论约束下, 合成是验证恢复方法的唯一逻辑可行路径, 真实数据反而无法提供验证所需的真值。所以这里"合成"不是可信度的降级, 而是验证逻辑的必要前提。正确的可信度判据不是"真实 vs 合成", 而是"A 层是否过保真度闸门 + B 层物理是否成熟 + 是否守住非循环铁律"。

误读2 "A 层用了学习模型 = 引擎不再是机制模拟": A 层引入 world model/embedding 可能让人以为引擎从"机制模拟"滑向了"数据拟合"。澄清: A 层学习模型只负责生成逼真的环境素材 (背景统计), 引擎的因果核心 (谁繁殖谁死) 仍由 B 层个体机制驱动 (决议三)。引擎本质仍是机制模拟, 学习模型是它的"布景师"不是"导演"。这个区分由 A/B 隔离 (决议五) 保证。

误读3 "三正交支柱已被证明 = 引擎来确认它": 引擎不是来确认三正交支柱的, 而是来检验它在 5 个对抗世界 (含故意让三通道共线的世界4、故意让 σ 可吸收的世界1) 里会不会失败。引擎对骨架的态度是攻击性的求真, 不是辩护性的确认。如果引擎只会输出"骨架对", 那它就是回声室。它的价值恰恰在于能输出"骨架在这个世界垮了"。

这三个误读的共同根源是把引擎当成"证明我们对的工具", 而它实际是"逼我们的理论在最逼真环境里诚实经受 BROKEN/CORROBORATED 判定的考场"。把这层心态钉死, 是理解本段全部决议的钥匙, 也是续接时不跑偏的定盘星。至此本 checkpoint 完成对 selop-v3-engine 第一段 brainstorm 的完整记录。

### 深入技术记录 32: 引擎设计的一条贯穿原则——每个选择都问"它服务哪个可证伪判定"

回看本段全部 32 条记录, 它们背后有一条统一的方法论纪律值得最后单独抽出: 引擎的每一个设计选择, 都必须能回答"它最终服务于哪一个可证伪判定 (P1/P2/θ/ε/包络/B5)"。这条纪律是防止引擎膨胀成"什么都模拟一点但什么都验证不了"的护栏。

应用举例: 为什么 tick 顺序要异相位 (技术记录13)? 因为它服务 P1 (克隆干涉需要时序错配)。为什么记账要细到每序列/每克隆 (技术记录17)? 因为 P1 需要逐单元的"富集比 vs φ"配对。为什么 do() 要穿透 A 层 (技术记录16)? 因为 P2 需要真因果而非观测条件化。为什么要一族拼接 (决议六)? 因为它服务 P1 的稳健性判定。每一条都能追溯到一个具体判定, 没有一条是"因为这样更全面/更真实"的无锚扩张。

反面警示: 任何无法回答"服务哪个判定"的引擎特性, 都是膨胀的征兆, 应当砍掉或推迟。比如"要不要模拟 B 细胞的代谢状态""要不要建抗体糖基化"——除非它们能连到某个可证伪判定, 否则就是把引擎做大但不做利。这条纪律让引擎的复杂度始终由验证需求拉动, 而非由"模拟得更像"这个无底洞拉动。它和技术18 的分期、技术14 亮点3 的职责隔离一起, 构成引擎"不膨胀、不偷懒、不自欺"的三道工程纪律。

这条原则也是对用户底线的最终呼应: 用户要"真能用、最终拿来用"的东西。一个每项特性都锚定可证伪判定的引擎, 它的每一行复杂度都在为"把骨架逼到该垮就垮"服务, 没有一行是为了好看或好验证。这就是"真能用"在引擎设计层面的精确含义——有用 = 每个部件都在产生对核心科学主张的判定力。本 checkpoint 到此结束。
