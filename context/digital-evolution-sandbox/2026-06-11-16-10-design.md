# Digital Evolution Sandbox — 设计文件

> **项目目标 (已锁, 2026-06-12 修正).** 本沙盒的目的是**采集一批原始时序数据**, 不承担任何"反推/验证"职责。原 `f=φ∘a` 锚定已在 selop 课题侧被判否、推翻 → 沙盒**不再以"反推 φ"为目标**。因此: ① 无"分析侧/反推成功判据"的设计负担; ② "不夹私货/φ干净"这些原则**仍然保留**(它们保证采集到的数据是干净的涌现动力学, 不被手写偏置污染), 但措辞里的"反推 φ"改读作"保持数据是纯涌现的"; ③ 数值轮的判据 = 动力学健康度(红皇后转不转、无无脑碾压、不一步全灭), 跑起来肉眼+简单统计即可, 不需复杂分析侧。
> **采集 = 开发期的事 (schema 已锁 2026-06-12, 甲)**: 每个 tick 推进后 dump 整个世界状态。**存储格式**: 每局一个 parquet 文件, 文件名 = 时间戳(`{timestamp}.parquet`)。**结构 = long format / 稀疏行式**, 每行 = `(tick, cell_x, cell_y, strain, count)`, 其中 **strain 列存完整序列字符串**(不做 ID 映射表 —— parquet 字典编码自动压掉重复序列, 单文件自包含、无需 join)。**只存非空记录**(空格不写), **每 tick 全量快照**(非增量 → 任意帧自包含, 读取 `df[df.tick==t]` 直接得该 tick 世界, `groupby('strain')` 看谱系兴衰)。无复杂处理, 忠实记录。

游戏玩法: 红皇后式对称混战.
战场为 512 * 512 2D 空间, 每个格子里最多 K 个 "个体 (individual)", 分属若干 "株 (strain)".
数据结构: 每个格子一个 `{strain(序列): count}` 的 dict —— key 是序列(= 一个株), count 是这个株有多少个体.
(举例: 某格 K=100, 内容 {株3: 60, 株7: 40} = 2 个株、100 个个体。)
我们初始打造四种不同的 "始祖株 (strain)", 作为可选角色来用.

核心区分 (1.1 的命门, 已确认):

- 个体无目标: 个体不知道、也不追求任何东西, 只是在当前这桌对手里多留/少留后代. 它不会变"绝对强", 只会"比此刻这群对手强"(标准随群体漂).
- 模拟有胜负: "某谱系统一全场 / 全灭 / 跑够 T 步" 是停机条件 (= 胜负线), 不是个体的目标.
  → 停机条件是运行层的事, 不告诉系统"什么算赢"; φ 依然藏着, "谁会统一"正是 φ 在背后决定、我们要反推的东西.

> **原则: "怎么好玩怎么来" vs 私货红线 (已锁, 2026-06-12).** 设计取舍上不设过多"什么不碰什么不碰"的洁癖约定 —— **机制丰富度全放开**(折叠解锁跨类新通道、随机性家族、各种组合技都欢迎), 因为这些是**玩法层**, 胜负仍从机制涌现。**唯一不可越的红线 = 私货层(手写谁强)**: 绝不给某个具体配置 / 某个始祖株偷偷加"它就是更强"的隐藏系数 —— 那会把 φ 手写进 a、反推数据当场作废、selop 课题塌掉(项目存亡线, 非洁癖)。两层各管各、不冲突。**守门人判据(配套, 仍有效)**: 任何"读状态定参数"须同时满足 ①只定义"做什么/往哪做"不内嵌"怎么最优"的判断 ②信息来源只能是 自身序列 或 沙盒明文规则, 不引外部先验数据(真实生物统计/外部矩阵)。gain / 猎物清单 / 初始count 必须设计时一次性锁死, 不可观察输赢来迭代调。

## 序列设计 (序列 = 个体的完整行为程序, 直接交互式)

定调: 序列直接对序列交互 (路 A). 不映射成形状向量.
个体的一切行为都由它自己的序列直接算出来 (1.3: 表型=序列的固定函数).
游戏本质 = 个体在骨架上拼装基元, 拼出来的整条序列固定地决定其行为.

### 概念

> **统一注册表模型 (已锁, 地基级; 2026-06-20 大手术: 拆脚手架).** 所有基元 —— 不分类别 —— 本质都是一张注册表里的一行 `name → formula`. name = 字母数字组合 (单字母 = residue 粒度; 多字母 / 带数字 = motif 或复合); formula = 该基元效果的形式化. 类别只是 formula 的**作用域**不同, 全在同一张表里 —— 这天然实现了"插槽不限种类、随便混"(插槽里坐的不过是表中某一行).
>
> **★拆脚手架 (2026-06-20, 用户权威修正).** 原「类4 脚手架」= 纯调制器(算子的算子, 无自身功能)整套抽象**已作废**。新模型: **不存在"脚手架"这个类; 每个基元都是功能单元**(都有自己的 F/Z/P 产出 + base 产出)。**"backbone"现在只指: 一条株序列里定死不突变的那些位置**; 坐在 backbone 上的也是真功能基元, 只是被锁住不变(突变算子跳过 locked 位)。坐 backbone vs 坐插槽**唯一区别 = 突不突变**。基元 formula 统一降为**一阶函数** `formula(x, i) → 输出束`(x=完整序列, i=所在位), 没有任何基元去"改写邻居的输出"。
>
> **基元类别 (每类都有 residue / motif 两种粒度):**
> 1. **繁衍池 F** — formula 吐三样: f / 波及格集合 / p_leave.
> 2. **突变池 P** — formula = p(x) (rate) + spectrum 候选表 (变成谁); 是执行突变的算子.
> 3. **对抗池 Z** — formula = z (交换比) + 猎物清单.
> 4. **变体类 / 可变池 (已锁, 升级为"突变阶梯总池")** — 它是**突变能变出来的东西的总池**, 数量**最大(前三类的几倍都可以)**。构成三部分: ① 前三类里**偏弱/偏废**的版本 + ② 前三类里**偏强/极端**的版本 + ③ 变体**独有**的奇特基元(含纯惰性中性底料 N₀..N₄)。其中纯中性那批仍"无主动功能但有被动可见性差异"(折叠读它/猎物清单读它); 带功能那批一旦突变进插槽就发挥对应 F/Z/P 效果。生物 = 抗体序列空间里待探索的全部变体。
>
> **上位效应 epistasis = κ 同通道族自协同 (2026-06-20 替代脚手架调制).** 拆脚手架只拆掉 epistasis 三个源里的源①(脚手架乘性调制); 源②(P(x) 读全序列)、源③(折叠区内外)原生存活。源①重铸为"基元读同通道族邻居"(pull 上下文算自己, 非 push gain 给邻居 → 通道正交天然守住): `量_adj(i)=clamp(量_base(i)·(1+κ)^{n_same(i)})`, `n_same(i)=Σ_{j:0<|i−j|≤W_max} W(|i−j|)·[族(s_j)==族(s_i)]`, `W(d)=1/(1+d)`。**9 个按字母 gain → 1 个全局 κ**(私货面骤降, 过 G1/G11); **κ=0 即退化为位置无关无上位效应** → 平滑梯度要不要 = κ 一个常数的事, 数值轮设, 可先取 0 跑通本体。否决方案"每基元自带 gain 拧邻居"= 脚手架换名复活(每基元都带 gain 更脏 + 破通道正交)。

> **强弱分层 + 变体=突变阶梯 (已锁, 2026-06-12).** 五类的 `{name, formula}` 含**相对强弱档**, 现在就定死(这是 formula 的一部分, 不留到数值轮; 留到数值轮的只是 μ/gain 等全局标定常数)。分层切法: **可选区(始祖株能直接选装)= 中~弱档** / **变体区(只能突变长出来)= 极端档(最强的、最废的、最奇怪的)**。→ 突变 = 真正的进化阶梯(往变体池爬, 可能爬到神级也可能爬到废物), 不是平级换词。强弱用**机制锚定**: 如可选繁衍最多撒 1-2 格、变体里才有撒 4/8 格的强繁衍(概率相近则是明确强弱对比); 对抗可选 z 低、变体 z 高; 突变可选加码小、变体加码大。

> **字母表与字母族 (已锁: 甲 = 族即类池; 2026-06-20 拆脚手架: 5族→4族).** 沙盒序列由一张字母表组成, residue 基元 = 单字母 → 各类的 residue 合起来**就是**这张字母表. 字母按**类池本身分族**(繁衍族 / 突变族 / 对抗族 / 中性族), 无需额外定义。**拆脚手架后 S 脚手架族作废, 族降为 4 档 `N=0<F=1<P=2<Z=3`**(原 S=4 那行删除, affinity 拓扑相应重算)。用途三合一: ① **spectrum 距离**(守门人否决 BLOSUM 后的替代): 同族替换概率高、跨族低, 族划分明文写进设计 = 反推者可独立重建的公共知识、不引外部生物先验; ② **猎物清单**可直接瞄整族(如 Broad Sweep 瞄繁衍族); ③ 折叠匹配 + **κ 同通道族自协同**(上位效应靠"族"判同通道)。否决乙(正交化学族, 多一层定义且蹭真实生物先验嫌疑)、丙(扁平不分族, 丢突变方向性)。

- **骨架 (backbone) = 株序列里定死不突变的那些位 (2026-06-20 拆脚手架后定义).** 不再是"脚手架字母", 而是**被锁住不变的功能基元位**(locked) + 插槽 (X, mutable)。坐 backbone 位的是真功能基元(自带 F/Z/P + base 产出), 只是突变算子跳过它。
  例: 某始祖 = `[F4Nr1]-X-X-[BroadSweep]-X-[P_base]`, 方括号位 locked(定死功能基元), X 是 mutable 插槽.
  - **骨架长度可变 + 插槽预算守恒 (已锁: 乙).** 各始祖株骨架**长度不同、插槽数不同**, 但靠一条守恒律平衡: **长度短的株插槽多、长度长的株 locked 功能基元多**。即"插槽 vs locked 位"是一个此消彼长的预算 —— 多插槽的株骨架短(自带代谢薄、可变空间大), 长骨架的株插槽少(自带 locked 功能基元多→base 代谢厚、折叠花样多, 但能突变探索的位少)。→ **没有"插槽多=天生强"的私货**(平衡从结构守恒涌现)。否决甲(统一长度: 物种差异少一维; 采集侧用变长记录即可)。
  - **同基元插不同位/不同 backbone 效果不同 = 上位效应 epistasis**, 仍严格是序列的固定函数, 不破 1.3。**机制 = κ 同通道族自协同**(见上"概念"节): 基元读窗口内同通道族邻居数 `n_same`, 自己的通道量按 `(1+κ)^{n_same}` 缩放。这天然解决"不同始祖株不同扩张速度": 由 backbone-locked 基元的族组成 × 插槽内容算出。
  - **★物种身份 = backbone-locked 功能基元组合 (2026-06-20 转化, 替代脚手架通道特化).** 多锁 F 族=扩张型 / 多锁 Z 族=战士型 / 多锁 P 族=进化型。通道偏好被 κ 自协同强化(backbone 锁 F → 给周围 F 基元提供同族邻居 → 任何突变进来的 F 被协同放大 → "天生擅长 f"机械涌现, 不靠手设 gain)。**作废**: A=f/B=z/C=p 脚手架字母通道特化、"算子的算子"高阶 formula 签名。
  - **通道权衡 (转化, 替代抑制器负 gain).** 原靠抑制器 Ā/B̄/C̄ 负 gain 主动压一通道换另两条。新模型靠**有限位预算 + 通道正交**: backbone 某位锁了 F 基元该位就不能锁 Z; 插槽装了 F 就装不了 P。权衡是位置/预算的, 不是负增益的(原设计也已承认"混搭真实代价=有限插槽被瓜分")。**更干净, 无 gain 私货旋钮。**
  - **κ 空间作用域 (沿用乙 窗口衰减).** 同通道族邻居的协同强度随**序列距离**衰减 `W(d)=1/(1+d)`(近端权重高、远端弱, 窗口 W_max 内都算)。保住"位置敏感"(同基元插不同位效果不同)又不像最近邻那样差一格天翻地覆。与折叠形成两条独立影响通道: **序列邻近 (本条, κ 自协同) vs 折叠邻近 (下条, backbone 折叠表指定)**。
  - **折叠区 = backbone 必填结构字段 (已锁; 2026-06-20: Λ 字母作废, 折叠机制完整保留).** 每个 backbone 除位布局外, 附一张**折叠配对表**: 用**显式成员列表**(非 Λ 连续区间, 表达力更强能指定离散位)声明哪些**非相邻位**在空间上折叠到一起构成"折叠区". 例: backbone 规定 `{1,3,4,5}` 是一个折叠区. → 在折叠区内 X 处插一个对抗基元, 它就和区内远端基元拼组合技: **既发挥自己效果, 又因落在折叠区里拼出跨类新通道**. 同一基元插折叠区内 vs 区外效果完全不同 = 可玩性核心 + 对应真实 CDR loop(折叠后远端残基拼成结合面)。仍是序列固定函数(折叠由 backbone 定死), 不破 1.3。**Λ 折叠锚字母作废**(折叠端点信息全交显式成员列表, Λ 一直冗余, 不再占序列位)。
    - 这就是"序列邻近 vs 折叠邻近"两条独立影响通道里的**折叠邻近**通道: 序列上离得远但折叠配对表声明相邻 → 算组合技. 组合技的具体 formula(远端基元如何拼、加成多少)留数值轮.

- **基元 (primitive)**: 玩家往插槽里塞的带功能零件. 两种粒度都提供:
  - **残基 (residue)**: 单字母功能单位 (序列最小单位).
  - **基序 (motif)**: 一小段残基组成的功能单位.
  - 同类基元堆叠 = 函数复合. 例: 三个插槽全填 F4 → 繁衍能力 `f4(f4(f4()))`,
    且每个 f4 的具体算法还被它所在插槽的骨架上下文 (周围 A/B/C) 拧一下, 引入额外修正.

- **三种基元池 (按行为分通道)**: 繁衍池 / 突变池 / 对抗池.
  - 每个池里 residue 和 motif 两种粒度都有候选.
  - **插槽不限种类**: 任意插槽可填任意池的基元, 三池随便混. 纯突变池填满整条序列也允许.
  - 例 (繁衍池): `F4` = 每 5 步向上下左右 4 格各撒本格该株 count 的 50%;
    `F8` = 每 7 步向 8 邻格各撒 25%.

### 对抗基元怎么读对手 (已确认)

对抗池的基元**可以读对方所有类型的基元** (繁衍/突变/对抗都能读, 不限通道).
对抗是唯一"我的序列撞上你的序列"的地方, 也是 φ 频率依赖的诞生处.
- 对抗结算: 每个对抗基元自带一张**猎物清单 (target list)**, 猎取对手身上清单内的某个 motif 当标靶.
  你身上有我清单里的 motif → 我能咬你那一块. ("我能瞄谁"由我序列定, "你有没有这块"由你序列定 → 干净归 a.)

### 对抗结算公式 (Z 类基元, 已确认核心)

设甲株 a 个、乙株 b 个 (乙整株共享一条序列 → 要么 b 个全可捕、要么 0 个, strain=单序列自动保证).
z = z1() 由**甲的序列**算出 (受骨架调制), 是"交换比".

```
本次击杀 kills = min(b, a·z)
乙' = b − kills          # 乙的损失
甲' = a − kills/z        # 甲的损失 (攻击有代价!)
```

- 验证: b < a·z → 乙全灭、甲损 b/z; b ≥ a·z → 甲打光(→0)、乙损 a·z. 与原始两情况一致.
- **硬含义 (已确认)**: 对抗是**互相消耗/同归于尽式**, 攻击方每杀 z 个敌、自损 1 个.
  - z>1 = 一个攻击者换掉不止一个敌 (很能打); z<1 = 杀敌自损更多 (不划算). 对应"对抗性强=很能打".
- 这是"攻破"机制 (a 侧): 打完两株都可能掉 count, 总数 < K → 空出名额, 再由繁衍填 (→ φ).

#### ⚠ 待拍岔路 (对抗)

- (1) **双向同时 vs 分先后**: 推荐**同时结算** —— 用开打前快照 (a,b) 同时算两个方向再一起扣.
  否则结果依赖出手顺序 (先手清空对方、对方没还手), 不干净.
- (2) **多株 / 多对抗基元叠加**: 推荐 v0 —— 所有攻击对开打前快照结算,
  每个受害株被打总量封顶在其 count, 每个攻击株折损也封顶在自身 count. 超额按比例分摊的细节标 TODO.
- (3) 乙的"防御"先不做: z 纯由攻击方序列算, 乙只贡献"有没有被猎的 motif". 防御作为以后旋钮 (TODO).

### 格子内动力学 (已确认部分)

- **满格冻结**: 格子满 K 后状态固定, **除非被攻破** (对抗死人空出名额) 才会变.
- **空名额争夺 = 繁衍 vs 对抗的权衡** (两类命运由此涌现):
  - 繁衍强/对抗弱: 疯狂扩张, 但屡屡被攻破.
  - 繁衍弱/对抗强: 缓慢一点点推进, 但很能打.
- **基元写意图, φ 是意图撞上有限世界的结果** (守住 1.1/1.3 的边界):
  例 F4 "想撒 50%" 是 a 给的能力; 撞上 K 上限 + 邻格被抢 + 对抗死人后真站住几个 = φ.
  φ 不手写, 从"攻破→空名额→谁填上"的动力学里涌现, 事后再读出来.

### 繁衍机制 (已确认, F 类基元)

一个 F 类基元吐**三样**东西 (具体数值由 backbone 计算, 留最后; 这里定机制形状):

1. **`f = F() ∈ [0,1]`**: 繁衍比例. 源格某株 a 个 → 每个被波及的邻格各**收到 `a·f` 个** (每格各 af, 不是总量分摊; 沿用 F4 "向四格各撒" 的语义).
2. **波及格集合**: 往哪几个格、几个格, 由 F 自身机制定 (F4→上下左右4格, F8→8邻格…).
3. **`p_leave`**: 源格迁出概率, 也由 F 算出. 源格按此概率掉一部分 (繁衍有迁出代价, 不是纯增殖).

例: `F4` = 每 5 步向上下左右 4 格各撒 50%; `F8` = 每 7 步向 8 邻格各撒 25%. (各基元自带时钟周期.)

### 空名额争夺规则 (已确认: 甲 = 等比例、无优先级)

- **多源抢同一空格 → 按各自送达量等比例分**, 不加任何隐藏权重.
  (这是 ground truth 藏身处: 加权重 = 手写 φ = 数据作废. 必须保持等比例.)
- **priority 砍掉**: 不设繁衍优先级 (那会把"谁赢空位"写进 a, 缩小 φ 地盘). F 基元只吐上面三样.
- **装不下蒸发**: 邻格只剩 r 空位而涌来 a·f > r 时, 超出部分**直接蒸发**, **绝不挤掉活体**.
  → K 上限成为繁衍天花板; "繁衍撞上 K 墙"正是 a 转成 φ 的那一下.
- φ 保持纯涌现: 胜负全交给 **对抗杀人 + 繁衍速度差 + K 墙** 三者耦合, 我们一字不预设.

### 折叠强化机制 (已确认: 甲)

- **折叠 (folding)**: 骨架决定折叠方式, 把序列上离得远的 motif 在"空间"上拼到一起 → 折叠区 motif 产生**组合技**.
  - 这正是真实抗体原理 (CDR loop = 折叠后远端残基拼成结合面), 且折叠由骨架决定 → 仍是序列固定函数, 不破 1.3.
- **WHEN = 甲 (已拍)**: 强化**只在"填空名额"那一刻**起作用. 满格仍冻结、**绝不直接驱逐活体**;
  换血只通过"对抗/零头开出来的空位"慢慢发生, 强化株凭优势抢空位, 一点点把格子成分换成强的.
  - 为何甲够用: 比例制繁衍让格子长期"快满没满"(永远差零头), 强化时时刻刻有发挥余地, 不憋屈.
  - 强化自然并入 φ 的涌现, 不需要给满格格子单写第二套内部动力学.
- **WHAT (已锁, 已升级: 解锁跨类新通道 + 抗被猎).** 折叠区内远端基元拼成组合技, 形式 = **解锁一条单独基元做不到的新因果通道** + 组合系数 α (backbone 折叠配对表给, 数值留最后):
  - Z+P 折叠配对 → 新通道"攻击注入突变"(命中时以 α·p 概率对目标施加 spec 编辑);
  - Z+F → "猎后繁衍"(命中时胜者额外获 α·f 拷贝);
  - P+F → "分裂突变"(折叠撒出的拷贝突变率 ×(1+α));
  - Z+Z → 无新通道(z 已相加)。
  - 另: 抗被猎默认效果 = 区内 motif 对猎物清单可见性 = 1−β_fold(对手清单以 β 概率遮蔽区内 motif), 与组合技叠加存在。
  - **设计演进**: 原锁"折叠只加抗被猎(中立区)、不碰z/f"已**解除**(2026-06-12)。新原则"怎么好玩怎么来"(见地基④补充): 折叠允许碰 z/f/p、解锁新通道, 因为这是**机制丰富度**(胜负仍从机制涌现), 不是私货。
- 数值 (哪些位折到一起、加成多少) = backbone 计算的一部分, 留最后.

### step loop (已锁: 快照驱动三阶段, = R.py 骨架)

一个 tick 的精确时序 (伪代码级):

- **PHASE 0 取快照**: `world_snap = deepcopy(world)`. 本 tick 所有基元算"量"都读快照, 不读活世界 (保证同 tick 内先动的基元不影响后动的)。
- **PHASE 1 对抗** (基于快照): 每格每对 (甲攻乙, 且甲 Z 基元猎物清单命中乙) 且 `T%Z_period==phase_offset` → 用快照 (a,b) 算 kills, 暂存 delta, 批量写入 world。
- **PHASE 2 繁衍+突变** (量读快照, 空位读对抗后活世界): 每株 `T%F_period==offset` → `candidates=a_snap·f`; **突变嵌入** = 落地时 `candidates·p(x)` 个带突变, 分裂为 [s 的 `candidates·(1-p)` 个 + mutant(s) 的 `candidates·p` 个] 进 `pending_arrivals`; 迁出 `world[g][s] -= min(a_snap·p_leave, a_snap)` 写活世界。**PHASE 2 只生成 pending, 绝不提前落地。**
- **PHASE 3 K墙仲裁** (统一消化 pending): 每目标格 total_arriving vs available=K−占用; 装得下全落; 装不下按 `ratio=available/total` 等比例分, 超额蒸发不挤活体。**折叠强化在此算** (落地写入前一次性算, 不是每 tick 维护状态)。mutant(s) 是全新序列则往 dict 插新 key; 收敛突变(变出已有序列)按序列作 key 自动合并。

**节奏博弈张力裁决**: 繁衍池怕"对抗先跑→爆发型F总在对手被打空后才爆发"。架构师判定顾虑结论错: 爆发型F博弈空间是**跨tick时序选择**(何时撒)非同tick内顺序; 改"繁衍先于对抗"反而给F单tick隐性先手(先占位破坏等比例仲裁=污染ground truth)。**快照保护**: 繁衍读快照a, 对抗先不先跑只改空位多少(K墙松紧)不改F撒多少。→ **对抗→繁衍(含突变)→K墙 顺序锁定。**

**时钟调度 = 异相位 (已锁)**: 个体出生记 `birth_tick`, 触发条件 `(T-birth_tick)%period==0`。同相位会全场每N步同步爆发(功率谱强峰非生物现实); 异相位平滑、接近真实克隆动力学。多基元独立时钟无冲突。

**对抗三岔路 (已锁, 三条全确认)**: (1) 双向同时结算(快照双向算一起扣); (2) 多株叠加 v0: `raw_damage>b_snap` 时 `ratio=b_snap/raw` 按贡献比例分摊, 攻击方自损同比例缩, 双向不出负 count; (3) 防御暂不做("身上无攻击方猎物清单 motif=等效免疫"已是涌现防御, z 纯攻方算)。

**基元 table 须加 3 执行字段 (已锁)**: `period`(时钟周期) / `phase_type`(在哪 PHASE: Z→1, F→2, P→嵌入2) / `read_from`(量读快照, 空位读活世界)。

### 基元复合规则 (已锁: 同类复合 + 跨类正交 + 四步管线)

**同类堆叠公式 (逐通道):**

| 通道 | 堆叠公式 | 语义 |
|---|---|---|
| 繁衍 f | `1 − Π(1−fᵢ)` | 概率独立叠加, 任一F触发即有后代, 自然天花板<1(3个F4叠出0.875, 强但不无脑满) |
| 波及格 | 并集, 各自独立撒 | 同格收 `a·f₁+a·f₂` 受K墙限流; F4+F8混搭≠两个F4(覆盖广 vs 同格密) |
| p_leave | `1 − Π(1−p_leaveᵢ)` | 同f, 多高p_leave叠加=高度游牧 |
| 对抗 z | `Σzᵢ` (暂无cap) | 交换比=多把刀真实相加; 注意z无cap可能成无脑最优(见风险), soft cap留数值轮 |
| 猎物清单 | `∪ targetsᵢ` | 广谱(多猎物小z) vs 专一(少猎物大z)两条路 |
| 突变 p(x) | `max(μ, 1 − Π(1−pᵢ(x)))` | 独立叠加 + μ地板 |
| 突变谱 q | `Σpᵢ(x)·qᵢ,ₑ / Σpᵢ(x)` 归一化 | 按各P触发率加权(触发强的话语权高), 仍只读序列归a干净 |

**跨类协同 = 通道正交 (已锁)**: F/Z/P 各管各的, 互不相乘。一旦隐式交互(如"Z大的株自动f小")=隐藏φ破红线。混搭真实代价=有限插槽被瓜分(已内建)。**折叠组合技是唯一合法跨类耦合且显式声明**(见折叠节)。

**计算管线四步 (已锁顺序; 2026-06-20 Step B 重铸)**: Step A 每基元独立原始输出 → **Step B κ 同通道族自协同**(`量·(1+κ)^{n_same}`, n_same=窗口内同通道族邻居加权计数, 夹回合法域 —— 替代原 backbone 乘性调制) → **Step C 折叠组合技**(解锁新通道+α) → **Step D 同类复合**(上表公式)。裁决 **B 先于 D**(每基元是 κ 协同后修正零件, 用修正值参与复合, 保住"同基元插不同位不同效果"分辨率); 折叠C在调制后复合前。

### 突变机制 (已确认主体: 甲)

突变是这个沙盒里**唯一造新序列**的机制 (繁衍只拷贝、对抗只杀人), 所以"序列从哪来"全压在它身上. 它不是可选扰动旋钮, 是"演化"二字的发动机.

- **前置开关 = 甲 (已锁): 有基线突变流 μ.** 每个个体每次复制都有一股恒定小突变率 μ (全局常数, 不由序列算), 保证任何株都能慢慢漂出新序列 —— 没有"被冻死的脉". 突变基元在 μ 之上**加码 / 调向** (提率、偏向改某类位点).
  - 为何否决"纯基元触发": 那样不带突变基元的株序列永久冻结, "能不能进化"被写进始祖株设计 = 往 a 夹私货.

- **可变元 P (序列编辑算子, 守住地基③).** 突变由一类基元 P 承载. 一个 P 坐在某插槽里, 自带一个 `p(x)`, x = 个体的**完整序列**.
  - `p(x) → 概率`. 因为吃整条序列, 同一个 P 插不同位 → p 不同 = 天然上位效应, 仍是序列固定函数, 不破 1.3.
  - **p(x) 的地板 = μ**: 没插 P 的株 `p(x)=μ` (基线漂); 插了 P 抬到 μ 之上. 一个函数同时办了"基线"和"加码"两件事.

- **两层结构 (rate × spectrum, 嵌套不是竞争 —— 解"我有点晕").**
  1. **RATE = 变不变**: `p(x)` 决定个体突不突变. n_x 个里约 `n_x·p(x)` 个发生突变.
  2. **SPECTRUM = 变成啥**: **已决定要变的**那些个体, 变成什么型 —— 这才是"列表 + 每型一个概率"那层 (列表内条件概率和 = 1).
  - 先掷第一次(变不变, p(x)), 掷中的再掷第二次(变成谁, spectrum). = 生物学标准 突变率 × 突变谱.
  - **SPECTRUM 形状 = 乙 (已锁): 带概率候选列表.** 一个 P 自带一张表 `[(编辑1, q1(x)), ..., (编辑k, qk(x))]`, Σq=1. **硬规则: q_i(x) 只能由序列算出 (归一化计数等), 运行中不可调参, 只读序列内容、不读 count / 不读对手 / 不读步数 → 仍是 x 固定函数, 归 a 干净.** 同 P 插不同位 q 不同 = 上位效应从 spectrum 也出. 否决甲(单靶点表达不了 SHM 方向偏好且致基元爆炸)、否决丙(等概率丢失"某替换天生更常见"的真实突变谱结构).

- **WHEN = 甲 (已锁): 突变 = 复制错误, 耦合进繁衍 (对应抗体 SHM 主线).**
  繁衍撒出 `a·f` 个新拷贝时, 其中约 `(a·f)·p(x)` 个在**落地那一刻带上突变** (复制出错). 没繁衍就不突变.
  - 为何强推: 这正是体细胞超突变 SHM —— 突变发生在 B 细胞增殖/复制时, 是亲和力成熟核心, 和要验证的 selop / CR9114 (组合点突变 DMS) 同构. 合成数据与真实课题对得上.
  - 把"每 tick"微调成"每次复制": `n·p(x)` 里 n 数的是**新拷贝**, 不是存量.

- **效果不用单独设计 (③ 的红利).** 突变只负责**编辑序列**; 表型 (f / z / 猎物清单 / 折叠组合技) 怎么变是序列固定函数下游免费涌现. 严禁写"这次突变让 z+0.1"那种私货 (破③).

- **backbone 不动 (已锁; 2026-06-20 拆脚手架后泛化).** backbone = 株序列里 **locked 的位**(坐定死功能基元 + 定死惰性位), 突变算子跳过它们; 突变只在 **mutable 插槽 X 里换零件**. → 各始祖株是一个"物种", 在自己体制内探索插槽配置, 谱系不糊。原"脚手架字母定死"泛化为"locked 位定死"(脚手架类已废, locked 位坐的是真功能基元)。
