# 三层验证路径 — 方案级拆分

> 课题:从免疫 repertoire 的多时间点、多选择环境观测中,学习驱动亲和力成熟的
> **条件依赖选择算子 f(序列表征, 条件) → 选择强度** —— 一个反问题(从演化轨迹反推
> 驱动力的能量场),突变算子已知(SHM理论),选择算子未知(深度学习要学的)。
>
> 本文档:把"合成验证 → 真实多条件 → 独立fitness对照"三层路径拆成方案级可执行单元。
> 每层五字段:**目的 / 输入→输出 / 关键步骤 / 判据(gate) / 依赖**。
> 层间用 gate 串联:上一层不达标不准进下一层(防自娱自乐、防对仿真器过拟合)。

---

## 符号约定

| 符号 | 含义 | 典型值 |
|---|---|---|
| T | 时间点数 | 6 (5次免疫+基线) |
| C | 筛选条件数 | ~2,000 |
| S | 序列数(per campaign) | ~150,000 |
| L | VHH 序列长度 | ~120 |
| f | 选择算子(要学的能量场) | f(D_h, D_c) → ℝ |
| f* | 合成数据里私藏的真值选择算子 | 已知,用于评估 |
| M / σ | 突变算子(已知,各向异性结构化噪声) | S5F/EPAM |

核心动力学:`dX = ∇f(X,c)·dt + M·dW`(选择漂移 + 已知突变扩散)

---

## Layer 1:白盒进化仿真器 + 可辨识性检验

**目的(为什么存在,不做会怎样)**

造一个**标准答案已知的试车台**。真实数据里没人知道 f 的真值,无法判断算法反推得对不对。
Layer 1 自己藏好一个 f*,于是能精确度量"算法推出的 f 离真值多远"。
- 不做的后果:`M_0 → M_*`(方法迭代)失去可信反馈信号,方法演化瞎走。
- 跳过直接上真实数据 = 没体检就上手术台,连"这题有没有唯一解"都不知道。

**输入 → 输出**

```
输入(你提供的设定):
  ├─ f* 真值选择算子    : (D_h,) × (D_c,) → ℝ   ← 手搓/从DMS拟合,藏好
  ├─ 突变算子 M (σ)     : 已知 SHM 模型(S5F/EPAM),固定不动
  ├─ 群体动力学规则     : Wright-Fisher / 分支过程,已知公式
  └─ 初始 naive 库      : IGoR 生成的起始序列群体

输出:
  ├─ 频率轨迹 X          : (T, C, S) 计数张量,模拟出来 ── 公开给算法
  ├─ 私藏真值 f*         : 评估时才拿出来对比 ── 算法看不到
  └─ (可选)私藏真值谱系 : 检验潜变量推断 ── 算法看不到
```

**关键步骤**

1. **搓 f\* 族**(不是单个):平滑 / 崎岖(带 epistasis) / 强弱条件依赖 —— 一整族,逼算法学通用反推能力而非记住特例
2. **串模拟管线**:IGoR(naive库) → netam/EPAM(套已知σ突变) → DIMSIM/SLiM(在 f* 驱动下演化) → 多时间点频率轨迹
3. **加观测噪声**:模拟测序深度、采样偏差,让假数据"脏"得像真数据
4. **跑算法反推**:X 喂给待验证 method,得到估计 f̂
5. **度量恢复精度**:f̂ 与私藏 f* 的距离(相关性 / ranking一致 / 能量场形状)

**判据(gate)**

- **必过**:干净数据 + 单一简单 f* 上,f̂ 高精度恢复 f*(否则算法有bug或问题不可辨识 → 停)
- **要测**:随 f* 复杂度↑、噪声↑、时间点↓,恢复精度怎么衰减 → 画**有效边界**
- **诚实声明**:连理想情况都推不准 = 问题在当前数据形态下 ill-posed,回去改问题定义而非改算法

**依赖**

| 角色 | 工具 |
|---|---|
| naive 库生成 | IGoR / OLGA / immuneSIM |
| 已知突变算子 σ | netam / EPAM / DASM(S5F) |
| 选择+群体动力学模拟 | DIMSIM、bcr-phylo/gctree、SLiM/fwdpy11 |
| f* 真实结构来源(防乱搓) | Koenig VHH DMS / CR9114 拟合 |

**方案级判断(待定)**:f* 用纯手搓还是从真实 DMS 拟合 —— 命门。
倾向**两者都要**:手搓用于可控的可辨识性扫描,DMS拟合用于保证"有意义"。

## Layer 2:真实多条件展示筛选 → 验证核心创新

**目的(为什么存在,不做会怎样)**

验证方法真正值钱的那一点 —— f(序列, 条件) 里的"**条件依赖**"在真实数据上成不成立。
Layer 1 证明"算法在理想仿真里能反推 f",但仿真是自己造的,算法可能**专门钻仿真器空子**
(对仿真器过拟合)。Layer 2 第一次用真实生物数据,把这种作弊筛掉。
- 不做的后果:只知道方法"在假数据上行",不知面对真实噪声/选择压力还行不行,
  论文核心 claim(条件依赖+因果识别)立不住。

**关键替换(精髓)**:开源**没有**公司那种"体内6点免疫时序",但有大量
**phage display(噬菌体展示,试管里筛抗体的离体技术)** 数据,天然是"多轮 × 多条件"。
验证锚点从"体内免疫时序"换成"离体展示筛选的轮次×条件动力学" —— 不是退而求其次,
而是新颖性本就在**轴 c(多条件)**,不在轴 T(时序)。

**输入 → 输出**

```
输入:
  ├─ 真实 counts 张量    : (轮次, 条件, 序列) 每条抗体每轮每条件出现次数
  │                        └ 轴T从"免疫时间点"变成"panning轮次"
  ├─ 序列集合 + 计算先验 : ESM/IgBERT embedding、IgFold结构、IgBLAST注释...
  └─ 条件元信息          : 每个条件是什么(靶标/浓度/竞争消减/洗脱严格度)

输出:
  ├─ 学到的 f̂(seq, cond) : 真实数据上的条件依赖选择算子
  └─ 条件依赖性证据      : "换条件→f随之变"的定量验证
```

**关键步骤**

1. **取数+对齐**:下载 Hanke / phage-seq,跑 IgBLAST+ANARCI 注释、PLM embedding,counts 与序列先验对齐
2. **诊断条件有效秩**:对 条件×序列 矩阵做 SVD,看 r_eff;若太低(<10-20),"多条件打破非可辨识性" claim 会塌,必须先确认
3. **学 f̂(seq, cond)**:把 Layer 1 验证过的 method 套到真实数据上
4. **验证条件依赖**:固定序列扫条件看 f̂ 是否实质变化;做"打乱条件标签"对照(打乱后效果不掉=条件没用上,假阳性)
5. **剥离突变假信号**:确认频率上升被正确归因到选择而非 SHM hotspot 多产变体

**判据(gate)**

- **必过**:f̂ 在留出轮次上能预测富集/消失方向(比"忽略条件" baseline 显著好)
- **核心**:条件标签打乱后性能显著下降 —— 证明条件依赖是真学到、非装饰
- **诚实声明**:若条件有效秩太低或打乱对照无差异,说明数据撑不起"条件依赖",需换数据或降 claim

**依赖**

| 角色 | 资源 |
|---|---|
| 多条件 panning 数据 | **Hanke 2022**(SRA PRJNA814713,2轮×3条件,counts全存,即用)、**phage-seq**(Zenodo,多轮×多抗原,带feature table) |
| 序列注释/编号 | IgBLAST / ANARCI / Change-O |
| 序列先验 | ESM-2 / IgBERT / AbLang、IgFold/NanoBodyBuilder2 |
| (备选)亲和力对照 | open-alphaseq(A-Alpha Bio,scFv+VHH,170万Kd测量) |

**方案级提醒**:Hanke 只有 2 轮,时序短;phage-seq 轮次多些。Layer 2 重点放在
"**条件**"轴而非"轮次"轴 —— 时序丰富度留给 Layer 3 之后的公司数据。

## Layer 3:独立 fitness 对照 → 跳出"自验"闭环

**目的(为什么存在,不做会怎样)**

彻底打破"自娱自乐"的一层。Layer 1 标准答案是手搓的,Layer 2 仍从同一批 panning 数据自洽。
Layer 3 用的标准答案是**别人实验室真刀真枪测出、和训练数据完全独立的**
Kd(解离常数,结合强度金标准,越小结合越强)。
- 验证命题:**学出来的 f 是否真的对应真实结合能力**,而非数学自洽的幻觉。
- 不做的后果:无法排除"f 只是拟合了统计规律、跟真实生物学无关" —— 审稿人和公司的第一问。

**为什么 CR9114 完美对口**:它本身就是"一条抗体骨架 × 多种流感抗原"的完整 Kd 地图,
结构上 = "同一抗体在不同条件下的适应度" = **你的 f(序列, 条件) 的真实版本**,
天然是 condition-dependent landscape,直接对标核心 claim。

**输入 → 输出**

```
输入:
  ├─ 学到的 f̂(seq, cond)  : 来自 Layer 1/2 验证过的 method
  └─ 独立实测 fitness 库   : CR9114 的 (序列变体, 抗原) → Kd 实测值
                            └ 关键:method 训练时从没见过

输出:
  ├─ 相关性报告           : f̂ 预测 vs 实测 Kd 一致性(Spearman/Pearson)
  ├─ 条件依赖验证         : f̂ 能否复现"同序列不同抗原下 Kd 不同"
  └─ 毕业判定             : 过/不过 —— 决定 method 能否进入真实部署
```

**关键步骤**

1. **取实测库**:CR9114/CR6261(Phillips 2021, 组合完备库 2¹⁶)、Bloom RBD escape 作补充
2. **零泄漏检查**:确认序列/抗原**没参与过** Layer 1/2 训练 —— 泄漏则验证作废
3. **跑预测**:用 f̂ 对 CR9114 的变体×抗原打分
4. **对相关性**:f̂ 排序 vs 实测 Kd 排序一致性
5. **验条件依赖真实性**:重点看"同变体换抗原,f̂ 是否跟 Kd 一起变" —— 从 Layer 2 的"自洽条件依赖"升级成"对照真值的条件依赖"

**判据(gate = 毕业线)**

- **必过**:f̂ 与独立实测 Kd 显著正相关(且明显高于序列相似度等朴素 baseline)
- **核心**:条件依赖在真值上复现 —— 同序列跨抗原的 f̂ 变化方向与实测 Kd 一致
- **诚实声明**:仿真器分数高但 CR9114 上不行 = 对仿真器过拟合,**直接淘汰回炉**,这正是 Layer 3 作为最终裁判的意义

**依赖**

| 角色 | 资源 |
|---|---|
| condition-dependent 金标准 ⭐ | **CR9114/CR6261**(Phillips 2021, eLife 71393, CC-BY, 组合完备库) |
| escape 对照 | Bloom RBD escape(jbloomlab GitHub) |
| 血清逃逸(多"条件") | Shlesinger/Reddy 2025 DML |
| 通用 fitness 基准 | ProteinGym(含抗体 assay 子集) |
| VHH 专属 DMS | Koenig PNAS 2017 |

## 三层如何串成 gate 链(收口)

```
Layer 1 (合成,可辨识性体检)
   │  gate: 理想情况能反推 f* 吗?推不出→问题ill-posed,改定义
   ▼
Layer 2 (真实多条件,核心创新)
   │  gate: 条件依赖真的被学到吗?打乱对照掉性能吗?
   ▼
Layer 3 (独立fitness,最终裁判)
   │  gate: f 对应真实结合力吗?跨抗原条件依赖在真值上复现吗?
   ▼
毕业的 M_*  →  真实部署 / 公司数据 / Foundation Model 叙事
```

**全程不依赖公司数据**就能跑通方法验证和发表。公司数据的不可替代价值留给入职后:
① 真正的体内 6 点免疫时序(轴 T 空白);② 多 campaign 跨域泛化。

## 数据轴覆盖速查(开源强弱)

| 轴 | 强度 | 说明 | 主力资源 |
|---|---|---|---|
| T 时序 | ▓▓░░░ 弱 | 目标域空白,只有双峰驼替代 | Li Yi 2024(最完整) |
| c 多条件 | ▓▓▓▓░ 强 | 意外够用,即用 | Hanke PRJNA814713、phage-seq |
| V 验证 | ▓▓▓▓▓ 很强 | CR9114 量身定做 | CR9114、Bloom escape |
| S 合成 | ▓▓▓▓▓ 很强 | 工具链全套 | IGoR+netam/EPAM+DIMSIM+SLiM |

**核心认知**:开源数据的"形状"逼着把验证锚点从"体内免疫时序"换成"离体展示筛选" ——
正好扬长避短,因为新颖性在轴 c 不在轴 T。
