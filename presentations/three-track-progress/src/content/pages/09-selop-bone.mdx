# 三、方案长什么样、卡在哪

## 1. 骨架与六决议：假世界怎么搭

这条线的地基刻意不堆补丁，也不吹「一个原理解释一切」。压力测试后留下的结构是四块：

$$\underbrace{1\ \text{动力学原理}}_{\text{Price 协方差 + 噪声-}N_e\text{ 标度}} + \underbrace{2\ \text{独立支柱}}_{\sigma\ /\ \text{可辨识性}} + \underbrace{1\ \text{诚实残余}\ R_{\text{irr}}}_{a_{\text{invivo}}} + \underbrace{1\ \text{适用包络}}_{}$$

- **1 个动力学原理** —— 两条「不管模型怎么搭都不能违反」的硬约束：Price 协方差（适应度变化必须等于性状的方差，演化的恒等式）和噪声-$N_e$ 标度（种群越小，随机漂变越压过选择）。
- **2 根独立支柱** —— $\sigma$（存活投影）和可辨识性（什么能从数据里分出来）。两者各管各的、不互相派生，所以能**分别被证伪**，不会一塌全塌。
- **1 个诚实残余 $R_{\text{irr}}$**（irreducible residual，不可约残余）—— 不是「暂时没做」，而是**数据本身原理上就分不出来的方向**：再多数据也补不上。$a_{\text{invivo}}$ 落在这里，湿实验才能测，计算路线老实承认碰不了。
- **1 个适用包络** —— 明说这套结论在什么范围内才算数，包络内外分开报告。

> 曾经有个更漂亮的主张「一个骨架统一三层」，被降级了——漂亮 ≠ 正确，把三个独立真命题强行捆成一个，反而更脆。

在这个骨架上，引擎按六条决议落地，先立三条底线（防止设计滑向自欺）：**不许怎么简单怎么来 / 不许交好看但没用的 MVP / 必须建在真实开源数据上**。核心是 **A/B 两层分家**：

| 层 | 职责 | 一句话 |
|---|---|---|
| **A 层** | 逼真环境 | 用多个真实开源数据集拼出一个像样的生发中心，无私货、过保真度闸门 |
| **B 层** | 三真值算子 | 驱动演化的那只手，每步主动决定谁选中、谁死 |

为什么必须分家：world model 学得到「像不像真」，但学不到「真 $\varphi$ 是多少」——逼真和真值得用两套来源分别供给。三个算子各有物理依据，且在生发中心里**天然彼此独立**（信号不混），这正是选生发中心的原因：

| 算子 | 物理机制 | 来源 |
|------|---------|------|
| $a_{\text{invivo}}$ | catch-bond 亲和门控（受力越大抓得越牢） | Bell-Evans |
| $\sigma$ | T 细胞帮助瓶颈竞争（资源稀缺，赢家通吃） | Victora-Nussenzweig |
| $\varphi$ | 频率依赖克隆竞争 + 克隆干涉（看对手是谁） | Gerrish-Lenski |

**A/B 严格隔离 = 防自欺的命根子。** A 层绝不能把 $\varphi$ 的信息偷偷烘焙进环境，否则等于自己出题自己抄答案。

⚠️ 一个具体陷阱：A 层若用 Diffusion 生成数据，它的 score 函数和 B 层算子在数学形式上同源，很容易被顺手拿去当真值——这一步隔离就悄悄破了。**规则：A 层只许生成可观测统计量，绝不当 B 层真值源。** 架构上只用 Transformer / Mamba / Diffusion，图作 attention bias，不另起 GNN / CNN / RNN。

## 2. A 层怎么搭：多视图对角整合

A 层要「逼真」，难在没有任何单一数据集能覆盖整个问题。手上是五类性质完全不同的数据——DMS 测亲和、repertoire 测克隆、结构测构象，各只覆盖 2~3 个维度，而且往往不是同一批样本（没有配对），硬拼不到一起。

这个形状在多组学领域有个标准技术对应，叫**多视图对角整合**（multi-view diagonal integration）：当几个数据集各测了同一类对象的不同侧面、既没有共享样本也没有共享特征时，把它们对齐到同一个空间里，拼出每个对象的完整画像。我们的难题（从几个低维切片拼出高维全貌）正是它的标准应用形状，所以直接借这套思路，不用另造轮子。

落到四步：

**第一步，各自 embedding。** 每类数据用各自成熟的预训练模型嵌成同一种数学对象——向量：

| 数据 | 覆盖维度 | 嵌入模型 |
|------|---------|---------|
| CR9114/CR6261 DMS | 序列 × 平衡亲和力 | 突变效应模型 |
| 抗体 repertoire | 序列 × CDR 语义 | AntiBERTy / IgLM（抗体专用语言模型） |
| 通用蛋白序列 | 进化耦合、结构倾向 | ESM-2 / ESM-C（蛋白语言模型） |
| 结构 | 序列 → 3D 构象 | AF 类结构嵌入 |

**第二步，对齐到共享隐空间。** 各模型嵌出的向量坐标系不同，不天然对齐。赌的是：这些模型都在相似的生物分子空间上训过，语义相近的输入会落到相近位置，于是给出一个经验性的共享坐标系。为什么非要先对齐——「从几个低维切片拼高维全貌」数学上**欠定**（解不唯一），必须注入一个结构假设；这里用**大规模预训练表征隐含的先验**当这个假设（公开、可复现、可被质疑），比私下拍脑袋一个低维共享因子更可辩护。

**第三步，生成全貌、补缺口维度。** 有些维度（暗区/光区、T-help、抗原浓度）没有任何数据集覆盖，要从共享隐空间一并生成。这步仍欠定，所以产出的不是一个全貌，而是**一族**都合理的全貌。

**第四步，过保真度闸门。** 每个全貌体检：引擎跑出的可观测统计（克隆大小分布、突变谱、多样性衰减、谱系树形状）像不像真实数据，不像就作废重来——硬门槛，不是软评分。

搭好的共享隐空间，是当 **A 层环境（舞台）**用的——注意它**不是**那个「输入亲和力、输出繁衍优势」的映射。把 $a$ 翻译成繁衍优势的是 **B 层的 $\varphi$ 那只手**，环境只负责给这只手提供逼真的上下文。两层一刻都不能混：

- **环境（A 层）的输入输出** = 当前世界状态 → 下一刻世界状态。它只把舞台布逼真。
- **$\varphi$（B 层）读什么、吐什么：**
  - 输入 = 某个虚拟 B 细胞此刻的**局部上下文 context**：场上有哪些克隆、各自频率多少、抗原浓度、T 细胞帮助还剩多少。**序列就在输入里**——每个 agent 随身带一条序列，它既是这个 agent 的身份，也定了它的亲和力 $a$。所以 $\varphi$ 读的是「这条序列的 $a$ + 它的 context」。
  - 输出 = 这条序列的克隆在这一 tick 的**增减**（净繁衍还是净死）。

这正是本项目的核心 reframe：$\varphi$ **不是** $a$ 的单值函数（喂一个亲和力吐一个固定的适应度），而是

$$\varphi:\ (\text{序列的 } a,\ \text{局部上下文}) \longrightarrow \text{这一 tick 的增减}$$

同一条序列，对手多又强时可能净减，资源足对手弱时净增——繁衍优势随上下文变号，这就是「频率依赖 + 克隆干涉」。

引擎正向跑下来，旁路记下每条序列每一 tick 的真 $\varphi$（答案钥匙），对外只暴露**可观测的「序列 → 命运」时序**（每条序列的数量轨迹）。恢复模型只看这个可观测时序去反推 $\varphi$，再跟旁路真值对账——这就是整条线的验证支点。

共享隐空间天生的欠定性还有第二个用途：当**检验工具**。它产出的是「一族都合理的全貌」而非一个，核心预测 P1 就在**整族**上都跑——全族成立 = 真信号，只在某一个拼接下成立 = 假象拒掉。本来是缺陷，直接编织进验证逻辑。

## 3. 最硬骨头：卡在两个未解难题

**问题 A — 共享隐空间还原机制。** 就是上面第二、三步「怎么对齐、怎么生成」具体怎么做，且不泄漏 $\varphi$。四个风险：

| 风险 | 威胁 | 为何要命 |
|------|------|---------|
| 隐空间不天然对齐 | 「嵌到相近位置」是猜想不是保证 | 硬对齐丢非线性结构，A 层失真 |
| 预训练偏置渗进来 | 模型偏向天然蛋白，对远缘突变体失真 | 违反无私货，须进闸门体检 |
| 缺口维度怎么生成 | 没数据覆盖的维度凭什么补 | 补法影响统计结构 |
| **表征泄漏 $\varphi$** | 某模型若隐含适应度信息 | **等于把答案泄漏进环境，A/B 隔离破功，验证从根上失效** |

第四个最致命——一旦触发，整套验证逻辑从根部塌掉，不是局部能补的。所以每个候选表征都要先审一遍「它知不知道 $\varphi$」。

**问题 C — tick 事件顺序。** 同步（所有算子同一拍并行）还是异步（每个 agent 独立触发）？这属于真值定义层面，会改变涌现动力学，必须在写代码前锁死。当前倾向异步。

关键开源数据其实**都已就位**（CR9114/CR6261 DMS、DeWitt GC-replay、OAS、S5F、OLGA 全本地或可达），但**数据齐 ≠ 可以动手**：问题 A 的风险若等代码写完才暴露，返工成本翻倍。设计未收口，不进实现。
