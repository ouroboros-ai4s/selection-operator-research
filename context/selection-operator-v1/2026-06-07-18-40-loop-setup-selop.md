## Checkpoint: Stage 1 LOOP SETUP 完成(v0 装载 + 部件审计 + state ledger 就位)

### Objective

Stage 1 是 loop 的"装弹"阶段,一次性、不迭代。目标是把混沌的 v0(老三层方案 + 候选脊柱,两份文档约 20 个部件散落)固化成 Stage 2 对抗循环可以逐一审判的结构化输入:(1) 每个部件的承重假设(它若承重是因为撑住了什么)与工程成本;(2) R1-R5 种子攻击队列,每项挂好 Stage 0 弹药与 judge 判据;(3) 一份 state ledger,作为 loop 的起点账本,记录版本、队列、三不变量、部件账、脊柱猜测。Stage 1 不下任何"砍/留"的最终决策——那是 Stage 2 loop 的事;Stage 1 只把审判台搭好、把被告(部件)逐一传唤就位、把控方证据(Stage 0 弹药)摆上。

为什么这一步必须独立成 stage 而非并进 Stage 2:对抗循环每轮要"增量攻击只打上轮动过的部件",这要求有一个稳定的、全集已知的部件清单作为基线。若没有 Stage 1 先把 20 个部件全部登记在册,Stage 2 的 critic 会每轮重新发现部件、judge 无法判断"这个部件上轮审过没有"、saturation 也无法判断"是否还有没审的部件"。state ledger 就是这个稳定基线。

### Key Findings(部件审计的深化承重逻辑)

这一节是 Stage 1 的核心交付,把 ledger 表里每个部件的"承重假设"展开成 Stage 2 审判时 defender/critic 可直接引用的论证。按预判命运分组。

**🔴 预判砍组(内部真值生产线,I1 判其零证据力)**:

- **P01 手搓 f* 族**:承重假设是"需要已知真值 f* 来度量恢复精度"。这个假设在老方案里是 Layer 1 的地基,但 Stage 0 附录 I 的理论收束已证明它是循环论证的核心:f* 与真实选择算子 f_真 无任何约束关系,"f̂≈f*"不蕴含"f̂≈f_真"。它生产的是内部真值,对 I1 零证据力。Stage 2 R1 会首先处决它。唯一可能的抢救方向:若 f* 不是凭空手搓而是从 CR9114 DMS 拟合而来(老方案 Layer 1 "方案级判断"提过"两者都要"),那它就不是纯内部真值——但那样它就等价于 P15(CR9114 数据)本身,不需要单独的"手搓"步骤。所以无论哪种解读,"手搓"这个动作都冗余。

- **P02 IGoR naive 库 / P04 DIMSIM/SLiM 模拟 / P05 多时间点轨迹生成**:三者是逼真仿真器流水线的三段。共同承重假设是"需要逼真的合成时序轨迹来训练/验证方法"。Stage 0 判定:时序轴 T 是部署野心(留公司数据),非方法成立必需;且仿真器生产的轨迹真值仍是内部真值(我们自己定的 f* 驱动的),对 I1 零证据力。R2 会攻击其必要性,R5 会确认时序留公司。抢救方向(2d resurrection):仿真器是否提供 CR9114 没有的不可替代东西?有两个候选——(a) 轴 T 时序(但判为部署野心);(b) "从轨迹反推 f"的可辨识性测试场(但这个最小功能可由 P07 玩具 SDE 以远低成本提供)。所以逼真仿真器整条流水线大概率被砍,其唯一承重的最小核(可辨识性测试)由 P07 接管。

- **P20 三层 gate 链外壳**:承重假设是"需要分层 gate 防过拟合/防自娱自乐"。这里要做精细切分:gate 的三个**意图**(L1可辨识性体检、L2条件依赖、L3独立对照)各自承重——它们正好对应 I2/I3/I1 三不变量。但"L1→L2→L3 三层串联"的**结构外壳**是冗余包装:把三个意图直接实现为 I1/I2/I3 三个不变量检验即可,不需要"层"的仪式感和层间 gate 串联的流程开销。用户嫌"三层 gate 像走流程"正是嫌这个外壳。所以 P20 拆成两半:意图🟢保留(化入三不变量),外壳🔴砍。

**♻ 复活候选组(砍后大概率被抢救回的承重核)**:

- **P06 可辨识性度量 → 改造**:原始形态"f̂ vs 私藏 f*"是循环的(依赖 P01 手搓 f*),随 P01 一起被砍。但"度量可辨识性"这个意图承重(I2)。改造方向:不比对私藏 f*,改为玩具 SDE sanity check（见 P07）。所以 P06 不是简单砍除,是"形态被砍、功能转移到 P07"。

- **P07 玩具 SDE 可辨识性 sanity check**:承重假设是"反推 f 在已知 σ 下的数学可行性需要一个最小验证"。这是 Stage 2 R3 最可能复活的部件。理由:I2=AMBER,Stage 0 指出"从轨迹反推 f"在本数据形态下的可行性无任何定理保证(gating ①②);若方案完全不做任何可辨识性验证,直接在 CR9114 上拟合 f 然后声称"学到了选择算子",会被 critic 一击致命——"你怎么知道这个反问题有唯一解?你的 ∇f 可能是无数个能解释数据的力场之一"。P07 用最小设定(已知简单 f + 已知 σ → 生成轨迹 → 反推,看能否恢复)回答"反推动作本身在数学上 work"。关键区分:P07 是玩具级 sanity check(成本低-中),绝不是 P01+P02+P04 那套逼真仿真器+手搓 f* 族(成本高)。loop 的"安全简化"机制在这里体现得最典型:砍掉重工程(P01/P02/P04),复活其中唯一承重的最小核(P07)。necessity-sufficiency 判 P07 是不是 load-bearing wall——预判是(没它 I2 悬空),但 Stage 2 的 factor-removal 会实测"砍掉 P07 后 I2 是否塌"来确认。

**🟢 预判承重组(外部真值锚定 + 方法落地,loop 应保留)**:

- **P11 学 f̂(seq,cond) on 真实数据**:承重假设"方法须在真实数据上落地"。这是方法本身,不是脚手架。承重无疑。

- **P12 固定序列扫条件 / P13 打乱条件标签对照**:P12 验"同序列换条件 f 是否实质变"(条件依赖的正向证据),P13 验"打乱条件标签性能是否掉"(条件依赖的反向排除假阳性)。P13 是 I3 最干净的检验,承重假设"需排除 f 退化成 f(seq) 的假阳性"。CR9114 的 nested 跨抗原结构保证:真模型打乱抗原标签后必掉(因为 nested epistasis 跨抗原不同,标签一乱结构就废)。这两个是 I3 的落地,R4 主场,承重。

- **P14 剥离突变假信号**:承重假设"需把频率上升正确归因到选择而非 SHM hotspot 多产变体"。这是方法正确性的关键——若不剥离,模型会把"突变热点导致某变体多产"误判为"选择青睐该变体",f 被突变偏好污染。σ 已知(S5F/EPAM)正好用于此:已知突变率 → 可扣除突变贡献 → 残差归于选择。承重。

- **P15 CR9114/CR6261 取实测库**:从老方案的"第三层对照"升级为 Stage 2 脊柱主干。三重身份:Spike0 的训练数据、O₁(留出突变组合 Kd)与 O₂(留出抗原 Kd)的外部真值源、I3 的 nested 真值场。承重假设三条:(a) 数据可访问✅、(b) 条件依赖真值存在✅、(c) 检测限 floor 须 censored 建模⚠️。第三条是 Stage 0 新发现:若不建模 floor,弱结合区 f 被假数据带偏,连累 P17/P18 用 Kd 做独立验证时弱结合区对照失真。承重核心。

- **P16 零泄漏检查**:承重假设"独立验证须确认序列/抗原未参与训练"。这是 I1 held-out gate 能成立的前提——若验证用的 O 其实泄漏进了训练集,O 就不再是"训练未见"的外部真值,非循环性破功。低成本但承重。

- **P17 f̂ 对变体×抗原打分 + 相关性对照 / P18 条件依赖真实性**:P17 是 I1 非循环 gate 的落地(用独立实测 Kd 检验 f̂,对照朴素 baseline);P18 验"同变体跨抗原 f̂ 变化方向与实测 Kd 一致"(I1+I3 交汇——既是独立真值对照又是条件依赖在真值上的复现)。两者承重,是 vN 的 I1 关主体。

### 附录 S1-M:T-core 张力两分支的 vN 形态详解(场景规划)

附录 S1-I 指出 loop 最硬张力是"Spike0静态拟合 vs SDE动力学反问题",其消解取决于 gating①(连续化是否梯度流)。这里把两个分支各自的定稿 vN 形态写清,Stage 2 R3 实测 gating① 后可直接落到对应分支,Stage 3 据此终检。

**分支 A:gating① 成立(连续化近似梯度流扩散)→ 两极统一,极简 vN**

若表征空间动力学确为梯度流 Itô 扩散,则静态势函数 f 与动力学选择算子统一:∇f 就是 drift,f 就是势。此时:
- Spike0 在 CR9114 上拟合的静态 f(seq,cond) **就是**选择算子的势函数,取梯度即得驱动力 ∇f。
- 不需要从轨迹反推——静态地形拟合直接给出 f,SDE 只是 f 的"使用方式"(把 f 代入 dX=∇f·dt+σdW 即可前向模拟演化,但学 f 不需要轨迹)。
- vN 形态:Spike0(CR9114 censored 回归学 f)+ gating①验证(证明取梯度合法)+ I3打乱对照 + I1 held-out gate。玩具SDE(P07)降级为"前向一致性 sanity check"(用学到的 f 前向模拟,看是否产生合理演化),非可辨识性反推。
- 工程量:最小。几乎就是用户期待的"简洁明了"。
- 风险:gating① 成立是强假设,需 R3 实测站得住。

**分支 B:gating① 不成立(连续化非梯度流,或动力学含静态拟合捕捉不到的成分)→ 两极分裂,分层 vN**

若静态地形拟合 ≠ 动力学选择算子(例如:真实选择有路径依赖/历史依赖,静态 Kd 地形丢失了"演化过程中的"选择信息;或表征空间 drift 有旋非梯度),则:
- Spike0 静态 f 只是"终点适应度地形",不等于"驱动演化的选择算子"。
- 必须保留独立的动力学反推步骤:从(玩具SDE的或真实phage panning的)轨迹/分布演化反推 drift。P07 从"sanity check"升格为"承重的可辨识性反推核"。
- vN 形态:Spike0(学静态地形,作为 f 的一个投影/约束)+ 玩具SDE可辨识性反推(学动力学 ∇f)+ gating②椭圆性处理 + I3打乱 + I1 held-out gate + 可能需要真实轨迹数据(Hanke panning 轮次,P09升格)。
- 工程量:中等(比分支A多一个动力学反推 + 可能需轨迹数据),但仍远少于老方案(无逼真仿真器、无手搓f*)。
- 这个分支里,"选择算子是动力学量"的 North Star 内核被严格保留,代价是方案没那么极简。

**分支判定的关键实测(R3 必做)**:附录A的两项检验——(1)若有轨迹(玩具SDE生成或Hanke真实),检验表征空间增量是否近高斯 + drift是否近无旋;(2)更根本地,检验"静态拟合的f取梯度"与"从轨迹反推的drift"是否一致(若一致→分支A;若系统性偏离→分支B)。这个一致性检验本身就需要 P07 玩具SDE 作为载体(已知f+已知σ生成轨迹,看两种途径得到的∇f是否吻合)——这是 P07 在两个分支里都至少需要存在的理由,也是它大概率被 resurrection 救活的根据。
