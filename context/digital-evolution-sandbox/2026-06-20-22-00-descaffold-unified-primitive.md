## 本轮起点(用户权威修正)

原设计「类4 脚手架族」= 10 字母(A/a/Ā/B/b/B̄/C/c/C̄/Λ),设计成**纯调制器(算子的算子)**:
本身无 F/Z/P 功能,只用乘性增益 `量=量₀·(1+gain)^n` 拧旁边插槽基元产出。
**用户否决整套抽象**:① 不存在"脚手架"这个类;② 每个基元都是功能单元(都有自己的 F/Z/P + base 产出);
③ "backbone"现在只指"一条株序列里定死不突变的那些位置",坐上面的是真功能基元、只是被锁住不变;
④ 这天然解冷启动(插槽全空时 backbone 自带功能基元就给该株真实 F/Z/P 代谢);⑤ 用户倾向 backbone 功能基元自带 base 产出(乙)。

## ★核心决议(用户已拍:κ 方案 + 拆脚手架新模型)

### 1. 统一基元模型
一个基元 = 注册表一行 `formula(x, i) → 输出束`,x=完整序列,i=所在位。
输出束 = {F-输出(f/波及格/p_leave/period) ∪ Z-输出(z/猎物清单/period) ∪ P-输出(rate/spectrum/period) ∪ base 产出} 的任意非空子集 + 执行字段。
**坐 backbone vs 坐插槽唯一区别 = 突不突变**(backbone 位 locked 突变算子跳过 / 插槽 X mutable)。基元身份与位置无关,位置只决定可变性。

**作废**:类4 脚手架整类(10 字母)、"算子的算子"高阶签名 `formula(邻居输出,上下文)→修正输出`、字母族 S 族(rank4)→ 族降为 4 档 `N=0<F=1<P=2<Z=3`。
**保留**:统一注册表模型、字母族(去S)、residue/motif 双粒度、F/Z/P/N/V 全部基元(从来不是脚手架)、各基元强弱档(formula 锁死锚)。

### 2. ★上位效应 epistasis = κ 同通道族自协同(用户拍定,архитектор 选型 iii)
拆脚手架只拆掉 epistasis 三个源里的**源①(脚手架乘性调制)**;源②(P(x)读全序列)、源③(折叠区内外)原生存活。
源①重铸为"基元读同通道族邻居"(pull 上下文算自己,非 push gain 给邻居 → 通道正交天然守住):
```
量_adj(i) = clamp( 量_base(i) · (1+κ)^{ n_same(i) } )
n_same(i) = Σ_{j: 0<|i−j|≤W_max}  W(|i−j|) · [ 通道族(s_j)==通道族(s_i) ]
W(d) = 1/(1+d)        # 谐和衰减无参,原协议 A 档已锁式直接搬
```
- **私货面骤降:9 个按字母 gain → 1 个全局 κ**(对所有基元同一规则,过 G1/G11)。
- **κ=0 即退化为纯路径(i)位置无关无上位效应** → 平滑梯度要不要 = κ 一个常数的事,数值轮设,可先取 0 跑通本体。
- 否决方案(ii)"每基元自带 gain 拧邻居" = 脚手架换名复活(每基元都带 gain 比旧模型更脏 + 破通道正交)。
- 计算管线四步 A→B→C→D **结构不变**,仅 Step B 调制源从"脚手架字母"换成"同通道族邻居 + 全局 κ"。

### 3. 通道特化 / 物种身份 = backbone-locked 功能基元组合(转化)
物种身份 = 哪些功能基元被锁在 backbone 上。多锁 F 族=扩张型 / 多锁 Z=战士型 / 多锁 P=进化型。
通道偏好被 Step B 自协同强化(backbone 锁 F → 给周围 F 基元提供同族邻居 → 协同放大)→ "天生擅长 f"机械涌现,不靠手设 gain。
**通道权衡**:原靠抑制器(负 gain)→ 改由插槽预算守恒 + 通道正交承担(装 Z 必挤掉 F 的位),更干净。
**作废**:A=f/B=z/C=p 字母通道特化、抑制器 Ā/B̄/C̄ 换路线、四体质字母示例。**保留**:始祖株骨架=身份证、长度可变+插槽预算守恒。

### 4. 折叠保留 / Λ 删除
折叠机制**完整保留**(本就挂 backbone 折叠配对表、不依赖脚手架字母):Z+P/Z+F/P+F 跨类新通道、抗被猎 1−β_fold、α₀ 全局,全部存活。
**Λ 字母作废**:BB0 早用"显式成员列表 {1,3,4,5}"声明折叠区,Λ 一直冗余 → 折叠端点信息全交显式成员列表,Λ 不再占序列位。
连带:Scaffold Piercer 改"瞄折叠参与位、命中暂停对手折叠组合技1tick";Ghost Spike(原瞄S族)改瞄 N 族或 backbone-locked 基元所属族。

### 6. 冷启动死锁结构性消失
backbone-locked F4Nr1 自带 base f>0 → 拷贝流动 → μ 地板每复制触发 → 突变进插槽 → 级联启动。结构解非补丁。
**`builtin_primitives` 一等字段不再需要**:被 locked/mutable 统一模型吸收(内建=恰好锁了非N₀基元的 locked 位);**"占惰性位"方案也被吸收**(没有待占的特殊惰性位,每个 backbone 位都装着某基元)。突变算子跳过所有 locked 位。

## 私货红线在新模型下(守门人 abf9cc)

- **新增私货面 + 堵法**:backbone 坐真功能基元比坐纯调制器更易藏私货("给某 backbone 定死强 F"=直接写 `f_baseline[species]`=G1 违规)。**判定规则:放哪个基元/放哪个位=合法物种身份(what/where);基元产出多强=必须走全局注册表,backbone 无权本地覆写(how-strong 禁止本地化)。** base 产出必须是 `base[primitive_name]` 全局表,绝不能 `base[backbone][primitive_name]`。
- **私货免疫定理仍成立**:只依赖"四拷贝全同",拆脚手架没动这个前提;反而少一个 gain 私货通道更干净。内建 Z 需 G10 同株免疫兜底(已在协议)。
- **G7-G10 改写**:G7 同一性不变(最硬);G8 中弱可反超从"gain档"换"f/z/p 产出档"(F4Nr1 0.30 < 变体 F_NOVA 0.80);G9 盲选=哪些基元锁进 backbone/放哪位由对株盲规则一次性定(新模型最大新攻击面,挑的是真功能件);G10 不变更关键;新增 G8b base 产出对称。
