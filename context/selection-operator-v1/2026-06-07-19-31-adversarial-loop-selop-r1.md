## 2a — 攻击 + 不变量体检(multiagent-debate: critic-defender-judge)

**【CRITIC 开局攻击】**

指控 1(核心循环):老方案 Layer 1 的验证逻辑是闭环自证。手搓 f*(P01)→ 仿真器在 f* 驱动下生成轨迹 X(P04)→ 算法从 X 反推 f̂ → 度量 f̂ 与 f* 的距离(P06)。结论"f̂≈f*"对真实选择算子 f_真 零证据力,因为 f* 是人编的、与 f_真 无任何约束关系。这是教科书级循环论证。

指控 2(流程伪装):三层 gate(P20)把这个自证过程包装成"L1→L2→L3 严谨流程",制造"我很科学"的错觉。但 Layer 1 整层没有任何真值来自真实系统,gate 再多也是在自证内部打转。

指控 3(预防性攻击,打到 defender 的退路):就算你搬出 CR9114 独立 Kd,我也质疑——你用 f̂ 预测留出突变组合的 Kd 排序,预测对了,凭什么说是"学到了选择算子"?会不会只是 f̂ 抓住了"突变数越多越可能高亲和"这种平凡统计规律?平凡统计也能在 held-out 上预测对,但它跟"选择机制"无关。你的非循环只是把"循环自证"换成了"平凡相关性冒充机制理解"。

**【DEFENDER 应战】**(弹药:Stage 0 附录 I 理论收束 + 8 先例 + CR9114 O₁O₂)

回应指控 1+2:**全部承认,这正是我要砍的。** Layer 1 的 f*-比对路径(P01/P04/P06)确是循环,三层 gate 外壳(P20)确是流程伪装。我不辩护它们——我砍它们。

回应指控 3(这是真正的战斗):非循环性的解法是结构性的,不是"换个统计检验绕过去"。把验证真值来源从模型内部(f*/仿真器)换成系统外部(湿实验 O)。我有三类外部真值:
- O₁ 留出突变组合的实测 −logK_D(CR9114,同抗原内 held-out)
- O₂ 留出抗原的实测 −logK_D(cross-condition,H1+H3 学→预测 B)
- O₃ 留出轨迹分布(若保留动力学成分)

针对你的"平凡统计"指控,我有两道防线:
- 防线 N3(打败 baseline):f̂ 必须显著优于朴素 baseline(序列相似度 / Phillips 的 pairwise epistasis 模型)。如果 f̂ 只抓住了"突变数→亲和力"的平凡规律,那 pairwise epistasis baseline 也能抓住,f̂ 不会显著更好。f̂ 显著胜出 = 它学到了 baseline 抓不住的高阶/条件依赖结构。
- 防线 B1(cross-condition,O₂,杀手锏):**平凡统计无法跨抗原。** "突变数越多越亲和"这种规律若存在,在 H1/H3/B 上是同向的;但 CR9114 的真实结构是 nested epistasis——同一突变在不同抗原上效应不同,对 B 的亲和力需要 H1/H3 不需要的特定 nested 组合。我在 H1+H3 上学 f̂,预测留出的 B 抗原 Kd 方向。如果 f̂ 只会平凡统计,它预测 B 必然失败(因为 B 的 nested 结构跟 H1/H3 不同);只有真正捕捉了"条件如何改变选择"的 f̂ 才能预测对 B。这一关平凡相关性必死,机制理解才能过。

回应"会不会还是循环":O₁O₂ 是 Phillips/Dailey 实验室用湿实验(Tite-Seq/DMS)测的,在我的生成过程之外。我无法通过调整模型来"制造"这些 Kd 的吻合——它们是固定的外部事实。这就断了循环。我还主动把 SBC 类自洽检验降级为 debug 工具,绝不当证据。

**【JUDGE 裁决】**

对**指控 1+2**:成立。v0 原样(依赖 P01/P04/P06 比对手搓 f* + P20 流程外壳)在 I1 上**判红**——核心 claim 真值全来自内部自证。这些部件确认为 R1 下的简化目标。

对**指控 3 vs defender 防线**:defender 的结构性解法成立,但**有条件**。裁决分层:
- defender 的"换外部真值 O"方向正确,这是非循环的正解(与 Abbeel-Ng/scDiffEq 范式一致)。
- 但 critic 的"平凡统计冒充机制"是真实威胁,defender 的防线 N3+B1 **必须落实为强制判据**才算数。即:任何声称"非循环验证通过"的方案,必须同时满足 N3(显著打败 epistasis baseline)+ B1(cross-condition 留出抗原预测成功)。只做"留出突变组合预测"(O₁)而不做 cross-condition(O₂)的,judge 不判 I1 绿——因为 O₁ 单独可能被平凡统计刷过。

**I1 裁决**:对"砍掉 P01/P04/P06/P20外壳、改用 CR9114 的 O₁+O₂ 且强制 N3+B1"的方向 → **I1 判绿(有条件:B1 cross-condition 必须是核心 gate 而非可选)**。对 v0 原样 → I1 红。

这个裁决比开局预案更严:**B1 cross-condition 从"加分项"升格为"I1 判绿的必要条件"**。理由:critic 的平凡统计攻击证明 O₁(同抗原 held-out)单独不足以排除非机制性预测,必须靠 O₂(跨抗原)的机制鉴别力。

### 三不变量快速体检(本轮强制)

**I1 体检**(专项 critic-defender-judge,见上):🟢 绿(有条件)——非循环路径成立,但 B1 cross-condition 升为必要 gate。当前 v0 原样为红,简化方向(锚 O₁+O₂)为绿。

**I3 体检**:⚪→🟡 微升。本轮 defender 的 B1 防线实际上**预先动用了 I3 的机制**:cross-condition 预测能力 = 条件依赖被真正学到。这说明 I1 的非循环性(B1)与 I3 的条件依赖在 CR9114 上是同一件事的两面——cross-antigen 泛化既证非循环(外部真值)又证条件依赖(用上了 cond)。真值场(CR9114 三抗原)就绪,但打乱标签对照(P13)本轮未执行,I3 仍未正式验证,标 🟡(机制已被 B1 间接调用,待 R4 正式打乱对照确认)。

## 2b — 张力提炼(deep-insight → problem-reformulation: tension-mining)

从 2a 的对抗结果抽本轮核心张力。

**候选张力筛查**:
- 候选 1:野心(逼真仿真完整时序)⟂ 非循环(只信外部 O)。这是开局预案预判的张力,2a 确实出现(critic 指控 1+2 vs defender 砍仿真器)。但它在 2a 已被干净解决——defender 直接承认砍,没有真正的"卡住",张力强度低。
- 候选 2(2a 实际浮现的真张力):**非循环验证的"鉴别力" ⟂ 验证的"最小性"**。critic 指控 3 揭示:最小的非循环验证(只用 O₁ 同抗原 held-out)不足以排除平凡统计冒充机制;要获得真正的鉴别力,必须加 O₂ cross-condition。即"越简单的验证越可能被平凡相关性蒙混,越有鉴别力的验证越不能省"。
- 候选 3(更深层,T-core 的影子):预测下游可观测量(Kd)的能力 ⟂ 理解上游机制(选择算子)。f̂ 在 held-out Kd 上预测对,不直接等于"理解了选择机制"。

**本轮核心张力(精确表述)**:

> **T-r1:验证的非循环性不是"有没有外部真值"的二元问题,而是"外部真值有没有机制鉴别力"的程度问题。** 用 O₁(同抗原留出突变组合)做验证是非循环的(真值外部),但它鉴别力不足——平凡统计("突变多→亲和高")也能通过 O₁。真正断开"伪非循环"(用外部真值刷平凡相关性)需要 O₂ cross-condition,因为只有"换抗原后预测仍准"才证明 f̂ 捕捉了条件依赖机制而非表面统计。

**卡在哪个部件**:卡在 **P17(f̂ 对变体×抗原打分对照)** 的设计粒度上。P17 若只做"同抗原内留出突变组合预测"(对应 O₁),它通过了"非循环"形式但没通过"机制鉴别";P17 必须升级为"跨抗原留出预测"(对应 O₂ + 强制 N3 baseline)才真正承重 I1。换言之,卡点不是"要不要 P17",而是"P17 的验证轴必须从'突变组合轴'升到'抗原条件轴'"。

**张力的意义**:这条张力把用户的"非循环"诉求精确化了——不是"找个外部真值就算非循环",而是"外部真值必须有能力证伪'平凡相关性冒充机制'"。这比 spec 原本的 I1 定义("用方法没见过且非手搓的真值检验")更锋利:加了一个"鉴别力"维度。这是 loop 硬化 I1 的实质进展,不只是砍工程。

### 2b 产出

- **核心张力 T-r1**:非循环性 = 外部真值的"机制鉴别力"程度问题,非"有无外部真值"二元问题;O₁ 非循环但鉴别力不足,需 O₂ cross-condition 断开伪非循环。
- **卡点部件**:P17 的验证轴必须从"突变组合轴(O₁)"升到"抗原条件轴(O₂)+ N3 baseline"。

## 2d — 抢救被砍部件(stress-test → counterfactual-probing: necessity-sufficiency + factor-removal)

对 2c 砍的每个部件做 resurrection。站在该部件一方论证其承重;necessity-sufficiency 判 load-bearing;factor-removal 验砍后三不变量是否塌。救活→复活回滚;救不活→真冗余。

### 被砍部件逐个抢救

**P01 手搓 f* 族**：
- 抢救论证：没有真值 f* 就无法精确度量"恢复了多少",失去定量反馈。
- necessity-sufficiency：PN(必要性)——砍掉 P01,I1 是否塌？否，I1 靠外部 O 不靠 f*。PS(充分性)——P01 在场是否保证 I1？否，P01 是循环来源,反而害 I1。
- factor-removal：砍 P01 → I1🟢(更干净)、I2 不变、I3 不变。三不变量无一因砍 P01 而塌,反而 I1 更健康。
- **裁决：抢救失败 → 真冗余**。P01 是循环病灶,砍除是 I1 的净收益。

**P02 IGoR naive 库 / P04 DIMSIM-SLiM / P05 多时间点轨迹（逼真仿真流水线）**：
- 抢救论证：仿真器提供 CR9114 没有的——(a)轴 T 演化时序、(b)已知真值下"从轨迹反推 f"的测试场。
- necessity-sufficiency：
  - 对(a)时序：PN——砍掉后 I1/I2/I3 是否塌？否。时序是部署野心(R5),非方法成立必需。CR9114 静态地形已足以验证 f(seq,cond) 的条件依赖。
  - 对(b)反推测试场：PN——砍掉**整条逼真流水线**后,可辨识性测试是否无处可做？这是关键。答：逼真度(IGoR真实naive库 + EPAM真实突变谱 + SLiM真实群体遗传)对"测试反推动作是否数学可行"零贡献——一个玩具 SDE(已知简单f + 已知σ)就能测可辨识性。逼真流水线 = 用高成本提供一个低成本玩具就能给的功能。
- factor-removal：砍逼真流水线 → I1 不塌(靠 O)、I3 不塌(靠 CR9114 三抗原)、I2 **暂悬**(可辨识性测试需要替代载体,但那是 P07 玩具 SDE 的事,不是逼真流水线的事)。
- **裁决：抢救失败 → 真冗余**。逼真度对三不变量零贡献;其唯一承重的最小功能(可辨识性测试)由 P07 以远低成本接管。**注意：这不等于砍掉可辨识性测试本身——P07 在 R3 待审,本轮不砍。**

**P06 比对 f* 的可辨识性度量（原形态）**：
- 抢救论证：度量可辨识性这个意图承重(I2)。
- necessity-sufficiency：PN——砍掉"比对 f* 的度量形态"后 I2 是否塌？否,因为度量意图可由"比对外部真值"或"玩具 SDE 自恢复"实现,不必比对手搓 f*。
- factor-removal：砍 P06 原形态 → I2 不塌(功能转 P07)。
- **裁决：原形态真冗余,但功能转移 P07(R3 确认)**。不是净砍,是形态替换。

### 附录 r1-B:r2 战斗预案(主攻 R2 仿真器必要性 + 预备 R3)

**r2 主攻问题(R2 精确化)**:Spike0(CR9114 直接拟合静态 f)是否充分支撑"方法学到了选择算子"这一核心 claim?还是说,因为 North Star 定义选择算子是动力学量(SDE drift ∇f),必须保留一个"从演化反推 ∇f"的动力学步骤?

这就是 T-core 张力的正面决战。预排:

**critic(站在"必须有动力学步"一方)**:Spike0 学的是"序列→平衡亲和力"的静态地形 f(seq,cond),这是终点快照的回归。但 North Star 要的选择算子是"驱动群体演化的力场 ∇f"——一个动力学量。静态地形 ≠ 动力学驱动力:真实选择可能有路径依赖、频率依赖(frequency-dependent selection)、上位历史依赖,这些在静态 Kd 地形里全丢了。你拟合 CR9114 静态 Kd,顶多得到"适应度地形",不是"选择算子"。砍掉所有动力学成分 = 偷换问题,把 SDE 反问题降级成普通 DMS 回归。

**defender(站在"Spike0 可能充分"一方)**:在 gradient-flow 假设下(gating①),选择算子的势函数 f 就是适应度地形,∇f 是其梯度——拟合静态 f 再取梯度即得驱动力,两者统一。CR9114 的 Kd 地形若就是 f,那 Spike0 直接学到了 f,∇f 唾手可得。是否需要独立动力学步,完全取决于 gating① 是否成立——这必须实测(附录 A 检验),不能靠嘴辩。

**预期 judge**:这一仗 judge 无法纯逻辑裁决,**必须依赖 gating① 的实测结果**。所以 r2 可能的走向是:judge 判"R2 悬而未决,转 R3 实测 gating①",或 r2 直接并入 R3 一起打(spec 允许一轮主攻多维度)。预判 r2 会把 R2 和 R3 合并处理,因为它们都系于 gating①。

**gating① 实测方案(R3 核心动作,附录 A 已备)**:需要一条轨迹做检验。但 r1 已砍掉逼真仿真器,哪来轨迹?这正是 P07 玩具 SDE 复活的触发点——用玩具 SDE(已知简单 f + 已知 σ)生成一条受控轨迹,在 PLM 表征空间检验:(a) 增量是否近高斯(Itô);(b) drift 是否近无旋(梯度场);(c) 最关键——"静态拟合 f 取梯度"与"从玩具轨迹反推 drift"是否一致。一致→分支 A(Spike0 等价动力学,极简);偏离→分支 B(需独立动力学步)。这个一致性检验本身就是 P07 承重的铁证(没有它,T-core 无法裁决)。

**P07 复活预判**:r2/r3 中 P07 几乎必然复活,但复活的形态取决于检验结果——分支 A 下 P07 是"一次性 gating 检验工具"(验完即可,不进定稿方案核心),分支 B 下 P07 升格为"定稿方案的动力学反推核"。两种形态成本都远低于逼真仿真器。

### 附录 r1-D:CV 条件轴验证模块详细规格(r1 产出,Stage 4 直接操作化)

r1 把 P12/13/17/18 合并为 CV 模块,这里给出它的详细规格,作为 v1 的核心验证组件,Stage 4 实验设计直接取用。

**CV 模块目的**:在 CR9114 三抗原(H1/H3/B)数据上,一站式验证 I1(非循环 + 机制鉴别力)与 I3(条件依赖),因为 r1 发现两者同源。

**CV 模块的四个子检验(原 P12/13/17/18)**:

CV-a(原 P12,正向条件依赖):固定一组变体,扫三个抗原,观察 f̂(x,H1)/f̂(x,H3)/f̂(x,B) 是否实质不同,且差异模式与实测 −logK_D 的 nested 结构一致。通过判据:f̂ 的跨抗原变化与实测方向相关性显著(如 Spearman > 预注册阈值)。

CV-b(原 P13,打乱标签对照,I3 核心):把训练集 (变体, 抗原, −logK_D) 的抗原标签随机置换,重训 f̂_shuffled,比较其测试性能与正常 f̂。通过判据:f̂_shuffled 性能显著低于 f̂(落入打乱分布尾部,如 p<0.05)。这证明 f̂ 真用上了条件信息,非退化成 f(seq)。

CV-c(原 P17 升级,cross-condition 非循环 gate,I1 核心):在两个抗原(如 H1+H3)上训练 f̂,在留出的第三抗原(B)上预测变体的相对 −logK_D 排序。通过判据(双重):(1) 预测 Spearman 显著 > 0;(2) **显著优于 N3 baseline**(序列相似度 + Phillips pairwise epistasis 模型)。这是 r1 升级的核心——cross-antigen 留出 + 打败 baseline,断开伪非循环。

CV-d(原 P18,条件依赖真值复现):同一变体跨抗原,检验 f̂ 的 Kd 变化方向是否与独立实测一致(I1+I3 交汇)。通过判据:同变体跨抗原的 f̂ 差与实测 Kd 差方向一致率显著。

**CV 模块的零泄漏前提(P16)**:CV-c 的留出抗原 B、CV-a/d 用于对照的实测 Kd,必须确认未参与 f̂ 训练。留出策略:按抗原留出(B 完全不进训练)+ 按突变组合留出(部分 2^16 组合留出)。

**CV 模块输出**:四个子检验的量化结果 + I1 判定(CV-c 通过=非循环+鉴别力成立)+ I3 判定(CV-b 通过=条件依赖成立)。

**CV 模块与老三层的对比**:老方案用 Layer 2(扫条件 + 打乱)+ Layer 3(独立 Kd 对照)两层、跨两批数据、串行 gate 来做这些;CV 模块在单一 CR9114 数据上、一个模块内、并行完成,且 I1/I3 判据更严(强制 cross-condition + baseline)。这是 r1 "合并简化 + 同时硬化" 的典型产物。

### 附录 r1-I:gating① 实测的 r2 落地细节(预备,使 r2 不停在口水仗)

r1 反复指出 T-core 决战系于 gating① 实测。为使 r2 能真正"测"而非"辩",这里把附录 A 的 gating① 检验落到 r2 可执行的最小步骤(玩具 SDE 载体):

**最小可辨识性 + gating① 联合检验(P07 玩具 SDE 承载)**:
1. 构造玩具:取一个低维(如 2-5 维)连续空间,设一个已知简单势函数 f_toy(x,c)(如二次型 + 一个条件依赖项),已知扩散 σ_toy。
2. 前向生成:跑 dX=∇f_toy·dt + σ_toy·dW,生成多条轨迹 + 多时间点分布快照(模拟我们真实数据的"population 边际"观测形态)。
3. 反推:用待验证方法从快照反推 f̂_toy。
4. 可辨识性判据:f̂_toy 是否恢复 f_toy(相关性/排序)?——回答"在已知 σ + population 边际下,反推 f 这个动作数学上 work 吗"(I2 最小验证)。
5. gating① 判据:在玩具的"PLM 类比"(若用真实 PLM,把玩具序列 embed)空间检验增量近高斯 + drift 近无旋。
6. **T-core 决战判据**:对同一个 f_toy,比较"静态拟合(x→f_toy 值的回归)取梯度"vs"从轨迹反推 drift"两条路径得到的 ∇f̂,是否一致。一致→分支 A(Spike0 静态拟合等价动力学,极简方案成立);系统偏离→分支 B(必须独立动力学步)。

**为什么这个检验本身不循环(重要)**:有人会问——你用玩具 f_toy 验证方法,这不又是"手搓 f* 反推"的循环吗?区别在用途:这里玩具 SDE **不是用来证明"方法学到了真实选择算子"**(那是 CV 模块用外部 CR9114 Kd 做的,I1),而是用来回答一个纯数学问题——"在已知 σ + 这种观测形态下,反推 f 的动作是否 well-posed(有唯一解)"。这是可辨识性(I2)的问题,不是"学对了没"(I1)的问题。可辨识性是数学性质,用已知 f_toy 验证它完全正当(就像用已知矩阵验证求逆算法,不循环)。circular 的是"用手搓 f* 证明学对了生物学",well-posed 的是"用玩具验证反推动作的数学适定性"。r1 的 I1 升级(锚 CR9114 外部真值)管前者,玩具 SDE 管后者,两者分工不冲突。

**这条厘清解决了一个潜在的 I1 红灯陷阱**:如果有人把"玩具 SDE 可辨识性检验"误当成"循环验证",会错误地砍掉 P07(以为它违反 I1)。附录 r1-I 明确:P07 玩具 SDE 服务 I2(数学适定性),不服务 I1(生物学正确性),所以它不触发 I1 循环红线。这是 r2/r3 审 P07 时 defender 的关键弹药——P07 的真值是"自己设的 f_toy",但它的 claim 是"反推动作适定",不是"学对了真实 f",所以不循环。
